## 三、Phase A: 快速修复（解除关键阻塞）

### A.1 Shadow calcium_rate 硬上限【步骤 3 最小版】

**问题**: EXP-015 显示 Shadow col calcium_rate 在 ~20k 步饱和到 0.97+，Xin 残差→0，DA 永久沉默。

**修改文件**: [shadow_sandbox.py](nexus_v1/components/shadow_sandbox.py)

**方案**: 在 CRI（CalciumRateIntegrator）的更新逻辑中加入硬钳位 `min(calcium_rate, 0.8)`。

具体位置需定位 `calcium_rate` 的赋值点。预计在 `ShadowSandbox.observe()` 方法中，spiking col neuron 的 `calcium_rate` 更新处添加：

```python
# 在 calcium_rate 更新后添加硬上限
n.calcium_rate = min(n.calcium_rate, 0.8)
```

**验证标准**: 运行 50k 步，shadow col calcium_rate 不再超过 0.8，DA 浓度维持 > 0。

**回滚**: 删除一行即可。

---

### A.2 基底自发运动注入【步骤 2 最小版】

**问题**: EXP-015 中无前庭输入 → 身体静止 → 热梯度恒定 → 无时变信号。L2.09 也证实当前"运动"是 VitalOscillator + deviation motor drive 产生的弱随机游走。

**修改文件**: [variant_adapter.py](nexus_v1/circuit/variant_adapter.py) （otolith 输入路径，约 line 642）

**方案**: 在 otolith 加速度信号上叠加弱自发噪声：

```python
# 当前:
OTOLITH_GAIN = 500.0
mechanical_inputs['oto_x'] = acc[0] * OTOLITH_GAIN

# 修改为（仅当加速度为零或很小时注入噪声）:
import random
BASELINE_OTOLITH_NOISE = 0.01  # 弱自发活动
_base_noise = random.normalvariate(0, BASELINE_OTOLITH_NOISE) if abs(acc[0]) < 0.001 else 0
mechanical_inputs['oto_x'] = (acc[0] + _base_noise) * OTOLITH_GAIN
```

需要对 `oto_x`, `oto_y`, `oto_z` 三轴均应用此逻辑。

**验证标准**: 零前庭输入下，Motor 有微弱但非零的发散输出，身体产生随机游走。

**回滚**: 将 `BASELINE_OTOLITH_NOISE` 设为 0 或删除噪声注入行。

**修改文件**: [hebbian.py](nexus_v1/circuit/hebbian.py) （`_motor_config` 函数）

## 四、Phase B: B.06 ν_th 实现（核心缺失）

**依赖**: Phase A 完成
**关键性**: 🔴 **这是项目关键路径阻塞点**。自足文档将其列为步骤 1。

### B.1 数学定义

```
ν_th = dE_thermal/dt = (T_skin[t] - T_skin[t-1]) / dt

核心关联:
  ν_th × ΔT > 0  →  朝热源移动，皮温升高 → DA↑ → 强化当前方向
  ν_th × ΔT < 0  →  远离热源，皮温降低 → DA↓ → 促进转向
```

这是 **klinokinesis**（细菌趋化性核心机制）—— 不是"知道热源在哪"，而是"知道自己是不是在变暖"。

### B.2 MotionState 扩展

**修改文件**: [motor_decision.py](nexus_v1/circuit/motor_decision.py)

在 `MotionState` dataclass 中新增字段：

```python
# 热势 ν_th
thermal_potential: float = 0.0       # dE_thermal/dt（当前步）
thermal_potential_ema: float = 0.0   # EMA 平滑版
prev_skin_temperature: float = 0.0   # 上一步的平均皮温（内部用）

# 已有的 thermal_gradient 字段（确认存在，如果不存在则新增）:
thermal_gradient: List[float] = field(default_factory=lambda: [0.0, 0.0, 0.0])
```

### B.3 皮肤温度采样与 ν_th 计算

**修改文件**: [variant_adapter.py](nexus_v1/circuit/variant_adapter.py)

在 `variant_adapter.py` 的主循环（或 `_compute_motion_state` 方法）中：

```python
# 1. 采样当前所有皮肤补丁的平均温度
patch_temps = [p.current_temperature for p in self.world.body.skin_patches]
current_avg_temp = sum(patch_temps) / max(len(patch_temps), 1)

# 2. 计算 ν_th（如果 prev 不为 0）
if ms.prev_skin_temperature > 0:
    ms.thermal_potential = (current_avg_temp - ms.prev_skin_temperature) / dt
    # EMA 平滑
    alpha_th = 0.1  # 热势 EMA 系数
    ms.thermal_potential_ema += alpha_th * (ms.thermal_potential - ms.thermal_potential_ema)
else:
    ms.thermal_potential = 0.0
    ms.thermal_potential_ema = 0.0

ms.prev_skin_temperature = current_avg_temp

# 3. 计算热梯度向量（前-后, 左-右, 上-下）
# 需要根据 patch_id 获取温度
temps = {p.patch_id: p.current_temperature for p in self.world.body.skin_patches}
ms.thermal_gradient = [
    temps.get('front', 0) - temps.get('back', 0),   # x 方向梯度
    temps.get('left', 0) - temps.get('right', 0),   # y 方向梯度
    0.0,                                              # z 暂不处理
]
```

### B.4 ν_th → DA 映射

**修改文件**: [variant_adapter.py](nexus_v1/circuit/variant_adapter.py)（DA 释放逻辑附近，约 line 663）

```python
# ── B.06: ν_th 驱动的 DA 调制 ──
# klinokinesis: 变暖=好，变冷=坏
DA_THERMAL_GAIN = 2.0  # 热-DA 耦合增益（可调）

if ms.thermal_potential_ema > 0:
    # 变暖 → 释放 DA → 强化当前行为
    thermal_da = ms.thermal_potential_ema * DA_THERMAL_GAIN
    for nid, neuron in self.da_neurons.items():
        neuron._membrane.inject(thermal_da, dt)
elif ms.thermal_potential_ema < 0:
    # 变冷 → 不释放 DA → 促进探索/转向
    # （DA 基线由 C3' 环流偏差和 bias_current 维持）
    pass
```

## 六、Phase D: 后续优化（条件性，仅在 Phase C 未达标时启动）

### D.1 BCM 滑动阈值扩展到影子层

**触发条件**: Shadow col 在硬上限后仍然失去分化

**方案**: 在 `shadow_sandbox.py` 的 `create_shadow_config` 中为 col 神经元启用 BCM（`theta_m_tau` 已设为 10000），并在 ShadowSandbox 的 `observe()` 中调用 BCM 更新逻辑。

### D.2 Shadow STDP 衰减加速

**触发条件**: 权重持续增长不收敛

**方案**: 调整 shadow bundle 的 `w_decay` 参数（当前默认 0.001），增加至 0.005-0.01。

### D.3 热梯度方向信号放大

**触发条件**: ν_th 信号太弱，DA 没有足够调制深度

**方案**: 增加 `DA_THERMAL_GAIN` 或对 ν_th 进行非线性放大（sigmoid/ReLU）。

### D.4 DA 脉冲式释放

**触发条件**: DA 浓度平坦、缺乏时间结构

**方案**: 改为脉冲式 DA 释放：仅在 ν_th 符号翻转时（冷→暖或暖→冷）释放一过性 DA 脉冲，而非持续注入。

### 7.1 每次修改的保护

```
修改前: 运行 test_regression.py → 记录基线（当前 20/21 PASS）
```

### 7.2 回滚层级

| 层级 | 操作 | 耗时 |
|------|------|------|
| L1 参数回滚 | 恢复参数原值 | 10 秒 |
| L2 功能禁用 | 设置开关=False | 30 秒 |

### 7.3 已知风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 硬上限导致 Shadow 完全失活 | 低 | 高 | 0.8 是保守值，可调至 0.9 |
| 基底噪声导致随机游走过大 | 中 | 中 | 从 σ=0.01 开始，可降至 0.001 |
| ν_th 信号太弱无法驱动 DA | 中 | 高 | DA_THERMAL_GAIN 从 2.0 开始可调 |
| 与 C3' 环流 DA 释放冲突 | 低 | 中 | ν_th 和环流偏差分别注入 DA 膜，不冲突 |

## 十、成功定义

### v1.0 最低可接受标准（任一满足即可标记 v1.0）

1. **热趋性涌现**: 100k 步中，身体向热源的位移 > 随机游走基线（无梯度对照组的位移分布）
2. **停留行为**: 身体到达热源附近（d < 5）后，平均速度降低 > 30%
3. **DA 不崩溃**: DA 浓度在 100k 步全程保持 > 0，且与热梯度方向有统计显著的相关性

### 如果全部不满足

→ 热趋性在当前架构下不可实现。需要重新评估：是否缺少了更根本的机制（如空间记忆、路径积分、或 efference copy）。此时转为 Phase D 深度分析或架构重设计。
