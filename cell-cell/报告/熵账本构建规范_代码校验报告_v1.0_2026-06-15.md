## 二、逐条校验

### ❌ V01 — 体感层不可见（修复方案错误）

**规范主张**: SomatosensoryChain 的 12 个神经元不在 `get_all_neurons()` 中，修复 = 追加 `self.somatosensory.*`

**代码实测**:
```
hasattr(VariantCircuit(), 'somatosensory')  →  False
test_smoke_soma.py  →  AttributeError: 'VariantCircuit' object has no attribute 'somatosensory'
soma/therm 相关属性  →  只有 'thermal_membrane'（ThermalMembrane 类）
```

**真相**:
- `VariantCircuit` **根本没有 `somatosensory` 属性** —— `SomatosensoryChain`（12神经元）**从未被实例化**到运行电路中
- 当前热感知走的是 **`ThermalMembrane` + 单一 `therm` 轴**（extra_axes=['therm']）
- `therm` 轴的编码/柱神经元（`enc_reg_therm`/`enc_irr_therm`/`col_therm`）**已经在熵账本中被追踪**（实测 ISI 熵：enc_reg_therm=0.244bits, col_therm=1.408bits）

**判定**: ❌ 修复方案 `neurons.extend(self.somatosensory.*)` 会抛 `AttributeError`。规范（连同源头的"盲区审计"文档）分析的是一个 SomatosensoryChain 已集成的版本，但当前主线代码用的是 ThermalMembrane 抽象方案。

**正确的缺口表述**: 不是"体感层不可见"，而是"SomatosensoryChain（12神经元的精细热/痛/中继链）在当前 VariantCircuit 中根本未接入；当前 therm 轴是抽象单轴方案且已被追踪"。若未来要用 SomatosensoryChain，需先在 `__init__` 中实例化并接入 `step()`，再谈熵账本覆盖。

### ✅ V02 — GovernanceLedger vs EntropyLedger 增益链矛盾（正确）

**代码实测**（相同 5000 步 oto_x 正弦输入）:
```
EntropyLedger 相关系数:     GovernanceLedger _effective_activity:
  L1→L2: 0.977 ✅              L1→L2: 0.0000 ❌
  L2→L3: 0.136 ✅              L2→L3: 0.0000 ❌
```
根因确认: `governance/ledger.py` `_effective_activity()` 对 hc_ 前缀读 `release_rate`（≈0），对 met_ 读 `|activation|`（0.97），量纲不一致 → 虚假零增益。

**判定**: ✅ 主张正确，根因定位准确。

### ❌ V03 — Fuse 熔断器不可见（根因错误）

**规范主张**: F3 仅当显式创建 GovernanceSystem 时可见，常规测试不创建 → 超界消失

**代码实测**:
```python
# variant_adapter.py L313-317: __init__ 中无条件创建
self.governance = GovernanceSystem(GovernanceConfig(debug_mode=False))

# variant_adapter.py L717-719: step() 中每步调用
if self.governance is not None:
    self.governance.post_step(self, tick, dt)   # ← fuse.check() 每步执行

实测: governance exists=True, fuse enabled=True, debug_mode=False
```

**真相**: GovernanceSystem **在每个 VariantCircuit 中都被创建**，`post_step()`（含 `fuse.check()`）**每步都执行**。Fuse 在常规运行中**始终在线**。规范的根因"常规测试不创建 GovernanceSystem"是**错误的**。

**判定**: ❌ 根因错误。Fuse 不触发的真实原因见 V09。

### ❌ V09 — F3 权重超界（事实错误）

**规范主张**: met_to_hc/hc_to_aff 权重达 5.8 > weight_max=1.0

**代码实测**:
```
最大权重 5.0827 → 位于 xin_to_da bundle（weight_max=10.0，在界内！）
met_to_hc_yaw:  w=0.53  (weight_max=1.0)   ✅ 界内
hc_to_aff_yaw:  w=0.92  (weight_max=0.95)  ✅ 界内
运行 3000 步: Fuse trips=0, 无崩溃
```

**真相**: 那个 5.8/5.08 的权重在 **`xin_to_da`** 束上，其 `weight_max=10.0`，**完全在界内**。met_to_hc/hc_to_aff 从未超界。**根本不存在 F3 违反**——Fuse 不触发是**正确行为**，不是 Bug。

> ⚠️ 注：此错误也存在于早期的《熵账本系统实测分析报告》中（误把 5.8 归因于 hc_to_aff）。本次校验予以更正。

**判定**: ❌ 事实错误。无 F3 违反，Fuse 工作正常。

### ⚠️ V11 — Xin 守恒未实现（部分错误）

**规范主张**: Xin 可漏可凭空出现，Noether/Xin 界限是软上限而非真正守恒

**代码实测**: `noether_probe.py` L243-273 **有严格的 Xin 记账**:
```python
# Conservation: adj_produced - adj_consumed == delta_remaining
xin_imbalance = xin_imbalance_abs / scale
if xin_imbalance > self.XIN_TOL:   # XIN_TOL=0.01
    → 记录 xin_conservation violation
```

**真相**: Xin 守恒**已实现且每步审计**（produced - consumed = Δremaining，容差1%，超限记违规）。实测 Noether violations=0 说明当前 Xin 守恒**通过**。规范"未实现"的说法**过时**。

**保留的合理疑点**: 概念演化链记录的"Xin 跨束全局相关性=0.99 导致度量坍缩"、"ρ 向量仍二元"是另一层面问题，与守恒记账无关。

**判定**: ⚠️ "未实现"错误；但 ρ 向量/测度坍缩的疑点（见 V12）成立。

### ⚠️ V12 — ρ 向量仍是二元（未充分验证）

**规范主张**: P/R/Xin 用二元有/无而非 8 分量连续测度

**校验**: 概念演化链文档确实记录此为 OPEN 项。代码层面 `xin_tension` 是连续标量，但 8 分量 ρ 向量（纤维测度）确实未见实现。需深入 bundle.py 的 P/R/Xin 模型才能完全确认。

**判定**: ⚠️ 方向可信，未做代码级穷尽验证。

### ❌ V13 — Ω 和 P_ν×H_flow 未实现（事实错误）

**规范主张**: Ω（系统规模参数）未定义，P_ν×H_flow≈const 约束未验证

**代码实测**: `noether_probe.py` 两者**都已实现**:
```python
# L356-360: Ω 已定义
n_sprouted = sum(1 for b in bundles if hasattr(b, '_sprout_depth'))
omega = n_neurons * n_bundles * max(n_sprouted, 1)
self._omega_history.append((tick, omega))

# L365-372: P_ν × H_flow 已实现
p_nu = getattr(ms, 'polarization', 0.333)
self._pnu_hflow_history.append(p_nu * h_flow_now)
```
SERIAL_DEPENDENCIES_v1.5 也记录: "P_ν × H_flow = 3.18 ± 0.59 (CV=0.185)" 已有验证数据。

**判定**: ❌ 事实错误。Ω 和 P_ν×H_flow 均已实现并有验证数据。
