# 运动势与时空测度：概念 · 定义 · 演化 · 开放问题

> **核心概念**: ds² = 时空测度（由影子层跨轴权重 w_cross 定义的内生度规），ν = 运动势（环流占比的变化率）。时空不是系统从外部读取的坐标——时空是系统通过自己的结构和运动**生成**的。
> **认识论基石**: [[docs/concept_C004_autogenous_spacetime]] — "自身 = 标尺"

---

## 一、基本定义

### 1.1 时空测度 ds²

**定义**:
$$ds^2 = \sum_{i,j} w_{\text{cross}}(i,j) \times a_i \times a_j$$

**各项含义**:
| 符号 | 含义 | 来源 |
|------|------|------|
| $w_{cross}(i,j)$ | 影子层跨轴权重（BCM 学习），**就是度规张量 g_ij** | `shadow_sandbox.py` cross-axis bundles |
| $a_i$ | 列层激活（自运动 + 外部输入） | `neuron.activation` |
| $ds^2 > 0$ | 空间关系（两个轴在同一方向上协同） | — |
| $ds^2 < 0$ | 时间关系（两个轴在因果上相关） | — |

**度规不是从外部给定的**——它是项目通过自身运动与感觉的关联学习出来的:

```
w_cross(yaw, pitch)  = "旋转和俯仰多耦合" → 自体几何
w_cross(oto_x, therm) = "x移动和热变化多耦合" → 外部对象的"方向"
```

### 1.2 运动势 ν

**定义**:
$$\nu = \frac{d\rho}{dt}, \quad \rho = \frac{\mu_p}{\mu_{\text{total}}}$$

**各项含义**:
| 符号 | 含义 | 物理对应 |
|------|------|---------|
| $\rho$ | 环流占比（某条路径的流量在总流量中的占比） | 注意力的分配 |
| $\nu$ | 占比的变化率 = "急迫程度" | 非平衡态驱动力的量度 |
| $\nu > 0$ | 环流占比上升 → 系统正在激活某个特定环流通道 | 积累误差，远离平衡 |
| $\nu < 0$ | 环流占比下降 → 系统正在回到更均匀的分布 | 趋近均衡，误差减少 |

**ν 不是物理速度——是项目感受到的自身状态变化的急迫程度。**

### 1.3 三个影子层计算量

| 量 | 符号 | 公式 | 含义 | 当前状态 |
|----|------|------|------|----------|
| **时空测度** | ds² | Σ w_cross(i,j) × a_i × a_j | 度规空间中的"距离" | 无人读取 |
| **运动势** | ν | dρ/dt = d(μ_p/μ_total)/dt | 环流偏移的急迫程度 | 无人读取 |
| **收缩/膨胀** | κ | std(activations) / mean(activations) | 信号丰富度 (κ>1=扩张, κ<1=收缩) | 无人读取 |

> **关键**: 三个量全部被计算，但**没有一个被任何下游系统读取**。影子层 = "高完成度的独立子系统，但完全断连。"

## 二、自身 = 标尺 — 内生时空的认识论

> 来源: [[docs/concept_C004_autogenous_spacetime]]

### 核心原则

**项目（生物体）没有访问 World.py 中的坐标函数。** 项目能感受到的只有输入流:
- 前庭输入流：col_yaw, col_pitch, ... col_oto_z（自身运动状态）
- 热输入流：col_therm（外部变化）
- 这些输入流之间的时序关联（绑定层 + 影子层）

### 具体的度量替换

| 度量 | 外部标准 | 项目内部标准 | 实现位置 |
|------|---------|------------|---------|
| **距离** | 米 | "移动了多少 → 热变了多少" | w_cross(oto_x, therm) |
| **方向** | 角度 | "转了多少 → 热变了多少" | w_cross(yaw, therm) |
| **时间** | 秒 | "多少个处理周期" + "能量耗散了多少" | dt, τ_RC, τ_methyl, ds²/ν |
| **速度** | m/s | "环流占比的变化率" | ν = dρ/dt |

项目不知道"热源在 x=65"。项目知道的是"往某个方向动 5000 步后温度变化 3.0"。
5000 步 × 身体尺度 = "那个方向有 5000 步远的热"。

### 认识论框架

> **这不是一个工程决策——这是项目的认识论立场。** 外部世界只通过输入流进入项目。时空只在项目内部通过自身结构生成。

| 哲学框架 | 对应 | 项目实现 |
|---------|------|---------|
| 康德先验直觉 | 空间和时间不是世界的属性，而是认知的形式 | ds² 和 ν 从结构生成 |
| 具身认知 | 身体是知识的基础 | 自身 = 标尺 |
| 自创生 (Autopoiesis) | 系统生成自己的交互域 | 权重学习 = 世界建构 |
| 自由能原理 (FEP) | 减少预测误差 | Xin = predicted - actual |

## 三、ds²/ν 的热力学含义

> 来源: [[2026-05/analysis_unified_master_method.2026.5.24]]

### ds² 和 ν 的量纲

```
ds² = Σ ΔV²_i × C_i    [V² × F = J] → 微分能量（系统的储能变化）
ν   = Σ I²_j × R_j    [A² × Ω = W] → 耗散功率（系统的即时功耗）
ds²/ν                   [J/W = s]  → 热力学时间
```

**ds²/ν 的物理意义**:

> 它是系统的"**热力学时间**"——不是钟表时间，而是"以当前耗散速率，系统的储能能维持多久"。
> 小鼠代谢快 → ds²/ν 小 → 生物时间快。大象代谢慢 → ds²/ν 大 → 生物时间慢。
> **REF: Kleiber's Law (1932): P ∝ M^{3/4}**

```
Motor → Body → Oto → Aff → Enc → Col → Motor'
   ↑                                     |

           闭环 = 空间的可能条件

  pre-before-post → LTP (增强)
```

## 八、驻波垫支（对话孤岛 B4）

> 来源: [[00_Dashboard/孤岛-B组-部分失联]] B4: 四维积体电路 + 驻波垫支

### 驻波垫支理念

"时间测度、空间测度、运动势都可以被系统结构垫支。垫支的实现形式就是结构，而垫支的内容由占据结构中信号轨迹的驻波来完成。"

**含义**: 影子层的结构**本身**就是一个预先铺设好的"轨道"，外部信息通过这些轨道时形成驻波（稳定振荡），驻波节点=信息位。

**四个维度**:
1. 拓扑维度 (T) — 突触连接的权重结构
2. 能量维度 (E) — 电压+电流+功耗分布
3. 时间维度 (τ) — 信号传输延迟 + 处理时间
4. 信息维度 (I) — Xin 张力 + 果实状态

这四个维度的张量积 = 影子层的"空间"。
