# 母本分化与补偿元件：设计 · 实现 · 缺陷 · 演化

> **核心主题**: "母本分化"（Mother Differentiation）——从一个统一的母本类（Neuron/Bundle）出发，通过配置差异而非代码分支来派生特化行为。
> **衍生主题**: "新元件构建"——在母本分化的框架下创造新的补偿组件（CRI/DN/D2R/NDR/FatigueCapacitor等），满足特定神经类型的特定需求。

---

## 一、母本分化的定义与哲学

### 根本原则

**来自 `analysis_meta_guide.2026.5.20.md` + 对话日志导出**:

> "所有分化与特化结构都要建立在元结构组成的网上。元结构是最小积木。"
> "从母本出发，逐块建模、调整。不修改母本代码来添加功能——通过继承/适配器覆盖。"

### 两个并列概念

| 概念 | 含义 | 典型场景 |
|------|------|---------|
| **母本分化** | 同一母本类（Neuron/Bundle）通过**配置差异**（NeuronConfig）而非代码分支，产生不同行为 | col vs enc vs motor 各有不同 config |
| **新元件构建** | 在母本的配置框架内创建**新的补偿组件**（CRI/DN/D2R/NDR等），形成新的"可选分化路径" | 为 spiking neuron 创建 CRI；为 DA neuron 创建 D2R |
| **跑车分化** | 运行时的**自适应调整**——不是配置差异，而是响应物理条件的变化 | Motor 能量低时 adapt_threshold 上移 |

### 分化层次

```
Level 0: 母本 (Neuron / Bundle / HebbianCircuit)
  └── 基础功能: 膜电容、RC放电、MOSFET门控、STDP、能量核算

Level 1: 工厂分化 (Factory Pattern — config 级别)
  ├── 不同层: Met vs HC vs Aff vs Enc vs Col vs Motor → 各自的 config 工厂函数
  ├── 不同域: 影子层 vs 主层 → create_shadow_config vs _column_config

Level 2: 元件分化 (Compensation — 组件级别)
  ├── CRI (CalciumRateIntegrator) — spiking neuron 专有
  ├── DN (DivisiveNormalization) — 感觉编码 neuron 专有
  ├── D2R (D2Autoreceptor) — DA neuron 专有
  ├── NDR — 负微分电阻 → 双稳态/振荡
  ├── FatigueCapacitor — 疲劳型（Motor/Column）
  └── 未来: 任何新的分化组件都应在此层创建

Level 3: 运行时分化 (Homeostatic — 物理驱动)
  ├── adapt_threshold: 能量低时上移 spiking 阈值 → 降低放电率
  ├── BCM sliding threshold: 成熟度驱动的可塑性调节
  └── 突触缩放: 后突触活动过高时降低 synapse_gain
```

## 二、v0.9.0 母本分化（2026-06-06）

> 来源: [[2026-06/CHANGELOG.2026.6.6]] · [[01_工程执行/walkthrough.2026.6.6]]

### 新增的三个补偿组件 (compensation.py 组件 H/I/J)

| 组件 | 类名 | 物理对应 | 功能 | 目标 |
|------|------|----------|------|------|
| **H** | `CalciumRateIntegrator` | CaMKII 浓度 | 将离散 spike → 连续的钙速率 ∈ [0,1]。Capacitor 积分 + Zener MOSFET 钳位 | Shadow col (spiking projection) |
| **I** | `DivisiveNormalizationReceptor` | 视网膜增益控制 | 输出 = raw / (σ + pool)。pool 通过 Capacitor 低通滤波估计输入 RMS | Shadow enc (sensory input) |
| **J** | `D2Autoreceptor` | VTA D2 自受体 | DA 浓度 → GIRK K⁺ 电流 → 超极化 → 抑制 DA 释放。MOSFET (Shockley) + Capacitor | DA neurons |

## 三、浅拷贝 Bug（基因隔离手术）

> 来源: [[archive/undated/analysis_mother_differentiation_audit.母本分化 × 自由能 × 全局污染 — 深度审计]] · [[01_工程执行/walkthrough.Walkthrough-基因隔离手术-F6信息热寂熔断]]

### 问题本质

母本分化依赖 `Neuron.split()` 和 `SynapticBundle.sprout()` 创建子代。两者都用 `copy()`（浅拷贝），导致子代与母本**共享可变对象引用**。

### 三个污染点

| # | 文件 | 函数 | 共享对象 | 写入频率 | 风险 |
|---|------|------|----------|----------|------|
| 1 | `neuron.py` L672 | `Neuron.split()` | `channels: List[ChannelConfig]` | 当前不写，未来可能 | 🟡 定时炸弹 |
| 2 | `bundle.py` L646 | `SynapticBundle.sprout()` | `silent_snapshot: dict` | `_manage_silent_synapses` 每步写入 | 🔴 活跃 bug |
| 3 | `hebbian.py` L988/L1004 | `_rewire_after_split()` | `silent_snapshot: dict` | 同上 | 🔴 活跃 bug |

### 实测验证 (bug #2)

```
b1.silent_snapshot is b2.silent_snapshot: True   ← 共享同一个 dict
After b2.silent_snapshot['test'] = True:
  b1.silent_snapshot = {'w': 0.5, 'test': True}  ← 母本被污染
  CONTAMINATION: True
```

### 修复（基因隔离手术）

**三处 `copy()` → `deepcopy()`**:

```diff
# neuron.py L672-674: Neuron.split()
- from copy import copy
- child_config = copy(self.config)
+ from copy import deepcopy
+ child_config = deepcopy(self.config)

# bundle.py L646-651: SynapticBundle.sprout()
- from copy import copy
- child_config = copy(self.config)
+ from copy import deepcopy
+ child_config = deepcopy(self.config)

# hebbian.py L988/L1004: _rewire_after_split()
- from copy import copy
- child_config = copy(bundle.config)
+ from copy import deepcopy
+ child_config = deepcopy(bundle.config)
```

**验证**:
```
parent ch[0].v_th = 0.3      ← 母本未被污染 ✅
child ch[0].v_th = 999.0     ← 子代独立修改 ✅
deepcopy isolation CONFIRMED
```

**F6 信息热寂熔断** (同一次修复):
- 新增: `k_ema < 1e-6` 且 `Σ|ξ| < 0.01` 持续 5000 步 → F6_INFORMATION_HEAT_DEATH
- 文件: `governance/fuse.py`

## 四、母本分化的早期起源（5.23 前的跑步分化）

> 来源: [[2026-05/analysis_four_questions_audit.2026.5.23]] · [[01_工程执行/implementation_plan.2026.5.23]] · [[01_工程执行/implementation_plan.2026.5.25]]

### Motor 母本分化（5.23）

v0.9.0 之前的母本分化只涉及 **Motor 层的能量管理**——不算完整的"元件构建"，而是**运行时自适应的"跑步分化"**。

| 机制 | 实现 | BIO 对应 |
|------|------|---------|
| VR 参数调整 | vr_base: 0.1→0.5 | 线粒体 ATP (Attwell 2001) |
| adapt_threshold 激活 | 每 100 步→能量<0.3→升高阈值 | Na⁺ 通道失活 |
| 突触缩放 | avg_motor_e < 0.4 → col_to_motor synapse_gain 下缩 | Turrigiano 2008 |

**关键问题（来自 5.23 Plan 的 Open Question）**:
> "adapt_threshold 修改的是 MOSFET.v_threshold——母本内部参数的运行时修改。
> 这是否违反'母本不变'原则？"

**回答**: 原则 11 明确"从母本出发，逐块建模、调整"。adapt_threshold 是母本自带的 MOSFET 方法——在 HebbianCircuit.step() 中调用是正当分化。

### 递归分化沉积（5.25）

| 触发 | 量 | 含义 | → 结构变化 |
|------|-----|------|-----------|
| ξ 持续高 | Xin > XI_SPROUT | 当前权重无法解释 → 需要新路径 | **Sprout** |
| κ → 0 | std/mean → 0 | 所有神经元趋同 → 退化为一维 | **Mitosis** (新神经元) |
| w → 0 | 权重衰减至零 | 死连接 → 释放资源 | **Prune** |

**核心洞察**: "结构本身成为可学习的变量"。

## 五、战役序列（6.17 的补偿组件修复）

> 来源: [[工作报告/战役三_源头降维_2026-06-17]] · [[工作报告/战役一_DA净空锁_2026-06-17]] · [[工作报告/战役二_钙流填谷_2026-06-17]]

这三个战役是**母本分化 + 补偿组件修复**的最新进展（6.17），直接针对 v0.9.0 分化后的遗留问题。

### 战役三 → 战役一 → 战役二（文档规定的执行时序）

### 战役三（最先执行）: 源头降维 XIN_GAIN

**病理**: 前庭链天然物理误差 |Ξ| 已高达 15-30。原 XIN_GAIN=3.0 将此信号放大三倍→影子 enc 积分饱和→钙撞死。

**修复**: XIN_GAIN: 3.0 → **1.0**。前庭 |Ξ| 本身已足够大，不需要人工放大器。

**同时**: 影子归一: vr_base_rate: 0.01 → **0.05**（加强除法规格）

### 战役一: DA 净空锁

**病理**: 皮层除颤后 Shadow Col 激活→影子→DA bundle (initial_weight=1.0) 传播大电流→DA 饱和至 1.0→soma→DA 热梯度信号被压制。

**根因**: D2R 负反馈 τ=100ms=100,000 步。在 10k 步时 DA 本地浓度仅 10%，D2R 反馈太慢→在高权重下无法及时抑制 DA。

**修复**: 降低影子→DA 初始权重 (1.0 → 0.1)，给 D2R 足够时间建立反馈平衡。

### 战役二: 钙流填谷

**病理**: 主层 Col `_activation_ema = 0.68`（健康）但 `calcium_rate = 0.000`。根因: Col→Motor bundle 读取 `calcium_rate` 但主层 Col 原本**没有 CRI**——`bundle.propagate()` 回退使用 `pre_trace`（τ=20ms 快速衰减→在发放间隙接近零）。

**修复**: 给主层 Col 神经元**安装 CRI**（calcium_rate 现在连续≠零→Col→Motor 连续驱动→STDP 时间求和物理上可行）。

### 战役序列对母本分化理论的贡献

这些战役揭示了母本分化理论的一个重要缺失:

> **v0.9.0 的母本分化只给影子层装了 CRI/DN/D2R。主层 Col 没有 CRI——因为当时认为主层不需要连续钙信号。**
> **但主层 Col→Motor 的 STDP 学习恰巧需要连续钙信号作为前突触驱动。钙信号缺失 → STDP 物理断裂。**

**教训**: 分化组件的分配不应基于"影子 vs 主层"的粗略区分，而应基于**信号链中每个连接的功能需求**。母本分化需要一个更精细的"信号路径分析"来指导组件分配。

## 八、母本分化的架构问题（盲区审计）

> 来源: [[archive/undated/analysis_differentiation_entropy_blindspot.母本分化-熵账本-盲区审计]]

**核心发现**: 母本分化体系**未覆盖体感层**。

- SomatosensoryChain 的 12 个神经元不在 `get_all_neurons()` 中
- 分化系统（CRI/DN/D2R）未应用到体感层
- 熵账本看不见体感层 → 无法为分化提供数据 → 分化也不覆盖 → 双重死角

**体感层缺失的 7/8 生命周期能力** (对比主层和影子层):
- ✅ 基础膜电容 + RC 放电
- ✅ MOSFET 门控（Thermoreceptor/Nociceptor 各有不同 V_th）
- ❌ CRI — 体感 spike→连续信号
- ❌ DN — 体感输入范围适应
- ❌ FatigueCapacitor — 体感疲劳
- ❌ TemporalCoupler — 体感跨时间尺度匹配
- ❌ 突触缩放 — 体感稳态
- ❌ adapt_threshold — 体感能量适应
