# 🧬 母本分化与元件构建原则（融合版）

> **定位**: CLAUDE.md 级别的核心法则。所有涉及神经元/神经束分化、新补偿组件创建的操作必须遵守本原则。
> **来源**: 两份独立起草的原则文档，经交叉比对后融合为单一权威版本。

---

## 核心教条

**绝对禁止**通过以下方式创建新类型:

- ❌ 修改母本类（`Neuron` / `SynapticBundle`）的内部核心代码来硬编码特化功能
- ❌ 派生特化子类（如 `ShadowNeuron`、`FastColumn`）
- ❌ 在 `step()` 中添加 `if/else` 分支来区分"主层 vs 影子层"的行为

**只允许**: 从统一母本出发，通过**物理配置差异**与**外挂补偿组件**来派生行为。

## 一、三层分化物理阶梯

```
Level 1: 工厂分化 (Config)
  └── 通过工厂函数注入不同基础物理参数
      例: _column_config vs _encoding_config vs _motor_config

Level 2: 元件分化 (Compensation)        ← 新元件的创建在这一层
  └── 针对特定物理机制，在 compensation.py 中建立独立组件
      通过 NeuronConfig flag 挂载
      例: CRI (钙积分), DN (除法归一化), D2R (自受体)

Level 3: 运行时分化 (Homeostasis)
  └── 由底层物理耗散驱动的动态自适应
      例: ATP 枯竭 → adapt_threshold ↑, BCM 滑动阈值, 突触缩放
```

### 各层的边界

| 维度 | Level 1 (工厂) | Level 2 (元件) | Level 3 (运行时) |
|------|---------------|---------------|-----------------|
| **修改单位** | 整个神经元配置 | 单个补偿组件 | 运行时参数 |
| **激活方式** | `_column_config()` 函数 | `use_xxx=True` flag | 物理条件触发 |
| **存放位置** | `hebbian.py` / `chain.py` | `compensation.py` | 不新增文件 |
| **是否修改母本** | ❌ 不修改 | ❌ 不修改 | ❌ 不修改 |

## 二、信号链路按需分配铁律

> **绝对禁令**: 禁止基于粗糙的"语义归属"（如"因为它是影子层所以要装，主层就不装"）来盲目分配补偿组件！

### 第一性原理

**组件的挂载必须基于物理信号链路的因果刚需。** 分析单位 = **连接（bundle）**，不是层（layer）。

### 分析方法

对于一条特定的突触连接（如 `Col → Motor`），追问:

```
Q1. 这条 bundle 承载什么物理信号？（spike? 连续? 时变?）
Q2. 下游神经元需要什么物理输入才能正常工作？
Q3. 上游神经元当前提供的信号是否满足 Q2 的条件？
Q4. 如不满足 → 需要什么补偿组件来桥接这个缺口？
```

### 血的教训：CRI 分配错误

```
v0.9.0 的错误决策:
  "Shadow Col 需要 calcium_rate → 装 CRI"
  "主层 Col 用 pre_trace 就够了 → 不装 CRI"
  ← 这是基于"身份标签"（影子 vs 主层）的分配

物理后果:
  主层 Col→Motor 的 STDP 需要时间求和
  → pre_trace (τ=20ms) 在两次 spike 之间快速衰减至零
  → 前突触信号断裂 → STDP 物理断裂
  → Col→Motor 学习链死亡

修复（战役二）:
  "这条 Col→Motor bundle 需要什么前突触信号？"
  → 需要持续的非零钙信号（用于时间求和）
  → CRI 必须在**所有向 Motor 输出的 Col** 上激活
  → 不管这 Col 在主层还是影子层
```

### 铁律重述

**只要某条突触链路依赖时间求和的 STDP，且突触前膜是脉冲放电，那么该前级神经元必须强制挂载 CRI——无论其处于哪个脑区。**

## 三、新元件构建与提议流程

### 前置检查（Step 0）

在创建新组件之前:
- [ ] 是否已有同等功能的旧组件可以**复用**？
- [ ] 是否已有关闭但可激活的组件（flag = False 但类已存在）？
- 例: CRI 在 v0.9.0 已为 Shadow Col 创建——战役二**不需要新类**，只需激活 flag

### 构建流程（Step 1-5）

```
Step 1: 识别信号路径需求
  → 不是"某层需要"，是"这条连接上的 STDP/学习需要"
  → 分析单位 = bundle，不是 layer

Step 2: 映射到生物/物理对应物
  → BIO: 注释（论文/教材/已有代码注释）
  → 例: "BIO: CaMKII 浓度 ∝ 平均发放率 (Lisman et al. 2012)"

Step 3: 使用四大原语组装
  → 创建类 → components/compensation.py
  → 只能使用: Capacitor / MOSFET / Memristor / PowerRail
  → 禁止: 在 step() 中使用 if/else 数学公式（如 clamp()、sign()、dot()）

Step 4: 在 NeuronConfig 中添加激活 flag
  → use_xxx: bool = False
  → 相关参数：xxx_capacitance, xxx_r_leak, xxx_q_spike...

Step 5: 在工厂函数中连线
  → 正确的工厂函数中设置 flag = True
  → 绝不通过修改 neuron.step() 的函数体来激活
```

### 物理原语强制

```python
# ✅ 正确: CRI 使用四大原语
class CalciumRateIntegrator:
    def __init__(self):
        self._charge = Capacitor(capacitance=1.0)      # P.01
        self._zener = MOSFET(v_threshold=1.0, ...)     # P.02 硬钳位
    def step(self, spiked, dt):
        self._charge.inject(0.02 if spiked else 0, dt)
        self._charge.leak(dt)
        return min(self._charge.voltage, self._zener.v_threshold)

# ❌ 错误: Python float + if 硬编码
class BadComponent:
    def __init__(self):
        self.value = 0.0
    def step(self, spiked, dt):
        self.value += 0.02 if spiked else 0
        self.value *= 0.999
        if self.value > 1.0:           # ← 硬编码阈值
            self.value = 1.0           # ← 不是物理元件
        return self.value
```

**判断标准**: 组件的 `step()` 中是否出现 `if self.value > threshold`、`clamp()`、`sign()`、`dot()`、`max()`？如出现→硬编码数学公式→违反物理锚定。用 MOSFET/Capacitor 的组合来物理实现同等功能。

## 四、基因隔离红线

> **在任何触发形态发生学的操作中，必须且只能使用 `copy.deepcopy(config)`。**

### 三个必须使用 deepcopy 的位置

| # | 文件 | 函数 | 共享风险 |
|---|------|------|----------|
| 1 | `neuron.py` L672-674 | `Neuron.split()` | `channels: List[ChannelConfig]` — 子代修改阈值 → 母本被污染 |
| 2 | `bundle.py` L646-651 | `SynapticBundle.sprout()` | `silent_snapshot: dict` — 子代休眠覆写母本记忆 |
| 3 | `hebbian.py` L988/L1004 | `_rewire_after_split()` | 同上 |

### 为什么浅拷贝是红线

```python
# 浅拷贝的实际后果（来自实测）:
from copy import copy
child_config = copy(self.config)
# → child_config.channels is self.config.channels      # True - 共享!
# → child_config.silent_snapshot is self.config.silent_snapshot  # True - 共享!

# 子代修改:
child_config.channels[0].v_threshold = 999.0
# → self.config.channels[0].v_threshold = 999.0  # 母本被悄悄污染
```

**后果链**: 浅拷贝 → 父代与子代共享可变对象 → 子代运行时修改 → 母本基因库被全域污染 → 行为退化 → F6 信息热寂熔断。

### 正确写法

```python
from copy import deepcopy
child_config = deepcopy(self.config)
# → 子代与母本完全物理隔离 ✅
```

## 五、检查清单

在创建任何新分化组件或修改既有组件分配时，逐条确认:

```
□ Level 2 新组件 → compensation.py? (不在 neuron.py 中)
□ 使用四大原语 (Capacitor/MOSFET/Memristor/PowerRail)?
□ 没有 if/else 硬编码数学公式?
□ NeuronConfig flag 添加? (use_xxx: bool = False)
□ 工厂函数中激活? (不是在 neuron.step() 中激活)
□ Q1 BIO: 生物/物理对应物已注释?
□ Q2 物理结构: Sources → Bundle → Targets 已定义?
□ Q3 参数: 每个参数有推导或实验来源?
□ Q2-bis: 信号路径分析 - 组件分配基于 bundle 需求，不是层归属?
□ 分裂/发芽操作 → deepcopy(config)? (不是 copy)
```
