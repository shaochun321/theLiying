**补充说明**（AI 未提及但应了解）:

"母本不变"原则的精确含义:
- **不能**: 修改 `Neuron.step()` 的函数体来添加新功能
- **可以**: 通过 `NeuronConfig` 的 flags (`use_calcium_integrator`, `use_dn_receptor` 等) 来激活已有的补偿组件
- **可以**: 在 `compensation.py` 中创建新的补偿组件——它们有自己的 step()，在 `Neuron.step()` 中被条件调用

### "信号路径分析"原则（AI 捕获了一个关键洞察）

AI 注意到: "组件分配不应基于'影子 vs 主层'的粗略区分，而应基于信号链中每个连接的功能需求"。

**这是对的**，而且是战役二的核心教训:

```
v0.9.0 的错误:
  CRI 安装在 Shadow Col (shadow 需要 calcium_rate)
  → 认为主层 Col 不需要 CRI (主层用 pre_trace 就够了)
  → 遗漏了主层 Col→Motor 的 STDP 需要 calcium_rate 作为连续前突触驱动
  → pre_trace 太快衰减 (τ=20ms) → 两次 spike 之间归零 → STDP 断裂

正确的分析路径:
  "Col→Motor 这条 bundle 需要什么前突触信号？"
  → 需要持续的非零钙信号（才能做时间求和）
  → CRI 必须在**所有**向 Motor 输出的 Col 上安装
  → 不管这 Col 在主层还是影子层
```

**这个原则应该加入 CLAUDE.md 的强制三问中**——在 Q2（物理结构）中增加"信号路径需求分析"步骤。

## 三、新构建提议机制：回应提案流程

### AI 理解的流程（再次确认正确）

AI 提取的"新元件构建流程"：

1. 识别信号路径需求（不是"某层需要"，是"这条连接上的 STDP 需要"）
2. 映射到生物/物理对应物（BIO 注释）
3. 创建类 → `compensation.py`
4. 在 `NeuronConfig` 中添加激活 flag
5. 在工厂函数中连线

**完全正确。** 补充一点细节:

### 第 0 步：确认这不是旧组件复用

在创建新组件之前，必须检查:
- 是否已经有**同等功能的旧组件**可以复用？
- 例如: CRI 在 v0.9.0 已创建（为 Shadow Col），战役二不需要创建新组件——只需要**激活**主层 Col 上的 CRI

### 第 3 步细节：补偿组件必须使用四大原语

```python
# ✅ 正确的: CRI 使用 Capacitor + Zener MOSFET
class CalciumRateIntegrator:
    def __init__(self):
        self._charge = Capacitor(capacitance=1.0)
        self._zener = MOSFET(v_threshold=1.0, ...)  # 硬钳位
    
    def step(self, spiked, dt):
        self._charge.inject(CALCIUM_INCREMENT if spiked else 0, dt)
        self._charge.leak(dt)
        clamped = min(self._charge.voltage, self._zener.v_threshold)
        return clamped

# ❌ 错误的: 用 Python float + if 语句
class BadComponent:
    def __init__(self):
        self.value = 0.0
    def step(self, spiked, dt):
        self.value += 0.02 if spiked else 0
        self.value *= 0.999
        if self.value > 1.0:  # ← 硬编码阈值，不是物理元件
            self.value = 1.0
        return self.value
```

**判断标准**: 如果组件的 step() 包含 `if self.value > threshold` 或 `self.value = max(self.value, limit)` 或 `sign()` 或 `dot()` —— 那就是硬编码数学公式，违反物理锚定原则。应该用 MOSFET/Capacitor/Memristor/PowerRail 的组合来物理实现同样的功能。

## 四、CLAUDE.md 补充建议

AI 关于补充 CLAUDE.md 的建议**合理**。将补充两段：

### 补充 1: 母本分化架构（在"结构构建原则"之后）

```markdown
## 母本分化架构（Mother Differentiation）

三种分化层次:
- **Level 1 (工厂分化)**: 同一 Neuron 类，不同 config 工厂函数
- **Level 2 (元件分化)**: 补偿组件 (CRI/DN/D2R/...)，在 compensation.py 中，通过 NeuronConfig flags 激活
- **Level 3 (运行时分化)**: 物理驱动的自适应 (adapt_threshold, BCM, 突触缩放)

原则:
- 不能直接修改 Neuron.step() 来添加功能
- 新组件 = 新类 → compensation.py → 使用四大原语 (Capacitor/MOSFET/Memristor/PowerRail)
- 激活 = NeuronConfig flag → 工厂函数中设为 True
```

### 补充 2: 信号路径分析（在"强制三问"的 Q2 之后）

```markdown
Q2-bis. 信号路径上每条连接需要什么物理信号？
    → 例: Col→Motor 的 STDP 需要连续的前突触信号（不能用快速衰减的 pre_trace）
    → 这决定 CRI 应该在所有向 Motor 输出的 Col 上激活（不是按"主层 vs 影子层"分组）
    → 分析单位 = 连接（bundle），不是层（layer）
```
