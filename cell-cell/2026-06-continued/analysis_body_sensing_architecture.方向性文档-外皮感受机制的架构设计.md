## 1. 感受 vs 运动判别：结构区分

```
运动判别（已有）：                    感受（待建）：
  内耳惯性力                            外部刺激作用于体表
     ↓                                     ↓
  MET 机械转导                          皮肤 patch 受热/接触
     ↓                                     ↓
  HairCell 放大                          感受神经元 transduction
     ↓                                     ↓
  Afferent (reg/irr)                    [可能无需 reg/irr 分化]
     ↓                                     ↓
  Encoding (2/轴)                       Encoding (新模态汇入)
     ↓                                     ↓
  Column ← 两路在此汇聚 →              Column
     ↓
  Motor
```

关键差异：

| | 运动判别 | 外皮感受 |
|---|---|---|
| 信号生成者 | 自身运动产生的惯性后果 | 外部世界施加的物理刺激 |
| 编码方式 | 6 轴（解剖固定） | N patch（体表面积） |
| 时间特性 | 频率/相位 (AC/DC) | 强度分布（空间梯度） |
| 空间分辨率 | 固定（半规管朝向） | 可调（patch 数量 = 感受带宽） |
| 自身原点需求 | 不需要（轴方向是解剖常数） | **必需**（"左"和"右"需要坐标系） |

## 2. 影子层的角色澄清

> 影子层是对外层神经系统/其他模块对客观现实表征后进行抽象处理的层级。

```
外部世界
  ↓ [物理过程]
体表 patch → 感受神经元 → Encoding → Column    ← 这是「表征」
                                       ↓
                                  Shadow Layer    ← 这是「对表征的预测」
                                       ↓
                                  预测误差 → DA
```

影子层不感受世界。影子层感受「Column 对世界的表征」是否符合预期。

这意味着：
- 外皮感受链的输出进入 Column（表征层）
- Shadow 预测 Column 的输出
- 当新的感受输入改变了 Column 的活动模式，Shadow 产生预测误差
- 预测误差 → DA → STDP 调制

**这就是学习闭环：外皮新刺激 → Column 表征变化 → Shadow 预测误差 → DA → 权重更新**

## 3. 刚体壳与自身原点

### 为什么自身原点是前提

没有坐标系的感受器只能报告「当前温度」。两个对称感受器可以报告「左右温差」，但要知道左右温差意味着什么（向右转可以降低温度），需要：

1. **体几何**：patch 在身体上的固定位置
2. **体朝向**：body heading（相对世界坐标）
3. **运动→感受因果**：我向右转 → 左 patch 暴露增加

```
自身原点 = 体框架坐标系
         = {patch 位置} × {body 朝向} × {运动→感受映射}
```

### 体积可调

你说得对——皮层体积是可调的。最小配置：

```
4 patch × 1 模态 = 4 感受神经元 → 4 bundle → Encoding
```

最大配置（如果需要高分辨率）：

```
32 patch × 3 模态(温度/压力/nociception) = 96 感受神经元
```

**感受带宽 = patch 数 × 模态数。** 这个值决定了皮层结构的规模。

### 带宽建模工具

已有的 A7 运动势提供了数学框架：

- **时间测度**：刺激变化速率 → 感受需要多快的采样率
- **空间测度**：梯度精度 → 感受需要多少个 patch
- **运动势 ν**：单位时间内动能变化 → 感受→运动转换效率的度量

可以推导：

> 最小感受分辨率 Δθ ≥ f(ν_target / 梯度信噪比)

## 4. 热源分化：物理路径 vs 语义标签

### 平行构建的做法（语义标签）

```python
class NutritiveThermalSource:    # 类型 = "食物"
class ThermalStimulusSource:     # 类型 = "有害"
```

系统在创建时就"知道"哪个是食物。

### 物理正确的做法

热源只有物理属性：温度 T、半径 r、能量密度 ρ。

分化通过**后果**发生：
- T < T_comfort → 接触后 EnergyStore 增加 → shadow 预测改善 → DA↑ → "好"
- T > T_nociception → 接触后 energy 损耗 → shadow 预测恶化 → DA↓ → "坏"
- T_comfort < T < T_nociception → 中性区间 → 无学习信号

**分化不是标签，是经验。**

### 逃跑/进食环流制约

环流比例电路已有三路：homeo / motor / feed。

```
正常: ρ_homeo=0.7, ρ_motor=0.2, ρ_feed=0.1

接触食物源: ρ_feed↑ → ρ_motor↓ (能量流向消化，运动减少)
                    → deviation↓ → DA↓ → STDP 弱化「继续移动」

接触有害源: nociception↑ → ρ_motor↑ (逃跑) → ρ_feed↓
                         → deviation↑ → DA↑ → STDP 强化「远离方向」
```

**进食和逃跑通过环流比例自然制约——不需要代码级的 `if food: approach; if hazard: escape`。**

## 5. 沉积（Sprouting）的角色

| 阶段 | 机制 | 类比 |
|---|---|---|
| 初始拓扑 | **先天结构**：patch→感受神经元→Encoding | 胚胎轴突引导 |
| 通路增强 | **沉积**：高活动通路长出新 bundle | 使用依赖可塑性 |
| 通路消退 | **修剪**：低活动通路萎缩 | 突触竞争 |
| 感受带宽扩展 | **有丝分裂**：感受神经元分裂 | 干细胞增殖 |

初始感受链是先天的（系统发育先验），与前庭拓扑等价。
但感受→运动的映射（"左热→右转"）应通过 STDP + DA 学习涌现。

## 7. 开放问题（明日思考）

1. **感受链是否需要 reg/irr 分化？**
   - 运动判别中 reg=DC(静态倾斜), irr=AC(动态旋转)
   - 感受中：DC = 恒温环境, AC = 温度突变？
   - 还是说感受天然是 DC 为主（温度变化远慢于机械振动）？

2. **自身原点如何建立？**
   - 选项A：先天几何（patch 位置是已知常数）→ 不需要学习
   - 选项B：自我探索（动一动看感受怎么变）→ 需要前向模型
   - 选项C：两者结合（先天粗糙 + 经验校准）

3. **感受带宽的物理约束是什么？**
   - 每个 patch 有代谢成本（维持感受神经元需要 ATP）
   - EnergyStore 的 P_inflow 限制了总带宽
   - 最优带宽 = argmax(感受信息增益 / 代谢成本)

4. **热源分化与环流制约的耦合点在哪？**
   - CirculationProportionCircuit 的 feed capacitor 由什么驱动？
   - 目前没有物理的「进食」过程——需要 EnergyStore.deposit() 与热接触耦合

5. **感受结构是否需要与 Binding Layer 交互？**
   - Binding 做跨模态关联。感受是新模态
   - 感受 × 前庭 → 新的 binding cell → 「运动中感受到的刺激变化」
   - 这可能是方向学习的关键信号源
