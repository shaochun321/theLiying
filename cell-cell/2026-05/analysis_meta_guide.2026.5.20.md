# 所有元结构 — 完整图鉴

## 先说全景

系统里一共有 **5 种元结构** 和 **4 种电子元件**:

```
元结构 (功能单元):        电子元件 (物理零件):
  ① 元单元 (MetaUnit)       ❶ 电容器 (Capacitor)
  ② 元管道 (MetaPipe)       ❷ 场效应管 (MOSFET)
  ③ 元块 (MetaBlock)        ❸ 忆阻器 (Memristor)
  ④ 元果实 (MetaFruit)      ❹ 电源轨 (PowerRail)
  ⑤ 元门控 (MetaGate)

关系:
  元单元 内部装着 ❶电容器 + ❷场效应管 + ❹电源轨
  元管道 内部装着 ❸忆阻器 (每条连线一个)
  元块 给 元单元 供电 (通过 ❹电源轨)
  元果实 长在 元管道 上
  元门控 是 元单元 的一种特化
```

---

## 4 种电子元件

先解释零件, 再解释由零件组成的结构。

### ❶ 电容器 (Capacitor) — "蓄水池"

**是什么**: 一个能存储电荷的容器。

**工作方式**:

```
注水 (inject): 有信号进来 → 水位上升
漏水 (leak):   没有信号 → 水位自然下降
水位 (voltage): 当前存了多少 → 这就是"状态"
```

**参数**:
| 参数 | 意思 | 物理直觉 |
|------|------|---------|
| capacitance (容量) | 池子有多大 | 大池子: 需要很多信号才能填满, 但一旦满了就不容易空 |
| charge (电荷) | 当前水位 | 代表"这个单元现在记住了多少" |

**为什么用它**: 神经元需要"记住最近发生了什么"。
电容器自然地做到了这一点——信号来了就充电, 没有就漏电。
漏电速度取决于 R×C (电阻 × 电容)——这就是**时间常数**。
大时间常数 = 记忆持久。小时间常数 = 快速遗忘。

---

### ❷ 场效应管 (MOSFET) — "水坝闸门"

**是什么**: 一个由电压控制的开关, 但不是"开/关"那种硬开关——
它有一个**渐变区**: 水位接近阈值时, 门缝隙慢慢变大。

**工作方式**:

```
水位低于坝高 (v < threshold): 
  门几乎关闭, 只有极微量的水渗过 (亚阈值泄漏)
  渗透量按指数衰减 → 水位越低, 渗透越少

水位高于坝高 (v > threshold):
  门打开, 水流 = 水位超出坝高的量 × 导通系数
  完全线性 → 超出越多, 流过越多
```

**参数**:
| 参数 | 意思 | 物理直觉 |
|------|------|---------|
| v_threshold (阈值) | 坝有多高 | 高坝: 需要很强的信号才能触发。低坝: 微弱信号就响应 |
| gm (跨导) | 门打开后水流多猛 | 高跨导: 一旦超过阈值, 输出很强。低跨导: 输出温和 |
| n_slope (亚阈值斜率) | 门关着时漏多少 | 影响"几乎达到阈值"时的行为 |

**为什么用它**: 这就是之前门控实验中 θ 参数的物理实体。
不同区域的 MOSFET 阈值不同 → 就是分化。
Motor 区的坝很高 (θ=0.25) → 保守。
Sediment 区的坝很低 (θ=0.03) → 敏感。

---

### ❸ 忆阻器 (Memristor) — "走过的路"

**是什么**: 电阻值会随着通过的电流历史而改变的元件。
走得越多, 路越平坦 (电阻降低)。走得少, 路重新长草 (电阻升高)。

**工作方式**:

```
有电流通过 → 根据 STDP 规则改变电阻:
  先来后到 (pre 先, post 后) → 路变平坦 (增强)
  后到先来 (post 先, pre 后) → 路长草 (削弱)

电阻低 = 连接强 → 信号容易通过
电阻高 = 连接弱 → 信号几乎不过
```

**参数**:
| 参数 | 意思 | 物理直觉 |
|------|------|---------|
| w (状态) | 路有多平坦 (0=荒废, 1=高速公路) | 这就是"突触权重" |
| r_min (最小电阻) | 路最平能到什么程度 | 物理极限: 再怎么走, 阻力不会是零 |
| r_max (最大电阻) | 路最荒能到什么程度 | 完全废弃但不消失 |

**为什么用它**: 学习 = 走路。走得多, 路就强化。
忆阻器用物理方式做到了这一点, 不需要人为设定"学习率"。
电流历史自然决定了权重变化的方向和大小。

---

### ❹ 电源轨 (PowerRail) — "电池"

**是什么**: 给每个元单元供电的电源。但它不是理想电源——
拉的电流越大, 实际能提供的电压就越低 (内阻压降)。

**工作方式**:

```
轻负载 (低电流): 几乎能提供满额电压
重负载 (高电流): 电压下降, 元单元性能降低
过载: 电压接近零, 元单元几乎无法工作

实际电压 = 额定电压 - 电流 × 内阻
```

**参数**:
| 参数 | 意思 | 物理直觉 |
|------|------|---------|
| vdd (额定电压) | 电池标称多少伏 | 由元块 (MetaBlock) 决定, 代表该区域的能量供应 |
| r_internal (内阻) | 电池内部阻力 | 内阻大: 稍微用力就掉电压。内阻小: 稳定供电 |

**为什么用它**: 代谢约束。生物系统中, 高活动 = 高能耗。
当能量不足时, 神经元不是"选择不工作"——而是物理上无法工作。
电源轨自然实现了这个约束。

## 5 种元结构

### ① 元单元 (MetaUnit) — "一个完整的处理器"

**由什么组成**:

```
一个元单元 = 
  1 个电容器 (记忆近期状态)
  + 1 个场效应管 (决定是否响应)
  + 1 个电源轨 (受能量约束)
  + 时序记忆 (STDP traces — 记录"什么时候响应过")
  + 自稳态调节 (钙传感器 — 防止过度兴奋或沉默)
  + 成熟阶段 (spine → column → area — 逐步获得更多权限)
```

**工作流程**:

```
信号进来
  → 电源轨检查: 有没有足够的电? → 没有 → 输出微弱
  → 电流注入电容器 → 电容器电压上升
  → 电压送到场效应管 → 超过阈值? 
    → 是: 产生输出 → 同时产生热 (I²R)
    → 否: 几乎没有输出
  → 更新时序记忆
  → 钙传感器检查: 我活跃度正常吗?
    → 太高: 提高阈值 (坝变高, 不那么容易响应)
    → 太低: 降低阈值 (坝变低, 更容易响应)
  → 成熟检查: 我积累够了吗?
    → spine → column: 获得侧向抑制权 + STDP加速
    → column → area: 获得更大抑制范围 + 更强STDP
```

**分化参数**:
| 参数 | encoding | column | sediment | 为什么 |
|------|----------|--------|----------|-------|
| target_rate | 0.05 (活跃) | 0.03 (中等) | 0.01 (低调) | 感知层需要快速响应; 沉淀层只处理微弱长期信号 |
| calcium_tau | 10 (快感知) | 20 (中速) | 50 (极慢) | 感知层需要立刻知道"我在干什么"; 沉淀层需要积累很久 |
| r_leak | 0.5 (漏得快) | 1.0 (正常) | 5.0 (漏得慢) | 感知层不需要长记忆; 沉淀层要保持微弱信号很久 |
| threshold (初始) | 0.002 (灵敏) | 0.003 (中) | 0.001 (极灵敏) | 沉淀层需要检测极微弱的 Xin 残差 |

**为什么这样设参数**:
每个参数不是"拍脑袋"——而是物理必然:

- **r_leak 决定记忆时长**: RC 时间常数 = R × C。
  R=0.5, C=1.0 → τ=0.5 → 快速遗忘 → 适合处理快变化的输入。
  R=5.0, C=1.0 → τ=5.0 → 慢速遗忘 → 适合长期积累。

- **threshold 决定灵敏度**: 场效应管的坝高。
  坝低 = 弱信号也能过 = 适合检测微弱的长期趋势。
  坝高 = 强信号才能过 = 适合忽略噪声。

- **target_rate 决定自稳态平衡点**: 
  高 target = "我应该经常激活" → 阈值会自动降低来满足。
  低 target = "我应该很少激活" → 阈值会自动升高来满足。

### ② 元管道 (MetaPipe) — "一条电缆"

**由什么组成**:

```
一条元管道 = 
  权重矩阵 (每条线一个忆阻器 → 连接有多强)
  + 电缆属性 (长度 → 延迟, 电阻 → 衰减, 速度 → 传导)
  + 脉冲队列 (信号在管道中传播, 不是瞬间到达)
  + 到达时序 (记录信号什么时候到的 → 给 STDP 用)
  + 学习规则 (Oja/BCM/冻结)
  + Xin 张力 (残差 → 预测错误的记录)
```

**工作流程**:

```
源端有信号
  → 信号乘以权重 (忆阻器导通)
  → 信号进入管道 (inject_pulse)
  → 经过延迟 (delay = length / velocity) 后到达
  → 到达时信号已衰减 (attenuation = e^(-R×L))
  → 目标端接收信号
  → STDP 检查: 源端先还是目标端先?
    → 源先到: 这条路有因果关系 → 增强
    → 目标先: 这条路无因果 → 削弱
```

**T/O/P/R/Xin 就是管道内部的 5 个处理阶段**:

```
T (传递):  propagate() — 信号从源到目标的传输
O (观测):  last_pre/post_activation — 记录"经过了什么"
P (预测):  基于权重, 预期目标应该收到什么
R (反馈):  STDP — 根据实际 vs 预期, 调整权重
Xin (残差): xin_tension — 预测和实际的差异, 是学习的驱动力
```

**分化参数**:
| 参数 | encoding→column | column→sediment | 为什么 |
|------|-----------------|-----------------|-------|
| cable_length | 1.0 (短) | 3.0 (长) | 近距离连接快, 远距离慢 |
| cable_resistance | 0.02 (低损) | 0.05 (高损) | 远距离信号衰减更多 |
| learning_rule | "bcm" (活跃学习) | "oja" (保守) | 巩固层用 BCM 竞争; 沉淀层用 Oja 稳定 |
| propagation_velocity | 1.0 (标准) | 0.5 (慢) | 深层信号不需要快速到达 |

### ③ 元块 (MetaBlock) — "变电站"

**是什么**: 给一组元单元供电的能量节点。
多个元块之间通过"电线"(导体边)连接, 形成供电网络。

**工作方式**:

```
消化系统产生能量 → 注入到源节点
  → 能量沿着导体边流向各节点
  → 每个节点给绑定的元单元供电
  → 远处的节点能量少 (导体有阻力)
  → 活跃的元单元消耗更多 → 本地能量下降
  → 能量下降 → 元单元的电源轨电压下降 → 性能降低
```

**供电网络**:

```
[消化源] ─── [encoding块] ─── [column块]
                  │                │
              [cpg块] ───── [motor块]
                  │
             [vestibular块]
                               [sediment块]
```

**关键**: 远离消化源的区域 (如 sediment) 能量供应弱 → 
那里的元单元只能做低能耗的事 → 自然形成"安静存储"的功能分化。

---

### ④ 元果实 (MetaFruit) — "便利贴"

**是什么**: 当预测出错时, 在出错的管道上贴一张便利贴,
记录"这里有问题"。便利贴会慢慢褪色 (trace 衰减)。

**如果在褪色之前, 有人来确认** ("第三因子信号" 到达):
→ 便利贴变成**永久标记** (结晶 crystal)
→ 权重被锁定, 这条路的学习成果被固定下来

**如果没人来确认**:
→ 便利贴褪色到看不见 → 消失 (dissolution)
→ 这次学习被遗忘

**工作流程**:

```
Xin 残差 > 阈值 → 贴便利贴 (fruit 创建)
便利贴存在中 → trace 每 tick ×0.995 (慢慢褪色)
  如果 第三因子信号到达 且 trace 还够大:
    → 结晶! → 权重固定 → 学习成果保存
  如果 trace < 0.01:
    → 便利贴消失 → 这次学习无效
```

**为什么需要果实而不是直接学习**:
直接学习 = 任何信号都会修改权重 → 噪声也会学进去。
果实机制 = "先标记, 等确认, 再固定" → 只有被确认的学习才保留。
这对应大脑中的 **突触标记-捕获假说** (synaptic tagging and capture)。

---

### ⑤ 元门控 (MetaGate) — "交通灯"

**是什么**: 一种特化的元单元, 不处理信号内容,
而是**控制其他管道是否允许学习**。

这就是之前 v2→v6 实验中验证的 GateTemplate。

**工作方式**:

```
输入: 同步性 (多少信号一起来) + 对比度 (信号差异有多大)
  → 电容器积分 (τ 决定反应速度)
  → 场效应管判定 (θ 决定灵敏度)
  → 输出门控信号: 0 = 禁止学习, >0 = 允许学习

  门控信号 → 调制相邻元管道的 STDP 学习率
    学习率 × 0 = 冻结 (巩固保护)
    学习率 × 1 = 正常学习
```

**与元单元的关系**:
元门控 **就是** 元单元, 只是扮演了不同的角色。
就像同一种细胞, 在不同位置分化成不同功能:
有的处理信号 (感觉神经元), 有的控制开关 (中间神经元)。

**分化参数**: 就是之前实验验证的 (τ, θ, cw):
| 参数 | 意思 | 物理实体 |
|------|------|---------|
| τ (时间常数) | 反应快慢 | 电容器的 R×C |
| θ (阈值) | 灵敏度 | 场效应管的 v_threshold |
| cw (对比权重) | 更看差异还是看同步 | 忆阻器权重的分配比例 |

## 为什么这些参数要这样设

一句话: **参数不是人选的, 是物理规律决定的。**

```
R×C = 时间常数 → 由区域的功能需求决定
  感知层需要快速响应 → RC 小
  沉淀层需要长期积累 → RC 大

  这不是"设置" — 这是物理约束。
  就像你想让水坝泄洪快, 就要开大闸门;
  想让水库保水久, 就要减小泄洪口。

V_threshold = 灵敏度 → 由信号信噪比决定
  信号强/噪声多的区域 → 阈值高 (只响应强信号)
  信号弱/需要检测微弱趋势 → 阈值低

  就像在嘈杂的工厂里, 你要喊才能被听到;
  在安静的图书馆, 轻声细语就够了。

cable_length = 延迟 → 由拓扑距离决定
  相邻区域 → 短 → 延迟小
  远程区域 → 长 → 延迟大

  不是"设置"——是放在那里就是那个距离。
```

> [!IMPORTANT]
> **所有参数都可以从两个物理量推导出来:**
> 
> 1. **该区域处理什么样的信号** (快/慢, 强/弱, 稀/密)
> 2. **该区域在网络中的位置** (近/远, 上游/下游)
> 
> 信号特性 → 决定 RC 时间常数 + 阈值
> 网络位置 → 决定电缆长度 + 能量供应
> 
> 不需要猜。
