## 影子层与时空环流的关系

你说得对——**有了环流之后，影子层的必要性确实显现了**。原因：

### 现在的问题

1. **反馈束 `transition→sig_mean` 的张力 = 1.45**（最高）
   - 这意味着：z_t 维度（transition）预测 sig_mean 会高
   - 但 sig_mean 实际为 0（信号经过 CPG 门控后被衰减了）
   - **这是 CPG 门控造成的系统性预测误差**

2. **练习层的对称张力 ±0.69**
   - practice_cortex 的结晶体（px_ 神经元）做出了预测
   - 但预测在"开门"和"关门"相位都失败了
   - **影子层正在记录 CPG 相位未对齐的后果**

### 时空环流如何让影子层有意义

```
环流检测 → 发现 circulation = 1.77
      ↓
信号在 sig→z_t→sig 中形成闭环
      ↓
闭环每半个 CPG 周期被打断一次（门关了）
      ↓
打断时 Xin 预测失败 → 张力积累
      ↓
张力超过 0.5 → 创建休眠果实（影子）
      ↓
影子 = "这里有一个周期性的预测失败"
```

**影子层正在做的事**：记录 CPG 门控导致的**周期性预测中断**。这正是 Column 前向模型应该解决的问题——Column 应该用这些影子的信息来**预测什么时候门会关闭**。

## 提议：连接影子层到 Column 前向模型

当前两个系统是独立的：
- Column 前向模型：检测 raw signal × (1-gate)
- 影子层：记录 predicted vs actual 的差异

**如果连接它们**：

```python
# Column 可以"读取"影子层的张力
# 高张力 → Column 知道有系统性的预测失败
# → Column 提前调整 CPG 相位来减少张力
# → 闭合：影子层提供反馈，Column 提供校正
```

这就是你说的"影子层的必要性"——它提供了一种**结构性的时序预测误差信号**，比 Column 当前用的"raw signal × (1-gate)"更丰富，因为它包含了**因果链上每一步的预测残差**。
