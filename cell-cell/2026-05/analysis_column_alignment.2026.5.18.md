# Column 作为"对齐/翻译"模块：假想与真实世界

## 你的假想

> Column ≈ 一个被训练后固定的结构，能够实行特定的"算法"。
> 在真实的神经系统中，那是千万年进化后塑造的神经结构。
> 可以用来做对齐，或者某种"翻译"。

这个假想**非常准确**。真实的大脑中确实存在这类结构——它们不是通过个体学习产生的，而是进化塑造的"硬件算法"，在个体生命中只做微调。

## 真实世界的三个原型

### 1. 小脑 (Cerebellum) — **前向模型/时序预测器**

```
                    ┌──────────────────┐
  运动指令 ──────→  │  平行纤维网格      │  ──→ 预测的感觉反馈
  (efference copy)  │  (颗粒细胞→浦肯野) │      (predicted sensory)
                    │  ≈ 固定拓扑结构    │
                    └───────┬──────────┘
                            │ 攀缘纤维 (climbing fiber)
                            │ = 误差信号 (actual - predicted)
                    实际感觉反馈 ──→ 修正权重
```

**核心算法**：给定运动指令，预测"感觉反馈应该在什么时间到达"。
- 平行纤维网格 = **固定拓扑**（进化塑造）
- 浦肯野细胞突触 = **可调权重**（个体学习微调）
- 攀缘纤维 = **误差教学信号**（实际 vs 预测的时序差异）

**与你的假想的对应**：
- Column 层 = 平行纤维网格（固定结构，执行"时序预测"算法）
- Column 的 STDP = 浦肯野突触微调
- Xin tension = 攀缘纤维误差信号

### 2. 丘脑网状核 (TRN) — **门控/路由器**

```
  皮层 ←────── 丘脑中继核 ←────── 感觉输入
                    ↑
              TRN (全抑制性)
              ├─ 接收丘脑→皮层的侧支
              └─ 接收皮层→丘脑的侧支
              → 选择性抑制丘脑中继
```

**核心算法**：不是开/关，而是**选择性路由**。
- 基本拓扑是进化硬连接的（TRN 位于丘脑和皮层之间，只发抑制）
- 但选择**哪个**通道被抑制，由皮层反馈和脑干唤醒状态动态决定

**与你的假想的对应**：
- 我们的 CPG 门控 = TRN 的极简退化版
- 真正的 TRN 有**多通道选择性**（可以让 acoustic 通过同时抑制 thermal）
- 我们的门控对所有通道一视同仁（全开或全关）

### 3. 上丘 (Superior Colliculus) — **坐标变换器**

```
  视觉坐标 (retinotopic) ──→ 上丘拓扑映射 ──→ 运动坐标 (saccade)
       (x_retina, y_retina)    固定的对数映射     (amplitude, direction)
```

**核心算法**：将一种坐标系翻译到另一种坐标系。
- 映射拓扑是进化固定的（对数极坐标 → 笛卡尔运动坐标）
- 个体经验只做微量校准

**与你的假想的对应**：
- "翻译" = 将信号空间映射到运动空间
- Column 的 `log2` 缩放阈值 (line 297-310) 已经有这个影子！

## Column 层应该做什么？

你的假想指向一个明确的设计：**Column = 小脑前向模型**。

### 当前 Column 的问题

```
现在：encoding z_t ──→ col_neuron (被动记录，EMA 平滑)
                       ↓ (无输出！)
                       死路
```

Column 接收编码信号，做了指数平滑，但结果**不去任何地方**。它是一个被动的数据收集器。

### Column 应该成为的样子

```
提议：
  ┌─────────────────────────────────────────┐
  │           Column 前向模型                 │
  │                                          │
  │  输入A：CPG 当前相位 (cpg_fast_a/b)      │
  │  输入B：上次信号到达的相位 (记忆)         │
  │  输入C：medium 传播延迟 (结构性常数)     │
  │                                          │
  │  计算：predicted_arrival = f(A, B, C)    │
  │  输出：phase_correction → CPG            │
  │         (加速或减速 CPG 以对齐信号到达)   │
  │                                          │
  │  误差：actual_arrival - predicted_arrival │
  │        → 修正 B 的权重                    │
  └─────────────────────────────────────────┘
```

这会把 CPG 门控从一个"哑开关"变成真正的**结构性对齐**：
- CPG 不再是固定频率——它被 Column 的预测**主动调整**
- 信号到达时间和 CPG 相位之间有**双向因果**
- Column 的权重记住了"信号通常在什么相位到达"

### 为什么这是"训练后固定"的？

> [!IMPORTANT]
> 关键洞见：Column 的学习率极低 (plasticity = 0.01，是 Spine 的 1/18)。
> 
> 这意味着 Column 的权重**几乎不变**——它在最初几百 tick 被"训练"后，
> 就近似固定，持续执行学习到的"对齐算法"。
> 
> 这正是你说的"千万年进化后塑造"：
> - 在我们的系统中，"进化"被压缩为"早期快速学习 + 后期近冻结"
> - 结构拓扑（Column 连接谁）是设计时固定的 = 进化
> - 权重（Column 怎么计算）是运行时学习的 = 个体发育
> - 低可塑性保证了学习后的稳定 = "算法固化"

## 你的假想 vs 真实世界：总结

| 方面 | 你的假想 | 真实世界 | 评估 |
|------|---------|---------|------|
| Column = 固定结构 | ✓ | 小脑/TRN 的基本拓扑是进化固定的 | **准确** |
| 执行特定"算法" | ✓ | 小脑 = 前向模型，TRN = 门控，上丘 = 坐标变换 | **准确** |
| 进化塑造 | ✓ | 基因决定拓扑，个体经验微调权重 | **准确** |
| 用于对齐 | ✓ | 小脑精确地做这件事：预测时序 → 调整相位 | **非常准确** |
| 用于翻译 | ✓ | 上丘精确地做这件事：视觉坐标 → 运动坐标 | **非常准确** |

> [!TIP]
> **你的假想不是假想——它是对真实神经计算的准确描述。**
> 
> 唯一的差异是：
> - 生物系统有**多个**这样的固定模块（小脑、TRN、上丘、基底节...）
> - 每个模块执行**不同的算法**
> - 我们目前只有一个 Column 层，且它没有执行任何有用的算法
