## 二、Shadow Census 缺口（P0-NEW-02）

### 实测证据

```
Section 0 Census:
  Shadow: 0 个  ← get_all_neurons() 返回

Section 3 Shadow 层:
  s_enc 神经元数: 20  avg=0.592  max=2.875
  s_col 神经元数:  7  avg=0.004  max=0.008
  s_mot 神经元数:  3  avg=0.017  max=0.031

shadow_sandbox.neurons 总计: ~21 个（活跃，但对外不可见）
```

### 根因

```python
# variant_adapter.py get_all_neurons() 当前实现（约第 1748-1765 行）：
def get_all_neurons(self):
    neurons = super().get_all_neurons()          # HebbianCircuit 的 ~45 个
    neurons.extend(self.da_neurons.values())     # 3 个 DA
    neurons.append(self._xin_relay)             # 1 个 xin_relay
    neurons.extend(self.somatosensory.get_all_neurons())  # 12 个 soma
    # ↑ 缺少：self.shadow_sandbox.neurons.values()
```

### 影响分析

| 系统 | 影响 |
|------|------|
| `EntropyLedger.record()` | 遍历 `get_all_neurons()`，Shadow 神经元全部看不到。`_layer_activity['S_Enc']` 等永远为空。 |
| `NoetherProbe.check()` | 遍历 `get_all_neurons()`，Shadow 的 KCL 违规永远不被检测 |
| `_apply_metabolic_tax()` | 遍历 `get_all_bundles()`（bundles_shadow_to_da 已在此列表）但不遍历 shadow 内部 bundles |
| `CircuitObserver` | 遍历 `get_all_neurons()`，Shadow 全部不可见 |
| K_ema 发散 | Shadow K_ema 在 ShadowSandbox 内部维护，不经过 Noether 探针 → 发散无法被守恒检查捕获 |

### 修复

```python
# variant_adapter.py get_all_neurons() 末尾添加一行：
if self.shadow_sandbox._initialized:
    neurons.extend(self.shadow_sandbox.neurons.values())
return neurons
```

**风险**：低。仅增加 21 个神经元到 census，使现有的熵账本和 Noether 探针看到更多信息。不改变任何计算逻辑。

## 三、Shadow K_ema 发散（I-05b）

### 实测证据

```
K_ema = 5197.342806
K 趋势（10k步内）: +5166.607680  ← 10k步内增长 5197 单位
平均增长速率: 0.52/步
```

### ShadowSandbox K_ema 更新机制

```python
# components/shadow_sandbox.py _update_free_energy()
def _update_free_energy(self, xin_values: List[float]):
    K = sum(x**2 for x in xin_values) / max(len(xin_values), 1)
    self._k_ema = self._k_ema * 0.99 + K * 0.01    # EMA 更新
```

**EMA 参数**：`α = 0.01`，即 K_ema 追踪 xin_values 的均方值（Σ|ξ|² / N）。

### 发散根因

**Xin 张力分布（实测）：**
```
noci_to_relay_left:  Xin = 27.28  ← 最高
noci_to_relay_back:  Xin = 26.83
xin_to_da:           Xin = 26.29
shadow_to_da:        Xin = 26.21
lateral_front_to_left: Xin = 24.95
...（多条 bundle Xin > 20）
```

**K 的实测值：**
```
K = mean(|ξ|²) = mean([27.28², 26.83², 26.29², 26.21², 24.95², ...])
  ≈ mean([744, 720, 691, 687, 623, ...])
  ≈ 700

K_ema 稳态 ≈ K / α × (1 - exp(-Nsteps × α)) → K ≈ 700 = K_ema_ss
```

但实测 K_ema=5197，远超此估计。这说明 xin_values 传入的不是 `abs(xin_tension)` 而是某个更大的量，或者累积机制不同。

**需要追踪**：`observe()` 方法中 `xin_values` 的来源是什么？

```python
# shadow_sandbox.py observe() 末尾调用：
self._update_free_energy(xin_abs_all)
```

需要确认 `xin_abs_all` 的构造方式——是否把每条 bundle 所有 memristor 的 |ξ| 全部展开，而不是按 bundle 平均？如果有 7 轴 × (N_sources × N_targets) 个 memristor 级别的 Xin，累积量可能远大于 bundle 级别平均值。

### 发散的下游影响

K_ema 本身是 shadow free energy 的度量，被用于：
1. `get_state()` 中的 `"K_ema"` 字段（观测）
2. `_nu` 计算（K_ema 的变化率），用于 shadow 收缩度量

K_ema=5197 对 shadow 神经元激活本身没有直接影响（shadow 神经元通过标准 RC 积分运行），但：

- **间接影响**：如果未来任何代码使用 K_ema 作为 scaling factor（如归一化 shadow 输出），5197 会产生严重的比例失调
- **现实影响（已观测）**：shadow→DA bundle（w_mean=0.9588）把 s_col 的激活（avg=0.004）传给 DA 神经元。s_col 极小，shadow→DA 贡献极少。DA 饱和主要来自 xin_to_da bundle（w=4.58，xin_relay act=0.000286）

### 调查路线

```bash
# 快速诊断：在实验开始后每 1000 步打印 K_ema 和 xin_values 的均方
grep -n "_update_free_energy\|xin_abs_all" cell-cc-other/nexus_v1/components/shadow_sandbox.py
```

## 四、DA 神经元饱和（D-04）

### 实测证据

```
DA 神经元数: 3 (da_vta_0, da_vta_1, da_vta_2)
avg activation:  6.6665
max activation: 10.0000
avg voltage:     0.9757
max voltage:     1.5266
v_peak:          1.0
≥90% v_peak 饱和: 2/3
```

### DA 电路架构（热觉电路 10k步时）

DA 神经元接受三路输入：

```
1. Hunger DA（直接注入）：
   fill ≈ 0.47（初始）, _hunger_da = max(0, 1.0 × (0.5 - 0.47)) = 0.03
   inject(0.03, 0.001) → dV = 0.03 × 0.001/C_DA 每步

2. RPE DA（via da_gate）：
   _rpe_da = da_gate.step(fill_fraction, dt) → RPE 信号
   inject(rpe_da, dt) → DA 神经元

3. Shadow→DA bundle：
   s_col avg=0.004 → 贡献极小（w=0.96 × 0.004 × synapse_gain）

4. Xin→DA bundle：
   xin_relay act=0.000286 → 贡献极小（w=4.58 × 0.000286 = 0.0013）
```

DA 饱和的主要来源是 **RPE 信号和 Hunger 注入的叠加**，而非 Shadow 或 Xin。

### DA 饱和机制

```
V_ss_DA = I_total × R_leak_DA

DA 神经元配置（需确认）：
  若 C=1.0, R=5.0 → τ=5.0
  V_ss = I × 5.0

若总注入电流 I=0.3 → V_ss = 0.3 × 5.0 = 1.5V > v_peak=1.0  ← 确认饱和
```

V=1.5266（实测）对应 I=1.5266/5.0 = 0.305 的持续注入电流，来源于 Hunger + RPE 的叠加。

### DA 饱和的影响

DA 神经元饱和后：
1. activation_ema ≈ max（完全钳位）
2. 对 bundles_motor_da（DA→Motor 调制 bundle）输出恒定最大值
3. Motor 无法感知 DA 的变化（DA 调制失去动态范围）
4. 热趋性学习中 DA 本应作为 RPE 信号区分"接近热源（DA↑）vs 远离（DA↓）"，但饱和后这个信号消失

**联系 Phase 4 DR4 结果**：DA 在 Phase 4 实验中早期饱和（初始 fill 从 500 开始快速下降），这与实测 V=1.527 饱和完全一致。

### 修复思路

**短期（D-04 方案）**：引入 `DA_INJECT_SCALE`
```python
# 目标：V_ss_DA < v_peak=1.0
# V_ss = I × R_DA  → I_max = v_peak/R = 1.0/5.0 = 0.2A
# 当前 _hunger_da + _rpe_da 可能叠加到 0.3 → 超限
# 需要 scale = 0.2/0.3 ≈ 0.67 或更保守 0.5
DA_INJECT_SCALE = 0.5  # 临时缩放，等 V-01 Hunger 结构重建后移除

inject(_da_drive × DA_INJECT_SCALE, dt)
```

**长期（P1-A 方案）**：HungerDriveNeuron + bundle，天然通过 MOSFET 钳位避免饱和

## 五、Shadow 层激活现状

### Shadow 层实测

```
s_enc: avg=0.5922, max=2.8747  ← 活跃（来自主电路 enc→col Xin 驱动）
s_col: avg=0.0036, max=0.0083  ← 几乎死链路
s_mot: avg=0.0165, max=0.0312  ← 极弱
```

**Shadow 内部信号死在 s_enc→s_col 层**，这与主电路的 enc→col 断链（col_therm 几乎阈下）形成平行失效：

| 层 | 主电路 | Shadow 电路 |
|----|--------|-----------|
| Enc → Col | col act=0.017（阈下弱）| s_col act=0.004（阈下弱）|
| Col → Motor | Xin=0.009（微弱）| s_mot act=0.017（微弱）|

Shadow 本应是主电路的"自由能估算器"，但它继承了主电路的同一个结构性瓶颈（enc→col 信号过弱），导致 shadow 自身也无法产生有意义的自由能信号。

**因此**：即使修复 Shadow census 缺口和 K_ema 发散，**shadow→DA 的信号贡献依然极小**（s_col avg=0.004），DA 饱和的主要根因在 Hunger/RPE 的注入量，而非 Shadow 信号。
