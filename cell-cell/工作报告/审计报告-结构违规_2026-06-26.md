## V-01 Hunger DA 直接注入 DA 神经元（P0）

### 现象

```python
# circuit/variant_adapter.py:894-903
_rpe_da = self.da_gate.step(self.energy_store.fill_fraction, dt)
_hunger_da = max(0.0, 1.0 * (0.5 - self.energy_store.fill_fraction))
_da_drive = max(_rpe_da, _hunger_da)
if _da_drive > 0:
    for nid, neuron in self.da_neurons.items():
        neuron._membrane.inject(_da_drive, dt)   # ← 直接注入
```

### 违规分析

本系统已有完整的 DA 结构电路：
- `bundles_shadow_to_da`：Shadow 列 → DA 神经元（STDP 可塑）
- `bundles_xin_to_da`：Xin relay → DA 神经元（STDP 可塑）

Phase 4-P1 新增的 Hunger 信号**绕过这两条 bundle**，直接调用 `_membrane.inject()`。虽然 `inject()` 本身 KCL 合规（追踪 `_q_in`），但它让 Hunger → DA 这条通路：

| 属性 | 通过 bundle | 直接 inject |
|------|------------|------------|
| STDP 可塑 | ✅ | ❌ |
| Xin 张力追踪 | ✅ | ❌ |
| 熵账本可见 | ✅ | ❌ |
| Noether 权重守恒检查 | ✅ | ❌ |
| 代谢税收扣除 | ✅ | ❌ |
| 可被 sprout/prune | ✅ | ❌ |

### 对 Phase 4 实验的影响

Phase 4 的核心失败（权重早期均匀饱和）的 DA 来源是 RPE（`_rpe_da`），而不是 Hunger。但 Hunger 信号在 fill<0.5 后持续激活（fill=0.2 时 hunger_da=0.3），使 DA 长期维持在高位，对 270k 步后的权重衰减阶段也造成干扰。

更重要的是：**Hunger→DA 这条通路在结构上是盲的**，STDP 永远无法对它产生调制。这违反了项目的核心设计原则——"行为从结构中涌现，不从公式中输出"。

### 修复方向

需要一个 `HungerDriveNeuron`（Capacitor 模拟饥饿状态积分）→ `SynapticBundle(hunger_to_da)` → DA neurons 的结构路径：

```
BIO: 下丘脑外侧区（LH）饥饿驱动 → VTA 投射（Wise 2004）
Sources: [hunger_neuron]  — 由 fill_fraction 驱动的 Capacitor
Targets: [da_neurons]
Bundle: initial_weight=0.5, stdp_lr=0.001, no decay
```

三问：
- Q1. 生物对应物：LH (lateral hypothalamus) → VTA 饥饿-DA 轴 (Wise 2004)
- Q2. 物理结构：hunger_capacitor → SynapticBundle(hunger_to_da) → da_neurons
- Q3. 参数依据：需从 fill=0.5 时 DA=0 到 fill=0 时 DA=0.3 推导 initial_weight

## V-02 VitalOscillator 直接注入 Motor（P2，继承遗留）

### 现象

```python
# circuit/variant_adapter.py:797-799（继承自主系统）
vital_outputs = self.vital_oscillator.step(self.energy_store, dt)
for i, mkey in enumerate(_VITAL_MOTOR_MAP):
    self.motor_neurons[mkey]._membrane.inject(vital_outputs[i], dt)
```

### 违规分析

VitalOscillator 的能量路径是合规的（从 EnergyStore 取电）。问题在于 VdP 振荡输出直接打到 Motor 膜，没有经过 `SynapticBundle`。

BIO 对应：心跳血流动力学脉动 → 运动神经元姿势震颤。这条通路在生物上是**神经连接**，不是直接电注入，应有：

```
BIO: 心脏节律 → 脑干网状结构 → 脊髓运动神经元
Sources: [vital_oscillator_relay]  — VdP 振荡器的"心率神经元"
Targets: [motor_x, motor_y, motor_z]
Bundle: 固定权重（非 STDP）
```

### 当前影响

对热趋性实验无直接影响（脉动进入所有 motor，不产生方向偏置）。但对 Noether 能量守恒的完整性有影响：vital energy 路径未经 bundle 追踪，从 EnergyStore 取出但没有对应的"发送路径"记录。

## V-03 Deviation Reflex 直接注入 Motor + 语义硬编码（P0）

### 现象

```python
# circuit/variant_adapter.py:912-917
DEVIATION_MOTOR_GAIN = 1.0
deviation = circ['deviation']
if deviation > 0.1:
    motor_drive = (deviation - 0.1) * DEVIATION_MOTOR_GAIN
    for key, mot in self.motor_neurons.items():
        mot._membrane.inject(motor_drive, dt)
```

### 双重违规

**违规 A — 语义硬编码**：`if deviation > 0.1 → 驱动所有电机` 是用 Python 条件把"体温偏差 → 运动增强"这个行为目标直接写进代码。

按 RULES.md §判断标准："如果你写了一个 if 或数学公式来决定电机的输出，就是硬编码。"

偏差对运动的影响应从 Circulation → Column 激活 → Motor 的物理电路中**涌现**，而不是用 `if deviation > 0.1` 触发。

**违规 B — 直接注入**：`mot._membrane.inject(motor_drive, dt)` 绕过 SynapticBundle，同 V-01。

### 量纲问题（见 D-01）

`DEVIATION_MOTOR_GAIN=1.0` 的校准在 `dt=1.0` 下进行（EXP-012 注释），实验在 `dt=0.001` 下运行，实际效果 1000× 小于预期（见 D-01 详述）。因此即使这个路径存在，在 Phase 4 实验中也**近乎无效**。

### 修复方向

Circulation 输出（deviation）应经由 `SynapticBundle(circulation_to_motor)` 驱动 Motor，STDP 决定哪个方向的偏差与哪个运动方向关联——这才是涌现。不应该用 `if deviation > 0.1` 硬接。

## V-04 Xin Relay 电压当电流直接驱动（P1）

### 现象

```python
# circuit/variant_adapter.py:982-990
self._xin_integrator.inject(delta_xin, dt)
self._xin_integrator.leak(0.5, dt)
xin_v = self._xin_integrator.voltage
...
self._xin_relay.step(self._xin_integrator.voltage * 0.5, dt)
```

### 违规分析

`_xin_integrator` 是 `Capacitor`，其 `voltage` 单位是 V（归一化电压）。
`_xin_relay.step(current, dt)` 的参数 `current` 单位是 A（电流）。

把电压（V）当电流（A）传入是**量纲错误**（见 D-04 关联问题）。

同时，Capacitor → Neuron 之间缺少 `SynapticBundle`。正确的结构应当是：

```
Xin Integrator (Capacitor) 
    → [SynapticBundle: xin_int_to_relay, synapse_gain≈1.0]
    → Xin Relay Neuron
    → [Bundle: xin_to_da]
    → DA Neurons
```

没有 bundle，Xin integrator 的状态无法被 STDP 追踪，xin 张力积累不能参与结构生长。

### 量纲核算

`xin_integrator.voltage` 在正常运行下约 0.0 ~ 1.0（Zener 箝位）。
传入 `_xin_relay.step(current=0.5×voltage, dt=0.001)` 中：
`dV_relay = current × dt / C_relay = 0.5 × 0.001 / C`

若 C_relay=0.15（Encoding 配置），dV = 0.0033V/step。这可以累积到放电阈值，所以功能上可能"工作"，但它是错误的量纲计算，与信号校准预期（synapse_gain=2.0, weight=2.5 等）完全不同。

## V-05 Lateral Inhibition 绕过 Bundle（P2）

### 现象

```python
# circuit/variant_adapter.py:882-890
col_activations = [self.column_neurons[ax]._activation_ema for ax in self._col_axes_order]
inhibitions = self.lateral_inhibition.compute_inhibition(col_activations)
for idx, axis in enumerate(self._col_axes_order):
    col = self.column_neurons[axis]
    if inhibitions[idx] < 0:
        col._membrane.inject(inhibitions[idx], dt)
```

### 违规分析

`LateralInhibition.compute_inhibition()` 是 Python 数学计算（WTA 或 k-winner）。结果直接注入 Column 膜。

CLAUDE.md 描述已有 `lateral_gain=0.3` 参数设定（Phase 4 修改 `SomatosensoryChain` 的 `lateral_gain`），说明 `InhibitorySynapse` 原语存在于系统中。Column 间的侧抑制应当：

```
BIO: 皮层抑制性中间神经元（Fast-Spiking PV+ interneurons）
    → Column A 激活 → [InhibitorySynapse] → Column B 抑制
```

当前实现把"所有轴竞争"的选择性逻辑硬编码进 Python 函数，而不是让 InhibitorySynapse 的物理属性（权重、增益、饱和）决定抑制强度。

### 影响

在 Phase 4 中，Lateral inhibition 把多轴同时激活的情况压到接近 0（已知问题：多轴输入下所有 Column 被驱动到零）。这个行为是因为 WTA 竞争太强，但因为实现在 Python 层，很难从物理上分析为什么——缺乏参数可调性（只有一个 Python 函数）和物理可解释性。

## 总结

| 编号 | 描述 | 严重 | 修复方向 |
|------|------|------|---------|
| V-01 | Hunger DA 直接注入 | P0 | 需 HungerDriveNeuron + hunger_to_da bundle |
| V-02 | Vital 直接注入 Motor | P2 | 需 vital_relay → motor bundle（继承遗留，低优先） |
| V-03 | Deviation reflex 语义硬编码+注入 | P0 | 应由 Circulation→Column→Motor 物理路径涌现 |
| V-04 | Xin relay 电压当电流 | P1 | 需 xin_int_to_relay bundle，量纲修正 |
| V-05 | Lateral inhibition 绕过 bundle | P2 | 需 InhibitorySynapse 替换 Python WTA |
