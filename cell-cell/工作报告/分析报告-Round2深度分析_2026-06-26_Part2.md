## 四、能量预算与可塑性窗口的定量分析

### 4.1 Fill 下降曲线重建

Round 2 关键数据（从报告重建）：

| 步数 | fill | yolk% | w_back |
|------|------|-------|--------|

| 500k | 0.000 | 0.000 | 0.483 |

### 4.2 fill 下降速率分析

```
阶段 1（0~100k）：fill 0.504→0.456，Δfill=-0.048，速率=-4.8e-7/步
阶段 2（100k~200k）：fill 0.456→0.109，Δfill=-0.347，速率=-3.47e-6/步 ← 加速！
阶段 3（200k~330k）：fill 0.109→0.000，速率=-8.38e-7/步
```

**阶段2的能耗激增**：100k~200k 期间 fill 下降速率是阶段1的 7.2倍。这正好对应身体**大幅 -x 运动**（x: 84.7→2.4，跨越 82 个单位）。大量电机力输出 → 能耗激增 → fill 快速下降。

这也解释了为什么 w_back 在 100k~200k 急速增长（LTP 最强）：正是大量运动 + fill>0（仍有可塑性）+ therm_back 激活 → STDP + 能量门控共同作用。

### 4.3 权重增长 vs. 可塑性窗口

```
权重增长定量（w_back）：
  0~100k：0.102→0.076（Δ=-0.026，初始衰减期，col尚未激活）
  100k~200k：0.076→0.335（Δ=+0.259，LTP主要阶段，fill=0.456→0.109）
  200k~300k：0.335→0.495（Δ=+0.160，LTP继续，fill=0.109→0.020）
  300k~330k：0.495→0.500（Δ=+0.005，fill接近0，可塑性将尽）
  330k~500k：0.500→0.483（Δ=-0.017，fill=0，无新LTP，轻微漂移）

LTP 效率（Δw/Δsteps）：
  100k~200k：+0.259 / 100k = 2.59e-6/步（高效期）
  200k~300k：+0.160 / 100k = 1.60e-6/步（中效期）
  300k~330k：+0.005 / 30k  = 1.67e-7/步（耗竭期，fill接近0）
```

### 4.4 关键洞察：330k 步已足够实现饱和

```
w_back 在 300k 步时已达 0.495（weight_max=0.5 的 99%）！

这意味着：
- 从功能角度看，Round 2 的学习在 300k 步就已基本完成
- 后续的 200k 步（330k~500k）仅是在 fill=0 状态下维持
- 延长可塑性窗口（如 Round 4 的 fill 全程>0）对 therm_back 增益有限
  （已接近 weight_max，无法再增长）
- 对 therm_front/left/right 的影响：可能在 Round 4 中看到这些束的新动态
```

### 4.5 能量预算公式推导（供 Round 4 设计参考）

```
Round 2 数据反推消耗速率：
  fill 在 100k~200k 阶段：-3.47e-6/步（净消耗 = consumption - lambda）
  lambda_R2 = 0.0015/步
  → consumption ≈ 0.0015 + 3.47e-6 ≈ 0.001503/步（大运动期）

  fill 在 0~100k 阶段：-4.8e-7/步（小运动期）
  → consumption ≈ 0.0015 + 4.8e-7 ≈ 0.001500/步（接近 lambda）

  fill 在 200k~330k 阶段：-8.38e-7/步（中等运动）
  → consumption ≈ 0.0015 + 8.38e-7 ≈ 0.001501/步

加权平均消耗率：≈ 0.001502/步

Round 4 参数验证：
  lambda_R4 = 0.002/步 > 0.001502/步 ✓
  net fill 增速 = 0.002 - 0.001502 = 4.98e-4/步（正向）
  fill @ 500k ≈ 0.5 + 4.98e-4 × 500k = 248.5 → 1.0（满格）✓
  
  yolk @ 500k = 1000 - 0.002 × 500k = 1000 - 1000 = 0 ✓（正好用尽）
```

**Round 4 设计验证通过**：lambda=0.002, yolk=1000 在理论上确保 fill 全程满格且 yolk 正好在 500k 步耗尽。

## 五、World 2.0 架构升级预研

### 5.1 当前架构的主要局限

基于三轮实验的观察，当前 World（World 1.0）存在以下局限：

| 局限 | 表现 | 影响 |
|------|------|------|
| **点热源分布** | 8个点热源，热场不连续 | 体在热源间运动时梯度反转，学习信号不稳定 |
| **yolk 线性消耗** | 代谢窗口固定（≤333k~625k步）| 能量约束主导行为，DR3/DR5 相互牵制 |
| **无进食机制** | 能量只来自 yolk | 无法构造"趋热→找食物→代谢补偿"的完整循环 |
| **体无转向** | 仅 move_x/y/z，无偏航 | "前进向热源"不可能，只能"倒退"或"绕行" |
| **align 指标** | 瞬时速度 vs 热源方向 | 无法区分"倒车趋热"与"随机游走" |

### 5.2 World 2.0 核心需求清单

#### 需求 1：连续温度场（Continuous Thermal Field）

```
当前：T(x) = Σ T_i / (1 + dist_i²)（点源叠加）
World 2.0：T(x) = Gaussian field 或 Perlin noise + 点源叠加
优先级：P1
实施难度：低（修改 World.temperature_at(pos)，不改神经回路）
预期效果：热梯度连续、单调，学习信号更稳定
```

#### 需求 2：物理接触进食机制（Energy-from-food）

```
当前：YolkSac 线性消耗，与位置无关
World 2.0：食物资源散布在热源附近，体接触食物时 energy_store 充能
优先级：P1
实施难度：中（需要 World 增加 FoodSource 对象，Body 增加 eating 触发）
预期效果：DR3 自然通过（活动→找食物→进食→补充 fill）
          align 指标可改为"趋食物"而非"趋热源"
          能量约束与学习目标一致（而非矛盾）
```

#### 需求 3：体旋转/偏航机制（Yaw control）

```
当前：body 始终面朝 +x，无转向
World 2.0：motor_z 输出驱动偏航角，body 可转向
优先级：P2（可选）
实施难度：中（需修改 body.update_position 添加 yaw 计算）
预期效果：允许"面向热源前进"策略，丰富可能的运动模式
注意：需同步修改皮肤贴片世界坐标计算（patch.world_position 依赖 body 朝向）
```

#### 需求 4：改进的趋热对齐指标

```
当前：align = dot(Δpos, src_direction)（瞬时）
World 2.0：align_cumulative = 累积位移与热源梯度的相关性
         或：thermal_gradient_following = V_dot_∇T / |V||∇T|
优先级：P1（仅影响评估，不影响神经回路）
实施难度：低（修改实验脚本的 thermal_alignment 函数）
预期效果：DR5 指标更能真实反映趋热行为（包括"倒车向热源"）
```

#### 需求 5：多轴感知对称性改善

```
当前问题：
  - x 轴：热场变化大（x=25 和 x=75 两组热源），front/back 差异>0.5°C
  - y 轴：体在 y=20，热源在 y=25/75，ΔT(left-right)=0.43°C（偏弱）
  - z 轴：类似 y 轴（更对称）

世界设计建议：
  - 将热源改为 1 个强热源（如 T=10 @ [50,50,50]）
  - 或 4 个热源分布在 4 个象限，形成均匀的双轴梯度
  - 使 x 和 y 轴的温度梯度大小相当（±0.5°C 左右差异）
优先级：P1（不改神经回路，只改世界参数）
```

### 5.3 World 2.0 最小可行版本（MVP）

按优先级，最小修改使 Phase 6 实验可运行：

```
MVP 变更（不改神经回路）：
  1. [热场] 将热源改为 1~2 个强热源（简化梯度）
  2. [食物] 在热源附近添加 4~8 个 FoodSource，接触即补充 fill
  3. [指标] 改进 align 计算：使用温度梯度方向而非点源方向

核心规则：
  - 所有变更在 World/Body 层实现，不触碰神经回路
  - SynapticBundle / Neuron / VariantCircuit 保持不变
  - 新功能通过 world.temperature_at() / body.metabolic_update() 注入
```
