## 五、DR5=34% 深度分析

### 5.1 align 指标定义与局限

```python
def thermal_alignment(pos_now, pos_prev, src):
    v = pos_now - pos_prev              # 当前速度向量
    g = src.position - pos_now          # 指向最近热源的方向向量
    return dot(v_normalized, g_normalized)
```

DR5 统计的是 align > 0 的步数比例（pos_frac > 50% = 朝热源运动时间超过一半）。

### 5.2 为什么 y 轴趋热导致 align 下降

Round 4 身体沿 +y 方向运动，热源分布在 y=25 和 y=75 两层。

| 阶段 | body y 范围 | 热源方向 | 体运动方向 | align 符号 |
|------|------------|---------|---------|-----------|
| 0~250k | y: 20→40 | +y（y=75 更近）| +y | **+（正确）** |
| 250k~350k | y: 40→70 | +y（接近 y=75）| +y | **+（正确）** |
| 350k~400k | y: 70→83 | -y（y=75 已越过）| +y | **-（过冲！）** |
| 400k~490k | y: 83→99 | -y（远离 y=75）| +y | **-（过冲）** |
| 490k~500k | y: 99→1.5（边界）| +y（y=25 现在更近）| +y→乱 | 混乱 |

**核心问题**：体在 350k 步之后已经超越 y=75 热源，继续 +y 运动实际上在"远离"最近热源，align 转为负值。

从 500k 步的采样数据分析（取 10k 步间隔的 log 行）：
```
正 align 步骤 (近似采样)：
  30k, 40k, 50k, 60k, 120k, 190k, 200k, 210k, 240k, 250k, 290k, 310k, 320k, 330k, 460k
  ≈ 15 个采样点

负 align 步骤 (近似采样)：
  10k, 70k, 80k, 90k, 100k, 110k, 130k, 140k, 150k, 160k, 170k, 180k, 220k, 230k, 260k,
  270k, 280k, 300k, 340k, 400k-490k（连续 9 个）, 500k
  ≈ 30 个采样点

采样比例：~33%，与 align_pos_frac=0.340 一致
```

### 5.3 DR5 失败不等于"趋热失败"

**实际行为评估**：

```
从初始位置 [75, 20, 25] 到 400k 时 [40.2, 83.9, 81.5]：
  y 轴位移：+63.9 units（从 y=20 到 y=83.9）
  最近的 y=75 热源集群（src[2/3/6/7]）确实已被接近并越过！

如果用"是否到达热源附近"判断：
  体在 350k~400k 时 dist≈19~31（距最近热源约 20~30 units）
  而初始 dist=5.0 到最近热源（src[4]=[75,25,25]），所以有时更近有时更远
  但 body 确实学会了朝 y=75 热源方向运动——只是没有"停止"机制！
```

**更准确的 DR5 判断指标**（建议 World 2.0 改用）：
1. **到达指标**：在某个步数窗口内，dist < threshold（如 dist < 15）的步骤占比
2. **热流量指标**：在身体经历的温度序列中，mean(T_experienced) vs mean(T_initial)（是否比初始更温暖）
3. **累积位移指标**：Δy_cumulative（对 World 2.0 修改后的单热源更有意义）

### 5.4 结论：DR5 是指标设计问题，不是学习失败

证据：
- 体确实向 y=75 热源方向移动（Δy@400k=+64）
- 权重发散已饱和（w_right=0.500，spread=0.494）
- 过冲是"无停止机制"导致，不是"没有学到方向"
- align=34% 主要由 350k~500k 的过冲段拉低（15万步连续负值）

## 七、遗留问题分析

### 7.1 过冲问题（无停止机制）

当前架构中，一旦 w_right 学会驱动 +y 运动，这个驱动力会**永久持续**，不会因为"已到达热源"而停止。

```
当前行为：热源@y=75 → 体学会→y → 越过y=75 → 继续→y → 越过y=100 → 回到y≈0
所需行为：热源@y=75 → 体学会→y → 到达y=75 → 停止（或维持在热源附近）
```

**潜在解决机制**（World 2.0 方向）：
- 物理接触进食（body 到达热源/食物后 energy_store 充能 → 改变代谢状态 → 改变驱动）
- 对侧抑制（体到达热源后，front/back 温差消失 → 两端 col 激活相同 → 驱动互相抵消）
- 这两种都需要 World 2.0 机制，当前架构无法解决

### 7.2 Column 活动稀疏（[!NOETHER] 持续）

全程 [!NOETHER] 报警（kcl_charge 和 xin_conservation 超限）是已知问题，与 Phase 5 功能性不直接相关，但表明系统内部物理守恒约束需要在 World 2.0 阶段系统性解决。

col 平均激活值（所有步骤）：
```
col_therm_front:  0.00091
col_therm_back:   0.00156
col_therm_left:   0.00588
col_therm_right:  0.00561
```
y 轴 col（left/right）的活跃度是 x 轴的 3.7 倍，这与 w_right 饱和的主导方向一致。

### 7.3 x 轴权重全面衰减（Round 4 新发现）

在 Round 2 中，w_back 饱和到 0.5。在 Round 4 中，所有 x 轴权重都衰减：

```
Round 4 @ 500k:
  w_front: 0.07036 → 0.01908（衰减 73%）
  w_back:  0.06308 → 0.01552（衰减 75%）
  
Round 2 @ 500k:
  w_front: ≈0.022（衰减）
  w_back:  0.483（增长到 weight_max）
```

这说明 Round 4 的 y 轴学习"压制"了 x 轴——两个轴都在竞争有限的 DA 资源。当 y 轴 col 更活跃（由于初始 ΔT(left-right)=0.43 > 0），y 轴抢先学习，x 轴反而被衰减抑制。
