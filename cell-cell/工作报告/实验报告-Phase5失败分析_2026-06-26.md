## 二、失败原因

### 根因 1：eligibility_gain=1.0 → 单步权重飞越

热觉束 (`therm_*_to_move_*`) 使用 `use_eligibility_trace=True`：

```python
ltp = eligibility_gain × trace × DA
```

在 dt=0.001 下，10k 步后 trace ≈ pre × post × 10（对比 tau=300），DA=0.785：

```
ltp = 1.0 × 10 × 0.785 = 7.85 (每步!)
dw = 7.85 × (0.5 - 0.08) × plasticity(0.18) = 0.59
```

**一步内 w 从 0.08 跳到 max=0.5（被 weight_max 钳位）。**

关键错误：eligibility trace 积分没有除以 tau 做归一化。dt=0.001 × tau=300 → trace 稳态 = tau = 300。但经过 10k 步后 trace≈10 就已经造成单步飞越。

### 根因 2：DA_V=0.785 超出设计基态

DA 设计目标：
- 基态 V=0.1V（tonic 9%）
- 最大 RPE V=0.6V（59%）

实测 Phase 5 第 20k 步 DA_V=0.785V（超出最大 RPE 目标 30%）。  
原因：DA_INJECT_SCALE=0.1（7× 旧值 0.014）+ shadow_to_da + xin_to_da 束联合贡献，实际稳态远超预期。

## 四、修正方案

### Fix A：归一化 eligibility_gain（必须）

将 `eligibility_gain=1.0` 改为 `eligibility_gain=0.003`：

```
target: ltp_per_step ≈ 0.0001（约合旧 stdp_lr×dt）
formula: ltp = 0.003 × trace(≈10) × DA(0.785) = 0.024
dw = 0.024 × 0.42 × 0.18 = 0.0018/步
→ 从 0.08 到 0.5 需要 ~230 步全激活 → 正常学习时间尺度 ✓
```

物理依据：eligibility trace 公式缺少 `÷ tau` 归一化。`eligibility_gain = 1/tau = 1/300 ≈ 0.003` 使积分量纲正确。

**文件**: `cell-cc-other/nexus_v1/circuit/hebbian.py`，热觉束 BundleConfig：
```python
# 旧
eligibility_gain=1.0,
# 新
eligibility_gain=0.003,     # = 1/tau = 1/300 (trace归一化修正)
```

### Fix B：确认 DA 基态（应检查）

在 10k 步探针下确认 DA_V 不超过 0.5。如仍偏高（>0.5），考虑：
- 将 DA_INJECT_SCALE=0.1 → 0.05（中间值）
- 或减小 shadow_to_da initial_weight（已为 0.05，可进一步降至 0.02）
