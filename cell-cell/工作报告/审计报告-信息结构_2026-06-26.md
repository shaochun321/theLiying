## I-01 ThermalMembrane 死计算（P1）

### 现象

```python
# circuit/variant_adapter.py:665-668
# Thermal membrane senses environment at new body position
# (legacy single-point sensor kept for backward compatibility)
therm_signal = self.thermal_membrane.sense(
    self.world, self.world.body, dt)
```

`thermal_membrane.sense()` 每步计算：
1. `T_raw = world.temperature_at(body.position)` — 采样世界温度场
2. 甲基化适应：`M += (T - M) × dt/τ`
3. 计算 `adapted = T - M` 和 `dT/dt`
4. 返回 `{'therm': adapted, 'dtherm': dT}`

但在 cell-cc-other 中：
```python
# circuit/variant_adapter.py:124-127
_patch_axes = ["therm_front", "therm_back", "therm_left", "therm_right"]
super().__init__(extra_axes=_patch_axes)
```

`extra_axes` 不包含 `"therm"` 或 `"dtherm"`，因此 `therm_signal` 字典被赋值后**从不被读取**。

### 影响评估

**直接影响**：每步浪费一次 `temperature_at()` 调用 + 甲基化状态更新。在 500k 步实验中产生 ~500k 次无效计算。更严重的是，这个"死传感器"维护了一个内部状态（`_methylation`），其状态随时间漂移，既不被任何神经元读取，也不出现在 entropy ledger 中。

**设计矛盾**：`ThermalMembrane` 的设计文档（`thermal_membrane.py` docstring）写道：
> "Direction information emerges from cross-modal learning... This is exactly how C. elegans achieves thermotaxis"

而 cell-cc-other 改用了 4-patch 的 `SomatosensoryChain`，已提供方向性温度信息。两个设计共存但只有一个被接入电路，另一个成为死代码。

### 修复方案

**选择 A（短期）**：删除 `thermal_membrane.sense()` 调用，如果确认不再需要：
```python
# 删除 circuit/variant_adapter.py:665-668
# 删除 self.thermal_membrane = ThermalMembrane() 初始化
```

**选择 B（长期）**：如果 C. elegans 式的单点时序热传感仍有价值，应当明确将其接入电路作为 `therm`/`dtherm` 第五和第六 extra_axis，与 4-patch 空间感知形成互补。

```
therm (temporal): → enc_reg_therm, enc_irr_therm (C. elegans klinokinesis)
therm_front/back/left/right (spatial): → enc_reg/irr_therm_XX (patch comparison)
```
这对应两种热趋性机制共存：时序比较（C. elegans）和空间比较（Drosophila/ vertebrate），可以相互验证。

## I-02 `grad_dot_v` 计算结果被丢弃（P2）

### 现象

```python
# circuit/variant_adapter.py:998-1008
T_p = {pid: patch_temps[pid][0] for pid in self.somatosensory.patch_ids}
grad_T = [
    T_p.get("right", 0.0) - T_p.get("left", 0.0),   # y 方向温度梯度
    0.0,
    T_p.get("front", 0.0) - T_p.get("back", 0.0),    # x 方向温度梯度
]
vel = list(self.world.body.velocity)
grad_dot_v = sum(grad_T[i] * vel[i] for i in range(min(3, len(vel))))
mean_dT = sum(abs(patch_temps[pid][1])
              for pid in self.somatosensory.patch_ids
              ) / len(self.somatosensory.patch_ids)
self.motion_state.thermal_potential = mean_dT   # ← 只存 mean_dT
# grad_dot_v 从未被存储或使用
```

### 丢失信息的诊断价值

`grad_dot_v = ∇T · v`（温度梯度与速度的点积）是判断**生物体是否在向热源方向移动**的直接指标：
- `grad_dot_v > 0`：移动方向与温度升高方向一致（朝向热源）
- `grad_dot_v < 0`：移动方向与温度降低方向一致（远离热源）
- `grad_dot_v ≈ 0`：横向移动

Phase 4 实验的验收标准 DR2（Δy(200k) > 1.0）就是在检验定向移动，但 `MotionState` 中没有实时记录 `grad_dot_v`，无法在实验过程中直接观测热趋性的有效性。只有 `mean_dT`（全局皮肤温变幅值），无法区分"朝热源走产生的 dT"和"随机游走恰好经过温变区产生的 dT"。

### 修复方案

```python
# 在 motion_state 中增加 grad_dot_v 字段
self.motion_state.thermal_potential = mean_dT
self.motion_state.thermal_alignment = grad_dot_v   # 新增：热趋性对齐度
```

同时，实验脚本中应当在每个检查点打印 `thermal_alignment`，而不只是打印 `col_therm_left/right EMA`。

注：`grad_dot_v` 的计算本身是 Python 语义计算（直接计算方向），不应作为控制信号。它只应出现在观测/日志层。

## I-03 Hunger DA 对熵账本不可见（P2）

### 现象

Hunger DA 信号通过 `neuron._membrane.inject(_da_drive, dt)` 注入，这意味着：

**熵账本（`EntropyLedger`）**：遍历 `get_all_bundles()`，计算每条 bundle 的权重熵、Xin 张力、能量流。Hunger DA 电流没有 bundle → 不在任何 bundle 里 → **熵账本看不到这条通路**。

**Noether 探针（`NoetherProbe`）**：检查权重守恒 `Σw_in = Σw_out`（`xin_conservation`）。Hunger DA 没有 Memristor 权重 → 不参与权重守恒检查。但它消耗了 DA 神经元的膜电流，这部分"能量"的来源在 Noether 探针中没有对应的 `_q_in` 记录（DA 神经元的 `_q_in` 会增加，但没有对应的 source-side `_q_out`）。

**代谢税收**：`_apply_metabolic_tax()` 按 `get_all_bundles()` 遍历，每条 bundle 收 BUNDLE_BASAL_COST=0.0005。Hunger 路径没有 bundle → 不被收税 → 能量收支不对称。

### 对 Noether 守恒的影响

在实验报告中，xin_conservation 累计了 1632 次违规（见 I-05）。这些违规的部分来源可能是：在 Noether 探针看不到的地方（Hunger inject）流入 DA 膜的电流，打破了电荷守恒检查中的 `_q_in - _q_out - dq_stored = 0` 假设。

但 Noether 探针检查的是**每个组件**的 KCL（`component.kcl_imbalance`），而不是 bundle 级别的守恒。DA 神经元膜的 `_q_in` 确实会随 `inject()` 更新（KCL 合规），所以单个组件不违规，但在**束级别的能量流守恒**中，这条通路是盲的。

### 修复方案

完整修复见《审计报告 A》V-01 的 `HungerDriveNeuron` 方案。
短期可行的最小修复：向 `MotionState` 增加 `hunger_da_current` 字段记录每步的 `_hunger_da` 值，使其至少在观测层可见。

## I-05 两个已知开放问题未开 DEG 案（P1）

### 问题 A：xin_conservation 累计 1632 次违规

**观察**：Phase 4 实验中 `xin_conservation` 违规从 0 增加到 1632，在约 140k 步时停止增加。

**已知事实**：
- xin_conservation 检查的是 bundle 级别的 Xin 张力守恒
- 违规在 140k 步停止累积（不是 Noether 探针冻结，因为 KCL fix 后 Noether 继续运行）
- 140k 步恰好在 `_da_circuit_initialized = True` 之后不久（DA 电路约在 1000 步后初始化）
- 可能的解释：DA 电路初始化后，某些 bundle 的 Xin 张力归一化到正常范围，不再产生守恒违规

**未调查的假设**：
1. DA 电路初始化前，`bundles_shadow_to_da` 和 `bundles_xin_to_da` 为空但被遍历，产生边界情况
2. Phase 4 新增的 4 个 therm extra_axes 在 140k 步之前产生大量 Xin 张力溢出
3. YolkSac 放电阶段（fill 变化快）产生的 Xin 张力不守恒

**应开 DEG 案**：
```markdown
### DEG-P4-001: xin_conservation 累计 1632 次违规后停止
- 发现时间: 2026-06-26
- 现象: Phase 4 实验中 xin_conservation 从 0 增加到 1632，@~140k 步停止
- 影响层: TOPRXinLedger, Xin 张力守恒
- 根因: 未查明（待调查：DA 电路初始化时序 / therm 额外轴 Xin 边界）
- 状态: OPEN - INVESTIGATING
```

### 问题 B：Shadow 层 `K_ema` 无界增长（shadow free energy 发散）

**观察**：来自 CLAUDE.md 已知开放问题："shadow-layer free energy diverges (`K_ema` grows unbounded)"。

**已知事实**：
- Shadow layer 是 Enc→Col→Mot 的只读结构副本，由 Xin 张力驱动
- `K_ema` 是 shadow 层的"自由能"估计，理论上应在 Xin 驱动下达到稳态
- 无界增长说明 shadow free energy 的衰减机制失效

**对系统的影响**：
- Shadow col neurons 的激活（`shadow_col._activation_ema`）无界增长时，`bundles_shadow_to_da` 的 DA 输入也无界增长
- 这可能是 DA 饱和的另一个来源（早于 RPE burst）

**应开 DEG 案**：
```markdown
### DEG-P4-002: Shadow 层 K_ema 无界增长（free energy 发散）
- 发现时间: 2026-06-26（记录于 CLAUDE.md v1.7.2 已知问题）
- 现象: K_ema 随步数单调递增，无稳态
- 影响层: ShadowSandbox, bundles_shadow_to_da, DA 神经元激活
- 根因: 未查明（shadow free energy 衰减机制？Xin 无上界？）
- 状态: OPEN - INVESTIGATING
```
