## 二、四项修复实施

### Fix 1：Noci 参数修正（P0-NEW-01）

**文件**: `somatosensory/chain.py` → `_nociceptor_config()`

**根因回顾**：v_peak=0.01 比 C-fiber AP 阈值低 25 倍，热瞬态触发 500Hz 爆发放电，
w_adapt 积累至峰值 ≈ 20，爆发结束后 V_ss = -w_adapt × R_leak = -20 × 50 = -135V。

```python
# 修复前
v_peak=0.01,   b_adapt=0.02,   tau_w=5.0

# 修复后
v_peak=0.25,   # BIO: TRPV1 C-fiber AP threshold V_th≈-47mV → norm=0.254 (Julius & Basbaum 2001)
b_adapt=0.0002, # BIO: w_adapt_ss=0.005 @ 50Hz; V_ss=-0.005×50=-0.25V (Caterina 1997)
tau_w=0.5,     # BIO: C-fiber AHP τ≈200-500ms (Caterina et al. 1997)
```

### Fix 2：热感换能系数校准（P0-NEW-03）

**文件**: `somatosensory/chain.py` → 类属性 + `step()`

**根因回顾**：T_skin=4.2 直接注入 thermo（设计假设 T_skin≤0.2），V_ss = 4.2 × 5.0 = 21V，
thermo 饱和 → relay avg = 7.26（超出 [0,1]）→ reg_therm 超饱和 → 方向差异被 MOSFET 钳位消除。

```python
# 新增类属性
THERMAL_TRANSDUCTION_GAIN: float = 0.1
# BIO: TRPV3/TRPM8 transduction efficiency ~0.1 (Brauchi et al. 2004)

# step() 中修改
self.thermoreceptors[pid].step(T * self.THERMAL_TRANSDUCTION_GAIN, dt)
```

### Fix 3：Shadow Census 补丁（P0-NEW-02）

**文件**: `circuit/variant_adapter.py` → `get_all_neurons()`

**根因回顾**：`get_all_neurons()` 遍历所有观测系统，Shadow 21 个神经元不在 census
→ EntropyLedger / NoetherProbe / 代谢税全部看不到 Shadow。

```python
# 在 somatosensory neurons 之后追加一行
if self.shadow_sandbox._initialized:
    neurons.extend(self.shadow_sandbox.neurons.values())
```

### Fix 4：热觉 Column v_peak 精确修复

**文件**: `circuit/hebbian.py` → 新增 `_thermal_column_config` + 列皮层创建循环

**难点**：全部 Column（前庭 + 热觉）共用 `_column_config`，不能直接改 v_peak，
否则会破坏前庭 Xin 频率（T5.1 回归）。沿用 encoding 层已有的 extra_set 分支模式：

```python
# 新增函数（mirrors _thermal_encoding_config 的模式）
def _thermal_column_config(name: str) -> NeuronConfig:
    # 同 _column_config，仅 v_peak=0.20
    # BIO: L2/3 pyramidal AP threshold V_th≈-52mV → norm=0.215 (Liu 2014)
    ...

# 创建循环修改（原来是单一调用）
for axis in self.all_axes:
    cfg_fn = _thermal_column_config if axis in extra_set else _column_config
    self.column_neurons[axis] = Neuron(cfg_fn(axis))
```

**中间回归**：直接改 `_column_config` v_peak=0.20 使 T5.1 失败（Xin 频率 0.49Hz → 0.98Hz），
因前庭 col 双峰触发导致频率翻倍。精确分支后 T5.1 恢复（0.49Hz）。

## 五、方法论记录

### 中间回归及处置

第一次 Fix 4 直接修改 `_column_config` v_peak=0.10→0.20，导致 T5.1 回归：
```
T5.1 Xin peak near input freq: 0.98Hz  ← FAIL（期望 0.5Hz ±0.2Hz）
```

根因：前庭 col 的阈值从 0.10 升到 0.20 后，在 0.5Hz 正弦输入的两个极值附近各触发一次，
Xin 频率翻倍至 ≈1Hz。

修复：仿照 encoding 层已有的 `_thermal_encoding_config` 模式，引入 `_thermal_column_config`，
在列皮层创建循环中用 `extra_set` 分支路由。全部用时 1 次回归 → 1 次精确修复。
