## 三、新发现（原报告未覆盖）

### P0-NEW-01：Noci 神经元电压失控

```
noci_front: V = -134.33V
noci_back:  V = -126.82V
noci_left:  V = -124.07V
noci_right: V = -135.72V
```

- **根因**：v_peak=0.01（比归一化方案 C-fiber 阈值低 20×），热瞬态触发爆发式放电，w_adapt 积累 → V_ss = -w_adapt × R_leak = -2.7 × 50 = -135V
- **影响**：Noether violations=90（全部来自 |V|>100）；noci→relay 链路死亡；irr_therm 完全死链路
- **详见**：审计报告-Noci失控

### P0-NEW-02：Shadow 神经元不在 census

```
get_all_neurons() → Shadow: 0
shadow_sandbox.neurons → 21 个活跃神经元（s_enc avg=0.592）
```

- **根因**：`VariantCircuit.get_all_neurons()` 未 `extend(shadow_sandbox.neurons.values())`
- **影响**：熵账本看不到 Shadow；Noether 探针看不到 Shadow；代谢税收不覆盖 Shadow；Shadow K_ema 发散无法被守恒检查捕获
- **修复**：一行 census 补丁（非结构变更）

### P0-NEW-03：reg_therm 编码层超饱和，phasic 通道全死

```
reg_therm_left:  activation_ema = 1.2293  ← 超出归一化上界 1.0
reg_therm_front: activation_ema = 1.1556
irr_therm_left:  activation_ema = 0.0000  ← 完全死链路
irr_therm_front: activation_ema = 0.0000
```

- **根因 1（reg_therm 饱和）**：thermo→relay 链路增益过大（relay avg=7.26 → enc 接收 7.26×synapse_gain×weight 的驱动，超出 [0,1] 范围）
- **根因 2（irr_therm 死）**：irr_therm 依赖 noci→dtherm 信号，noci 深度超极化后 EMA=0；在静止身体条件下 dT/dt≈0 也无 AC 信号
- **影响**：col_therm 只接到常数饱和信号，无差异 → STDP 无法学方向 → |Δw|=0
- **详见**：审计报告-热觉链路失效

### P0-NEW-04：前庭电路 3 个 NaN/Inf 神经元

```
Phase 1（dt=1.0 前庭电路）：NaN/Inf 神经元: 3
```

- 前三份审计报告未检查 NaN/Inf（只检查 |V|>100）
- dt=1.0 下也存在 NaN/Inf，说明与 dt 无关
- 需要进一步诊断：哪 3 个神经元？属于哪一层？
- 暂标为 **INVESTIGATING**（不影响前庭回归 21/21 PASS，可能是已钳位的非活跃神经元）

## 五、方法论反思

本次审计的主要收获不只是发现新问题，而是揭示了一个**方法论缺陷**：

以代码阅读为主的审计（原三份报告）能发现结构性问题，但**无法区分理论隐患与实际瓶颈**。具体体现：

| 问题 | 代码审计判断 | 实测发现 |
|------|------------|---------|
| D-03 col v_peak | 理论隐患（量纲不对） | 实测瓶颈（col 勉强到阈，热觉 col 几乎死） |
| I-05a xin_conservation | OPEN（1632 违规） | 实测关闭（0 违规） |
| Noci 参数 | 未发现（未读 chain.py） | 实测 P0（Noether 90 违规根因） |
| Shadow census | 未发现 | 实测 P0（21 神经元不可见） |

**结论**：每次重要参数变更前后，应运行 `run_full_audit.py`，以实测数据驱动优先级，而非以代码阅读推断。
