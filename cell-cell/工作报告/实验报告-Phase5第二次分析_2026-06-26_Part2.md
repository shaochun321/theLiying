### 原始校准逻辑（针对 dt=1.0）

```python
# hebbian.py line 683-701 注释的校准依据：

# 设计场景（dt=1.0）：
#   relay_Phase4_buggy ≈ 2.35（somatosensory relay EMA，dt=1.0）
```

| 场景 | dt=1.0 relay | dt=0.001 relay（10k步稳态） |

## 六、根因归纳（三层叠加）

| 层级 | 根因 | 效果 |
|------|------|------|
| L1（已修复）| eligibility_gain=1.0 | 权重单步飞越 |
| L2（当前根因）| relay 建立时间 >> body 停留时间 | col_therm 永远不激活 |
| L3（潜在）| DA_V=0.756~0.811（偏高）| 即使 col 激活后 LTP 仍可能过快 |

### L2 根因的物理本质

relay 是 somatosensory chain 内的积分神经元，其稳态值（EMA）需要持续热刺激来积累。在 Phase 5 实验中：

1. **起步近热源**（dist=5）：body 没有运动控制，受随机噪声漂移
2. **10秒内漂离**：relay 建立到 0.040 → body 已在 dist=17.9
3. **后续热源接触**：每次靠近 ~1-2k 步就漂离，relay 重新从低值积累
4. **relay 从未建立**：全程 relay ≈ 0.04 → enc 不激活 → col 不激活

注：即使 relay 建立到 4.37（10k 冻结步），enc_act 仍只有 0.041（col V=0.083 < v_peak=0.20），原因是：
- enc_act 跟随 relay 的时间均值，而非瞬时值
- 需要更高的 EXTRA_AXIS_GAIN 或更长的停留时间让 enc 达到充分激活

## 七、修复方案分析

### 方案 A：提高 EXTRA_AXIS_GAIN（最小改动）

**目标**: 让更小的 relay 值也能激活 enc_therm

推导：要使 relay=0.1（建立中的早期值）就能让 col V 超过 v_peak=0.20：

```
col V需求链：
  V_col = enc_act × w × synapse_gain × coupling_factor × r_leak_col
  
实测比例（10k步，relay=4.37，enc_act=0.041，col_V=0.083）：
  col_V / enc_act = 0.083 / 0.041 = 2.02

要使 col_V > 0.20：
  enc_act > 0.20 / 2.02 = 0.099

要使 enc_act = 0.099：
  gm(V-v_th)² = 0.099
  0.5(V-0.10)² = 0.099 → V = 0.10 + √0.198 = 0.545

要使 enc_V_ss = 0.545：
  enc_input × r_leak = 0.545
  enc_input = 0.545 / 5.0 = 0.109

要在 relay=0.1 时得到 enc_input=0.109：
  EXTRA_AXIS_GAIN = 0.109 / 0.1 = 1.09 ≈ 1.0
```

**EXTRA_AXIS_GAIN = 1.0** 能让 relay=0.1 就激活 col。

**T2.2 回归测试影响分析**（dt=1.0，quiet relay≈1.0）：
```
enc_input = 1.0 × 1.0 = 1.0
enc V_ss = 1.0 × 5.0 = 5.0 → 完全饱和

T2.2 检查: quiet_encoding_EMA < 0.5
如果 enc 饱和 → T2.2 失败！
```

**结论：EXTRA_AXIS_GAIN = 1.0 会打破 T2.2。** 需要兼顾。

### 方案 B：使 EXTRA_AXIS_GAIN dt-aware（推荐）

在 hebbian.py 第 701 行，EXTRA_AXIS_GAIN 是硬编码常量。可改为 dt 感知：

```python
# 旧
EXTRA_AXIS_GAIN = 0.04

# 新：dt 修正，保持 dt=1.0 时行为一致
EXTRA_AXIS_GAIN = 0.04 / max(dt, 0.001)  # dt=1.0→0.04, dt=0.001→40
```

**分析**：
- dt=1.0 (回归)：EXTRA_AXIS_GAIN=0.04，relay_quiet=1.0 → enc V=0.20 → quiet ✓
- dt=0.001 (Phase5)：EXTRA_AXIS_GAIN=40，relay(1k步)=0.040 → enc V=0.040×40×5=8.0 → 饱和

enc 饱和（V=8.0）时的 enc_act：
```
enc_act = gm(V-v_th)² = 0.5×(8.0-0.10)² = 31.2 → 被 sigmoid/clamp 截断到 1.0
```

**问题**：enc 和 col 都会饱和。需要 col v_peak 也相应调整。

**方案 B-2**（更保守）：
```python
EXTRA_AXIS_GAIN = 0.04 * (1.0 / max(dt, 0.001)) ** 0.5  # 平方根修正
# dt=1.0 → 0.04
# dt=0.001 → 0.04 × √1000 = 0.04 × 31.6 = 1.26
```
使 relay=0.1 时：enc_input = 0.1 × 1.26 = 0.126 → enc V = 0.63 → enc_act = 0.5×(0.53)² = 0.14 → col V = 0.14 × 2.02 = 0.28 > 0.20 ✓

T2.2 影响：dt=1.0，relay_quiet=1.0：enc V=1.0×0.04×5=0.20 → 与当前相同 → T2.2 不变 ✓

**方案 B-2 最优：EXTRA_AXIS_GAIN = 0.04 / sqrt(dt) = 0.04 / sqrt(0.001) = 1.265**

### 方案 C：降低 col_therm v_peak（代价最小）

在 hebbian.py 热觉编码 → 列束处降低 col v_peak = 0.20 → 0.08：

- col V当前 = 0.083（已接近 0.08）
- 这样 col 就直接在当前驱动下激活

**风险**：col v_peak 是在前一次修复中提高的（防止 col 无热源也激活）。降回可能导致 T3.2 失败（col_therm < vestibular col），因为低 v_peak 使 col_therm 过于敏感。

### 方案 D：增加 enc→col 束的 synapse_gain（局部热觉增益）

将热觉 enc→col synapse_gain 从 4.0 提高到 9.0（当前 col V=0.083，× 2.4 → 0.20）：

- 不影响 EXTRA_AXIS_GAIN → 不影响 T2.2
- 不影响 col v_peak → 不影响 T3.2
- 计算：enc_act=0.041，当前 gain=4.0 → col V=0.083；需要 gain=4.0×(0.20/0.083)=9.64≈10

**设 synapse_gain=10**：
- enc_act=0.041 → col V = 0.083 × 10/4 = 0.208 ≈ 0.20 → col 刚好激活 ✓
- 回归 T3.2：需要 col_therm EMA=0.5，需要 enc_act 足够高。在回归测试（dt=1.0）：
  - relay_dt1.0 ≈ 2.35（Phase 4 水平）
  - enc V = 2.35 × 0.04 × 5.0 = 0.47 → enc_act = 0.5×(0.47-0.10)²=0.068
  - col V = 0.068 × 10 × 2.02 = 1.37 >> 0.20 → col 激活 ✓

**方案 D 关键验证点**：对 T3.2 的影响（col_therm EMA在回归中会否过高或过低）。

## 九、次要问题：DA_V 持续偏高

### 实测轨迹

| 步数 | DA_V |
|------|------|
| 10k | 0.205（基态合理）|
| 20k | **0.756**（突然跳升）|
| 30k~90k | 0.760~0.811（持续偏高）|

**目标范围**: 基态 V=0.1（tonic 9%），最大 RPE V=0.6（59%）。  
**实测**: V≈0.79 持续超出目标上限 32%。

### 原因分析

10k→20k 之间（body 在 dist=19.77→30.99 漂离热源时）：
- yolk_frac 0.513→0.496（yolk 消耗，饥饿信号增加）
- RPE = 预期奖励 - 实际奖励（向热源方向运动没有带来能量）
- I_inject = RPE × DA_INJECT_SCALE = RPE × 0.1

shadow_to_da 束（w=0.05，7个 shadow 神经元）和 xin_to_da 束（w=0.1）也在贡献注入电流。联合注入使 DA_V 超出设计上限。

### 是否阻塞 Phase 5？

**当前阶段（col_therm 静默）**: 不构成阻塞——没有 LTP 发生，DA 再高也是空转。  
**修复 col_therm 后**: DA 偏高会加速 LTP（ltp 正比于 DA 浓度），可能导致学习过快。

建议：先解决 col_therm 静默问题（方案 D），再观测 DA 行为。若 DA 仍 > 0.7，再考虑将 DA_INJECT_SCALE 从 0.1 减至 0.05。
