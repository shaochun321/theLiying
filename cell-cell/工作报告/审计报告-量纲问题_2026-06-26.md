# 审计报告 B：管线量纲未对齐（D-01 ~ D-04）

**报告日期**: 2026-06-26  
**审计范围**: `cell-cc-other/nexus_v1/`  
**违规依据**: RULES.md §原则1（真实对象约束）/ §原则4（归一化方案）/ §原则5（文献溯源）  
**核心问题**: Phase 4 实验在 `dt=0.001` 下运行，多处参数源于 `dt=1.0` 下的校准，产生系统性 1000× 量级偏差

---

## 背景：dt 缩放问题的根本来源

本系统有两套运行环境：

| 环境 | dt | 使用场景 |
|------|-----|---------|
| 回归测试 | `dt=1.0` | `test_regression.py`, EXP-012 校准 |
| Phase 4 实验 | `dt=0.001` | `exp_phase4_directional.py` |

`Capacitor.inject(current, dt)` 的量纲是：
```
dQ = current × dt
dV = dQ / C = current × dt / C
```

当校准在 `dt=1.0` 完成，参数带入 `dt=0.001` 时，**所有 inject 路径的电压效果缩小 1000 倍**。

这个问题在 KCL 违规排查时已经暴露（`kcl_rel` 在 dt=0.001 下 1000× 大）。同样的 dt 缩放影响了所有直接电流注入的量纲。

## D-01 DEVIATION_MOTOR_GAIN 校准环境 dt=1.0，实跑 dt=0.001（P0）

### 现象

```python
# circuit/variant_adapter.py:909-917
# Phase 1 calibration (EXP-012): 0.05 produced 0.0000045V/step
# ("whispering in a hurricane"). 1.0 → 0.09V/step at deviation=0.19
DEVIATION_MOTOR_GAIN = 1.0
deviation = circ['deviation']
if deviation > 0.1:
    motor_drive = (deviation - 0.1) * DEVIATION_MOTOR_GAIN
    for key, mot in self.motor_neurons.items():
        mot._membrane.inject(motor_drive, dt)
```

### 量纲推导

**EXP-012 标注的预期（dt=1.0）**：
```
deviation = 0.19 → motor_drive = 0.09
dV = 0.09 × dt / C_motor = 0.09 × 1.0 / 0.3 = 0.3 V/step
```
注：注释写的 "0.09V/step" 是数值混淆，实际是 inject 值 0.09，不是 dV。
真实 dV@dt=1.0 = 0.3V/step —— 几乎等于 motor v_peak（0.2），每步都能触发放电。

**Phase 4 实际效果（dt=0.001）**：
```
dV = 0.09 × 0.001 / 0.3 = 0.0003 V/step
```

在 Phase 4 实验的 500k 步中：
- 每步 dV = 0.0003V
- Motor 膜时间常数 τ = C×R = 0.3×5 = 1.5（仿真单位），dt/τ = 0.001/1.5 ≈ 0.00067
- 每步 leak：V × exp(-dt/τ) ≈ V × (1 - 0.00067)
- 稳态 V = inject_rate / leak_rate ≈ 0.0003 / (V_steady × 0.00067)
- 求解：V_steady ≈ 0.0003/0.00067 ≈ 0.45V（若 motor 不放电）

但 v_threshold=0.15，所以膜电压会在远低于 0.45V 时就放电复位。实际效果是 deviation reflex 会微弱地增加 motor 放电率，量级远小于 VitalOscillator 的基础驱动。

**结论**：`DEVIATION_MOTOR_GAIN=1.0` 在 dt=0.001 下并非"完全无效"，但效果只有 dt=1.0 下的 1/1000，已**不足以作为有意义的热趋性驱动力**。

### 修复方案

若保留 deviation reflex，需：
```
DEVIATION_MOTOR_GAIN_DT001 = 1000.0  # = 1.0 × (1.0/0.001)
```
或改为按 dt 归一化：
```python
motor_drive = (deviation - 0.1) * DEVIATION_MOTOR_GAIN / dt
```
后者是正确的单位处理——`inject(current, dt)` 的 current 应以"每秒电流量"为单位，而不是"每步注入量"。

## D-02 LangevinNoise tau=3.0 超出 BIO 参考范围（P1）

### 现象

```python
# components/langevin_noise.py:69
# BIO: hair cell membrane time constant τ_hair ≈ 0.1-1.0 s
# Chosen as 0.5 s to sit within the hair cell integration window.
tau: float = 3.0   # Phase4: extended from 0.5s — macroscopic displacement
```

### 量纲问题

**原始设计（tau=0.5）**：
- BIO 基础：毛细胞膜时间常数 τ_hair ∈ [0.1, 1.0] s
- 推导：`σ* ≈ θ_sys/√2 ≈ 0.07`，`σ_target = σ₀ × √T_bath`，锚定于 Kramers 逃逸率理论
- tau=0.5 位于 τ_hair 范围内 ✅

**Phase 4 修改（tau=3.0）**：
- 变更目的：产生宏观身体位移（"characteristic displacement ∝ τ"）
- 但 tau=3.0 对应的生物对象不是毛细胞膜，而是**身体运动的惰性时间尺度**（locomotion integration time）
- 文献范围：τ_body ∈ [1-10 s]（取决于 OU 过程拟合的身体质量/阻力参数），但这和原来 BIO 标注的 τ_hair 完全是两个不同的生物对象
- 参考文献缺失：Berg & Brown 1972 的 klinokinesis 论文描述的是 E. coli 的 tumble/run，时间尺度 ~1s，与 3.0 接近，但原 docstring 未引用这层含义

### 违规诊断

| 检查项 | 状态 |
|--------|------|
| tau=3.0 在 BIO 标注范围内？ | ❌（超出 0.1-1.0s） |
| BIO 注释更新？ | ❌（仍写 τ_hair，与变更目的不符） |
| 变更有文献推导？ | ❌（仅经验说明"宏观位移 ∝ τ"） |
| RULES.md §原则1 标注？ | ❌（应加 `# UNGROUNDED`） |
| RULES.md §原则5 引用？ | ❌ |

### OU 过程特征位移计算

OU 过程的特征位移（均方根）：
```
⟨x²⟩^(1/2) ∝ σ₀ × √(T_bath × τ)
```
tau=0.5 → 特征位移 ∝ √0.5 ≈ 0.707（相对单位）
tau=3.0 → 特征位移 ∝ √3.0 ≈ 1.732

**位移增加 2.45×**，而不是 1000×（时间不是 6× 而是 6× 对位移贡献仅 √6 ≈ 2.45×）。
Phase 4 实验中 Δy=+2.63（vs 目标>1.0@200k steps），实际 y 方向位移比预期慢，tau 增大有效但不足以单靠 Langevin 达到目标。

### 修复方案

选择一：**明确变更 BIO 标注**：
```python
# BIO: Locomotion-timescale OU noise (C. elegans klinokinesis: Berg & Brown 1972)
# τ_run ≈ 1-10 s for cell-scale organisms; τ=3.0 gives characteristic
# displacement ≈ 1.73 normalized units per correlation interval.
# NORM: τ_norm = 3.0 (SI seconds at dt=0.001 → 3000 steps/correlation)
# REF: Berg & Brown 1972, PNAS 69:1678; Crenshaw 1993 (helix locomotion)
tau: float = 3.0
```

选择二：**拆分**——保留 τ_hair=0.5 用于毛细胞量纲，增加一个独立的 `LocomotionDiffusion` 组件（τ=3.0）对应身体漂移物理。这符合"一个组件对应一个生物对象"的原则。

## D-03 Column `v_peak=0.10` 无 BIO/NORM 推导（P1）

### 现象

```python
# circuit/hebbian.py:105（cell-cc-other 版）
v_peak=0.10,    # Phase4-P0: lowered from 0.25 — coupler compresses enc→col; 0.10 restores firing
```

### 量纲分析

**归一化方案（RULES §原则4）**：
```
V_norm = (V_bio - E_K) / (E_Ca - E_K) = (V_bio + 80) / 130
V_rest = 0.115
```

Column 神经元的类比对象是皮层柱整合神经元（cortical column integrator）。
文献中 Layer 4/5 皮层神经元的动作电位阈值 V_th ≈ -55 mV（Liu et al. 2014）：
```
V_th_norm = (-55 + 80) / 130 = 0.192
```

当前设置 `v_threshold=0.08, v_peak=0.10`：
- `v_threshold=0.08` 对应 `V_bio = 0.08 × 130 - 80 = -69.6 mV`（低于 E_K！异常）
- `v_peak=0.10` 对应 `V_bio = 0.10 × 130 - 80 = -67 mV`（也低于 E_K！）

这说明 Phase 4-P0 的参数**在归一化方案下没有对应有意义的生物膜电位**。E_K=-80mV 以下的"电压"不对应正常的神经元状态。

**原始值（v_peak=0.25）**：
- `V_bio = 0.25 × 130 - 80 = -47.5 mV`（低于但接近 AP 阈值，合理）
- `v_threshold=0.08` 对应 -69.6mV（亚阈值休息，合理——MOSFET 通道阈值不等于 spike 阈值）

v_peak=0.10 的问题在于：Phase 4-P0 降低 v_peak 是为了让 Column 更容易放电，但从归一化角度看，这个值已进入"静息电位以下"的不合理区间。

### STDP 学习影响

Column 的 CalciumRateIntegrator（CRI）：
```python
cri_q_spike=0.02  # 每次放电注入 Ca²⁺
Ca_ss = f × q × R = 15.5 Hz × 0.02 × 1.0 = 0.31
```

f=15.5 Hz 是在 v_peak=0.25 下测量的 Column 放电率。若 v_peak=0.10，Column 更容易放电，f 可能更高，但 v_peak - v_threshold = 0.10 - 0.08 = 0.02（极窄）会使放电极度敏感于输入噪声，可能导致放电率不稳定，Ca_ss 不可预测。

### 修复方案

按 RULES §原则3 流程：
1. 定位：Column DEAD 的根因是 coupler 压缩（enc→col 增益不足）
2. 生物文献：L4/5 皮层整合神经元 V_th ≈ -55mV → V_norm=0.192
3. 数学模型：`V_ss(bias) = bc_current × R_leak = 0.01 × 5 = 0.05`；需 `V_ss < v_peak/3` → v_peak > 0.15
4. 修复：提高 `synapse_gain` 或 `initial_weight` 使 Column 激活，而非降低 v_peak
5. 替代：若确需降低 v_peak，不低于 0.15（V_bio=-60.5mV），标注推导

## D-04 DA Hunger 注入电流量级未校准（P2）

### 现象

```python
# circuit/variant_adapter.py:899-903
_hunger_da = max(0.0, 1.0 * (0.5 - self.energy_store.fill_fraction))
_da_drive = max(_rpe_da, _hunger_da)
if _da_drive > 0:
    for nid, neuron in self.da_neurons.items():
        neuron._membrane.inject(_da_drive, dt)
```

### 量纲核算

DA 神经元配置（需从 `_init_da_circuit` 中读取）：典型 C_da ≈ 0.1 ~ 0.15。
设 C_da=0.1，R_leak=5.0，τ_da = 0.5：

**fill=0.2 时**：`_hunger_da = 0.3`
```
dV_da/step = inject/C × dt = 0.3/0.1 × 0.001 = 0.003 V/step
leak/step = V_da × dt/τ_da = V_da × 0.001/0.5 = 0.002 × V_da
稳态：dV = leak → 0.003 = 0.002 × V_da → V_da_ss = 1.5 V
```
DA 神经元 v_peak 通常设为 0.2~0.35。1.5V >> v_peak，**DA 神经元以最高速率持续放电**。

**fill=0.0 时**：`_hunger_da = 0.5`
```
V_da_ss = 0.5 / (0.1 × 0.002) = 2.5 V（更高）
```

这意味着一旦 fill<0.5，DA 神经元几乎立即进入饱和放电，与早期 RPE burst（`_rpe_da=1.0`）的效果叠加，导致 Phase 4 前 20k 步权重均匀饱和的问题比预期更严重。

### 与 RPE 的量纲对比

RPE 信号（`DADifferentialGate`）：
```python
_rpe_da = self.da_gate.step(fill_fraction, dt)
# da_gate: η_da=7.5, clip_max=5.0
# _rpe = max(0, η_da × Δfill/dt)，clip 到 5.0
```
`_rpe_da` 最大值 = 5.0（显式上限），单位是"填充率变化速率"（/s），注入后：
```
dV_da = 5.0 × 0.001 / 0.1 = 0.05 V/step（峰值）
```

而 `_hunger_da` 的最大值 = 0.5（fill=0），注入后：
```
dV_da = 0.5 × 0.001 / 0.1 = 0.005 V/step（峰值）
```

表面上 RPE 峰值比 Hunger 峰值大 10×，但**稳态行为相反**：RPE 是瞬时响应（几步后 Δfill 趋零），Hunger 是持续的（fill 长期低于 0.5）。两者的量纲体系（Δfill/dt vs fill_fraction）完全不同，`max()` 合并没有物理依据。

### 修复方案

1. 量纲统一：若要合并 RPE 和 Hunger，需先换算到同一单位（例如都转为 DA 释放率 Hz）
2. 幅度校准：θ_hunger=1.0 应推导为 `想要在 fill=0.2 时 DA 稳态在 v_peak 的 X% 激活水平 → I_hunger = v_target × C / (R × dt) × (1 - exp(-dt/τ))` 反推
3. 结构解法（对应 V-01）：用独立的 `HungerDriveNeuron` 进行物理积分，权重通过 STDP 自动校准，不需要手动调节 θ_hunger

## 量纲问题全局影响评估

| 问题 | 在 Phase 4 的实际效果 |
|------|---------------------|
| D-01: deviation 1000× 缩放 | deviation reflex 在 dt=0.001 下近乎无效 |
| D-02: tau=3.0 无 BIO 锚定 | Langevin 位移有效但校准基础错误 |
| D-03: v_peak=0.10 在归一化方案外 | Column 放电极不稳定，CRI 积累不可预测 |
| D-04: Hunger DA 量级过大 | 与 RPE burst 叠加加剧早期权重饱和 |

以上四个量纲问题中，**D-01 + D-04 组合**是 Phase 4 实验失败的直接量纲根因：
- D-04：fill 下降时 Hunger DA 持续在高位 → 所有热束均匀 LTP → 权重饱和
- D-01：deviation reflex 1000× 缩放后无效 → 无纠错机制
