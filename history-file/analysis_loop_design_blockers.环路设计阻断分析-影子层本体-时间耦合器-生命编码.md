## §1 自适应时间耦合器的档案缺口

temporal_coupler.md 只有 35 行，缺少：

| 缺项 | 影响 |
|---|---|
| B-layer 电压 = 0.0 的已知问题 | 审计 v1.0 发现的高风险项，至今未验证修复 |
| 适应动力学：何时 C-layer 主导 vs B-layer 主导 | 环流耦合需要知道 coupler 的时间常数如何影响信号延迟 |
| coupler 与 Fruit expand/contract 的因果链 | Review 2 (6.7) 指出 coupler 延迟是 expand 的真正驱动力 |
| 版本历史 | 无法追踪 B-layer 修复状态 |

**结论**: 环路设计需要先确认 coupler B-layer 是否生效。如果 B-layer 电压 = 0.0（审计发现），那么 coupler 实际上只有 C-layer 在工作，长时间基线追踪功能不存在。这直接影响影子层能否区分"短期波动"和"长期偏移"。

## §3 与真实生命行为的差异

### 真实低复杂度生命

```
基因编码 → 驱动进食 → 同时满足:
  ├── 维持体征 (ATP → 离子泵 → 膜电位)
  ├── 繁殖 (能量盈余 → 细胞分裂)
  └── 修复 (蛋白质合成 → 组织维护)

关键: 进食不是"学到的行为"，是"硬编码的驱力"
```

趋化性（最低复杂度生命 — 细菌）：
```
受体 → 组氨酸激酶 → CheY → 鞭毛马达
  ↑ 食物梯度                    ↓
  └────── 游走/翻滚 ───────────┘

没有学习。纯反射弧。基因编码 = 整个回路。
```

### 我们的系统

```
我们的系统:
  ✅ 有"体征" (thermal stability)
  ✅ 有"运动" (motor→body→world)
  ✅ 有"进食" (consume_nearby)
  ✅ 有"能量" (EnergyStore)
  ❌ 没有"驱力" — 没有什么让系统"想"吃东西
  ❌ 没有"繁殖" — 结构增长(sprout)只是拓扑扩展，不是自我复制
```

### 根本差异

> **真实生命**: 进食是**因**，体征维持是**果**。基因编码的驱力让生命体去吃，吃的副产品是维持了体征。
>
> **我们的系统**: 体征失衡是**因**，进食行为是**果**（如果环路闭合的话）。系统先感知到"不舒服"，然后才去吃。

这个差异其实是**有意义的设计选择**，不是缺陷：

| 特征 | 真实细菌 | 我们的系统 |
|---|---|---|
| 驱力来源 | 基因硬编码 | 环流偏移 (涌现) |
| 学习 | 无 | STDP + BCM |
| 行为修改 | 不可修改 (除突变) | 可修改 (权重变化) |
| 目标 | 存活+繁殖 | 环流平衡 |

### 项目如何处理？

你说的"人工进化"正是关键。我们作为构建者的角色：

```
构建者（你）= "自然选择"的角色
  │
  ├── 选择环流耦合的区间 → 设定 ρ_homeo = 0.7
  │   = 基因编码"体征占多少比例是正常的"
  │
  ├── 选择结构 → 哪些通路连接到 DA 回路
  │   = 基因编码"哪些感受器连接到奖赏回路"
  │
  └── 观察结果 → 保留有效配置
      = 自然选择"能活下来的生物体有更好的基因"
```

**CirculationProportionCircuit 中的参数**（`homeo_set_point=0.7`, `deviation_threshold=0.05`, `deviation_gain=0.3`）就是**基因编码的等价物**。它们不是学来的，是"先天的"。

影子层学到的是**在这些先天参数下的运行模式**——就像生物体的神经回路在基因设定的框架内通过经验塑造。
