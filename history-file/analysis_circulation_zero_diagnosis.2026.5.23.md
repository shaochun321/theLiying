# 诊断结果：环流=0 和影子=0 的根因

## 环流 μ = 0 的原因

### 真实数据

```
bind_yaw_pitch 激活 = 10.0          ← 绑定层在工作！
col_yaw 激活 = 1.73                 ← 列层在工作！
bind→mot 权重 = 0.001               ← 极弱
feedback_trace = 0.004              ← 弱但存在
motor spikes = 3                    ← 有脉冲

flow = 1.73 × 10.0 × 0.001 × 0.004 = 0.00007
MIN_FLOW = 0.001
0.00007 < 0.001 → 被过滤
```

### 根因

**不是 MIN_FLOW 阈值太高。是 `binding_motor_weights` 初始值太低。**

绑定层的激活已经很强（=10.0），但绑定→运动的连接权重只有 0.001。

```python
# variant_adapter.py 中的初始化（约 L130-150）
self._binding_motor_weights[bid][mid] = 0.001  # ← 瓶颈
```

这是预先存在的设计问题：
- 绑定→运动的权重是手动设的常数
- 没有学习机制来增强这个权重
- 0.001 × 10.0 = 0.01 的有效电流太弱

### 两个解决路径

| 路径 | 做法 | 风险 |
|------|------|------|
| A: 降低 MIN_FLOW | 改为 1e-5 | 低：只改测量阈值 |
| B: 提高 bind→mot 权重 | 0.001→0.01 或加学习 | 中：改变动态行为 |

> [!IMPORTANT]
> 路径 A 只让我们"看到"弱环流，不改变物理行为。
> 路径 B 改变物理行为——需要更谨慎。
> 建议先走 A，观察环流模式后再决定 B。

## 影子层 ν = 0 的原因

### 真实数据

```
影子层神经元:
  s_enc_reg_yaw:   act=-0.000000, energy=0.0100  ← 能量耗尽！
  s_enc_irr_yaw:   act=-0.000000, energy=0.0100  
  s_enc_irr_pitch: act=-0.000000, energy=0.6341  ← 还剩一点
  
影子层突触:
  s_enc_to_col_yaw: xin=0.000000, w=0.100000  ← Xin 为零
```

### 根因

1. **影子层大部分神经元能量耗尽**（energy=0.01=最低值）
   → 没有足够能量产生激活
   → 激活为零 → Xin = predicted - actual = 0 - 0 = 0

2. **Xin 为零 → 影子层没有驱动信号**
   → 影子层的输入来自主环路的 Xin tension
   → 如果主环路的 Xin 也很小（权重在稳定变化），影子层就没有动力

3. **实验代码读了错误的 ν**
   → 读的是 `circulation_state.nu`（环流 ρ 向量的变化率）
   → 应该读影子层内部的自由能变化率

### 影子层能量耗尽的原因

参考 modeling_016：
- I²R 产热 = 0.045/步 × gain=10
- energy=5.0 的神经元在 ~111 步就耗尽
- VR base_rate=0.01 的恢复速率不够快
- 结果：大部分影子神经元在 5000 步时已经能量耗尽

> 这是 modeling_016 中已知的问题。
> 当时的修复（energy=5.0, VR=0.01）只够支撑前几千步。
