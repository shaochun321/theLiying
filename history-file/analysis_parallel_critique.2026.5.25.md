## 二、需要批判的部分

### 2.1 "拓扑盲目性"的批评 — **部分误导**

> "ξ 是标量，不提供空间引导。随机连接在 N 维空间中效率极低。"

这个批评从工程效率角度成立，但**与项目理念冲突**。

> [!WARNING]
> 项目的核心假设是：**结构通过试错和选择涌现**，不是通过计算最优方向。

外部分析建议的"协方差引导"本质上是计算 ∂Loss/∂topology——这是**梯度引导的结构搜索**，跟 NAS (Neural Architecture Search) 属于同一类方法。这：

1. **违反 S0**：协方差计算是纯数学操作作用于信号路径（"哪些下游神经元与误差共变最强"需要维护误差-激活协方差矩阵）
2. **违反涌现原则**：预先计算最优连接方向 = 设计复杂度进去，而非让复杂度自组织

**正确的做法**：让萌芽的 bundle **盲目**地生长，然后由 STDP 强化有用的、由修剪移除无用的。这正是生物学中的策略——轴突过度生长 (overshoot) → 突触竞争 → 修剪 (pruning)。效率低，但**涌现的结构不需要设计者知道什么是"好的"**。

**结论**：拓扑盲目性不是缺陷，是**特性**。外部分析在这里引入了优化主义的偏见。

### 2.2 "吸血规则"— **概念混淆**

> "萌芽时新权重必须从同节点其他突触'吸血'（权重重分配）"

这个建议的物理映射有问题。权重 = Memristor 的电导。电导不是守恒量——两个电阻器的电导不需要加起来等于第三个电阻器的电导。这不是能量守恒。

**S1 CHECK 1 分析**：
- 权重 (w) 的类型 = 电导 (G)
- 电导不守恒。守恒的是**电荷**和**能量**。
- 正确的守恒：萌芽消耗**能量**（已在计划中），不消耗"突触质量"。

> [!CAUTION]
> "突触质量守恒"是一个不存在的物理量。外部分析在这里引入了一个非物理的守恒律。

**结论**：只需守恒**能量**，不需要守恒"权重总量"。新 Memristor 的初始电导应该是物理合理的小值（如 1e-4），不是从其他 Memristor "扣除"的。

### 2.3 κ 的命名问题 — **有效但方向偏**

> "度规缩减度（v2.0）与激活变异系数需分别命名，不可混用"

这个批评**有效**。但外部分析建议的 `ν_deg`（退化度）这个命名引入了一个新的符号，而项目已经有 ν（motion potential）。

更好的做法：沿用 κ 作为状态空间收缩度（来自度规 ds²），引入 **κ_act** 作为激活变异系数的显式名称。这保持了与数学框架的一致性。

### 2.4 "分裂时权重等分" — **物理上可疑**

> "父神经元的所有入边权重总和应守恒地分配给两个子神经元"

同 2.2 的问题。这假设"总权重"是守恒量。但在组件实现中：

```
父 neuron 有 10 个入 Memristor，每个 w=0.5
分裂后：子A 有 10 个 Memristor w=0.25，子B 有 10 个 w=0.25
```

这意味着每个 Memristor 的电导**被人为减半**。物理上，如果你把一根导线切成两段并各自接到一个新设备上，每段的电导不会自动减半——它取决于几何。

**正确的做法**（符合 S0）：
- 子 A 和 B **继承**父的 Memristor 配置（电导不变）
- 上游 bundle 分裂为两个，每个连接到一个子神经元
- 总输入电流确实增加了（两个子神经元合计接收 2× 原来的电流）
- 这个"多余"的电流由 **PowerRail** 自然限制（IR drop）
- 如果系统承受不了翻倍的电流，PowerRail 的 V_supply 会下降，自动抑制

这是让**组件的物理特性**解决守恒问题，而非人为强制。

## 四、修正后的实施原则

基于批判分析，对 implementation plan 的修正：

### 修正 1: 萌芽保持盲目

```python
# 不计算协方差引导
# 新芽继承 parent 的 source 和 target（完全相同的连接拓扑）
# STDP 负责决定新芽是活是死
child = sprout(parent_bundle)  # 盲目复制拓扑
# 之后 STDP 自然强化有用连接，修剪清除无用连接
```

### 修正 2: 只守恒能量，不守恒权重

```python
# 萌芽代价 = 能量消耗（governance 可追踪）
for src in parent.sources:
    src.energy -= SPROUT_ENERGY_COST
# 新 Memristor 从极小值开始，STDP 自己决定长到多大
child_config.initial_weight = 1e-4
```

### 修正 3: 分裂时用 PowerRail 替代人为权重等分

```python
# 子 neuron 继承父 neuron 的完整权重
# PowerRail 的 IR drop 自动限制总电流
# 如果两个子 neuron 加起来需要太多电流：
#   I_total = I_a + I_b > I_max
#   V_supply = VDD - I_total * R_supply → V_supply 下降
#   所有下游自动衰减
# 不需要人为干预
```

> [!IMPORTANT]
> **核心原则**：外部分析试图用**工程约束**（守恒律、引导算法）来控制结构生长。但项目的理念是让**组件的物理特性**（PowerRail 的 IR drop、MOSFET 的阈值、Memristor 的 STDP）自然产生约束。这就是"客观实现"（objective realization）在代码层面的对应。
