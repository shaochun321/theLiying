```
s_enc_reg_yaw:   act = -0.0000

```

**所有跨轴 bundle 仍在 w = 0.001**（初始值，未变化）。

## 根因分析

### 信号链路追踪

```
主系统 Xin 张力 ≈ 0.02
       │
       ▼ 注入为 input_current
影子 Neuron.step(input_current=0.02, dt=0.01)
       │
       ▼ 在 neuron.py L258:
       scaled_current = 0.02 / inertia(1.0) = 0.02
       │
       ▼ 在 neuron.py L264-266:
       v_avail = PowerRail.draw(0.02) ≈ 0.02
       injected = 0.02 × v_ratio ≈ 0.02
       │
       ▼ 在 neuron.py L283: (膜电位更新)
       dV = (I_channel - V/R_leak) / C × dt
       dV = (0.02 - 0/5.0) / 10.0 × 0.01
       dV = 0.02 / 10.0 × 0.01
       dV = 0.00002
```

**每步膜电位变化 = 0.00002**

50000 步 × 每 10 步更新一次 = 5000 次影子更新
理论最大 V = 5000 × 0.00002 = **0.1**

但同时有 leak 电流 `V/R_leak`，所以实际稳态电压：

$$V_{\text{ss}} = I \times R_{\text{leak}} = 0.02 \times 5.0 = 0.1$$

**但**：`activation = V_membrane`（neuron.py L307），而 V 的变化被 C=10 极大地平滑了。达到 0.1 稳态需要 τ = C × R = 10 × 5 = **50 时间单位**。

影子层的 dt = 0.01（每 10 主步更新一次），所以到达稳态需要 50/0.01 = **5000 次影子更新** = 50000 主步。

**这就是问题：影子层的 τ 设得太大了。50000 步刚好到达稳态的边缘——几乎没有足够的时间让 STDP 看到有意义的激活。**

### 数学证明

影子 RC 电路：$\tau = C \times R = 10 \times 5 = 50$

主系统 RC 电路：$\tau = 1 \times 5 = 5$

影子层 effective dt = 0.01（SHADOW_K=10 × dt=0.001）

到达 63% 稳态需要 $t = \tau$ = 50

需要影子步数 = $50 / 0.01$ = **5000 步**

需要主系统步数 = $5000 \times 10$ = **50000 步**

**我们刚好跑了 50000 步——影子层才刚开始"活过来"。**

### 对比：如果 C=2（2× main 而非 10×）

$$\tau = 2 \times 5 = 10$$

到达稳态需要 $10 / 0.01 = 1000$ 影子步 = **10000 主步**

**在 10000 步时就能看到有意义的激活→STDP→收缩。**

## 修正方案

| 方案 | 修改 | 效果 |
|------|------|------|
| **A: 降低影子 C** | C: 10→2 | τ: 50→10，10000步即可活跃 |
| **B: 放大 Xin 输入** | input *= gain(10) | 加速充电，不改τ |
| **C: A+B** | C=3, gain=3 | 平衡：适度慢+适度放大 |

> [!IMPORTANT]
> **推荐方案 C**：C=3（慢 3× 而非 10×）+ Xin gain=3
>
> 理由：
> - C=3 保持影子层"慢于主系统"的设计意图
> - gain=3 补偿 Xin 本身太小的问题（主系统的 Xin ~0.02 是归一化后的值，实际预测误差信号应该更大）
> - τ = 3 × 5 = 15, 到达稳态需要 15/0.01 = 1500 影子步 = **15000 主步**
> - 在 50000 步内有 ~3 个时间常数的学习窗口

## 能量耗尽的佐证

```
s_enc_reg_yaw:   E=0.0005  ← 能量几乎耗尽!
s_enc_irr_yaw:   E=0.0005
s_enc_reg_pitch: E=0.0005
s_enc_irr_pitch: E=0.0005
```

yaw 和 pitch 的 encoding 神经元能量极低，但激活 ≈ 0。

这说明**E_remodel 正在消耗能量**（STDP 权重变化很小但非零），但消耗的能量没有产生足够的激活来驱动跨轴收缩。

**能量在"微弱的学习"上被浪费了——STDP 在做极小的权重调整（Δw ~ 10⁻⁸），每次都消耗 κ × |Δw|，积累 50000 步后耗尽了能量。**

## 第二个问题：能量恢复太慢

影子层的 VR base_rate = 0.0005，能量恢复速率 = 0.0005 × dt = 0.0005 × 0.01 = 5×10⁻⁶/步

而 E_remodel 消耗 ~10⁻⁵/步（κ=0.01 × Δw~10⁻³）

**消耗 > 恢复 → 能量单调下降 → 学习最终停止。**

修正：VR base_rate 需要 ≥ E_remodel 消耗率，或者降低 κ。
