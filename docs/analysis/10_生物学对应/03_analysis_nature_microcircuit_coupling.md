# Nature 论文、小环流、耦合 — 在项目中的位置

## 那篇 Nature 论文说了什么

**Park et al. 2026, Nature Neuroscience**
主题：小脑攀爬纤维通过**去抑制环路**实现条件性学习信号

### 用最简单的话说

```
传统认为：
  老师说"你做错了" → 学生改正 → 学习完成

论文发现：
  不是这样。老师一直在说话（攀爬纤维一直在放电）。
  关键是一群"守门人"（MLI1 中间神经元）。
  正常时守门人挡着 → 学习被锁住。
  
  只有当多条攀爬纤维同时放电（同步）时：
    同步 → 激活"去抑制神经元"（MLI2）
         → MLI2 压制守门人（MLI1）
         → 守门人倒下 → 学习窗口打开
         → 浦肯野细胞改变连接权重 → 学到了

  如果攀爬纤维不同步：
    守门人继续站着 → 学习窗口关闭
    → 已经学到的东西被保护
```

### 论文的核心发现

| 发现 | 含义 |
|------|------|
| 攀爬纤维**一直在放电** | 不是"有信号=学习"，而是信号的**同步性**决定是否学习 |
| MLI2 是**去抑制神经元** | 不是直接激活学习，而是通过**压制抑制**来间接开启学习 |
| 同步性是**群体属性** | 单条攀爬纤维里不存在这个信息，只有多条**一起**才有 |
| 阻断 MLI2 → 无法学习 | 证明这不是辅助机制，而是**必要条件** |

## "小环流"是什么

### 你之前说的原话

> "有些影子层的节点可能是一些小环流的直接参与者，或者所谓的潜意识。"
> "以历史切片为小环流的回归基线并存储，然后输入现在的切片，看对应的环流相对基线是会导致元单元特殊组合内的活跃状态变化来判断大小——运动势的变化。"

### 小环流 = 局部的信号循环回路

```
主环流（大循环）：
  编码 → 列 → 运动 → 反馈 → 编码
  ↑ 信号走完整个系统，一圈很长

小环流（局部循环）：
  编码(yaw) → 列(yaw) → 绑定(yaw_pitch) → 反馈 → 编码(yaw)
  ↑ 信号只在几个相关节点之间打转，一圈很短

更小的环流：
  神经元 A → 神经元 B → 神经元 A
  ↑ 两个神经元之间的直接回响
```

### 在代码中的位置

小环流就是 [circulation.py](file:///d:/cell-cc/nexus_v1/circuit/circulation.py) 中检测到的**每条 CirculationPath**：

```python
# 每条路径就是一个小环流：
path = "yaw → bind_yaw_pitch → move_x"   # 小环流 1
path = "pitch → bind_yaw_pitch → move_y"  # 小环流 2
path = "oto_x → bind_oto_x_y → move_z"   # 小环流 3
```

- **P 路径** = 最强的小环流（主路径）
- **R 路径** = 第二强的小环流（备选路径）
- **其他路径** = 更弱的小环流（背景）

### 和 Nature 论文的关系

```
Nature 论文中：
  多条攀爬纤维 ← 每条都是一个"小信号环"
  它们的同步性 = 群体属性 = 不存在于任何单条中

项目中：
  多条小环流 ← 每条都是一个局部循环路径
  它们的同步性 = 绑定层检测的共激活 = 不存在于任何单轴中
```

**绑定层就是 MLI2（去抑制神经元）** ——它检测多个轴是否同时活跃（= 同步），
当同步发生时，它的输出信号可以触发学习窗口打开。

## "耦合"是什么

### 三种耦合

| 耦合类型 | 中文 | 在哪里 |
|---------|------|--------|
| **信号-电路耦合** | 外部信号进入内部电路 | CPG 门控（50% 窗口开关） |
| **小环流间耦合** | 不同轴的环流互相影响 | 影子层跨轴收缩 + 绑定层共激活 |
| **Xin 跨层耦合** | 预测残差从底层流向高层 | 递归 Xin 流（快波动→慢累积） |

### 1. 信号-电路耦合（论文3 §5.4）

```
外部信号 → 介质传播（有延迟） → CPG 门控（有窗口）→ 内部神经元

耦合 = 外部信号必须在 CPG 窗口打开时到达才能进入
     = 信号的到达时间和内部节律必须"对上"
```

### 2. 小环流间耦合（你最关心的）

```
小环流 A（yaw）:   编码→列→运动→反馈→编码     流量 = 0.3
小环流 B（pitch）:  编码→列→运动→反馈→编码     流量 = 0.2

耦合前：A 和 B 独立运行，互不影响

耦合后（通过绑定层）：
  绑定(yaw_pitch) 检测到 A 和 B 同时活跃
  → 绑定节点激活
  → 创建跨轴连接
  → A 的活动影响 B，B 的活动影响 A
  → 两个小环流"锁相"（同步振荡）
```

这在物理学中叫 **Kuramoto 耦合**——多个振荡器通过弱连接逐渐同步。
项目中每个轴有一个 Kuramoto 振荡器（oscillator），它们通过绑定层耦合。

### 3. Xin 跨层耦合（Nature 论文中的第三因子）

```
底层 Xin（突触级预测误差）
  │ 独立扰动 → 被 √N 稀释 → 上层看不到
  │ 相关扰动（= 同步！）→ 不被稀释 → 上层看到了
  ↓
上层 Xin（环路级残差）
  │ 如果持续存在（底层无法消解）
  │ → 迫使系统在更长的时间窗口中处理
  ↓
最上层 → 触发结构改变（结晶/成熟升级）
```

**这就是耦合的全部含义：不同层级、不同环流之间的信息如何相互影响。**

## 总结

| 概念 | 中文大白话 | 在项目中的位置 |
|------|-----------|--------------|
| **Nature 论文** | 多个信号源的同步性决定是否学习 | 绑定层 = 同步检测器 |
| **小环流** | 局部的信号循环回路 | circulation.py 中的每条 CirculationPath |
| **耦合** | 不同环流/层级之间的互相影响 | 绑定层（轴间）+ Xin（层间）+ CPG门控（内外） |
| **第三因子** | 触发学习窗口打开的信号 | 绑定层的同步输出（目前尚未连接到 STDP） |
| **分化的门控** | 每个区域有自己的学习门，参数不同 | 母本不动 + 参数分化（v2→v6 验证通过） |

> - **同步门控 → 结晶率从 0% 到 100%** ✅
