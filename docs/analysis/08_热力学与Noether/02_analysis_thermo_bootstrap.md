# 热力学自举 — 数据裁决

## 核心发现

### 1. T_stat 收敛到**负值** (-0.072)

```
∂S_config/∂E < 0  →  T_stat < 0
```

物理含义：**能量增加时，构型熵减少。**

这不是错误——这描述了系统的本质：
- 能量输入 → 驱动学习 → 权重稀疏化 → S_weights ↓
- 信号输入 → 激活集中 → S_activations ↓
- **系统是信息积累器**: 能量被转化为结构（低熵），不是热（高熵）

### 2. 系统处于弱非平衡态

| 指标 | 值 | 含义 |
|------|-----|------|
| 负温度比例 | 29.7% | 不是严格平衡态 |
| T_stat std | 0.34 → 0.15 | 在收敛，但波动大 |
| Q/\|ΔE\| | 0.00 | 散热极小，做功主导 |
| S_config 趋势 | 5.25 → 5.16 | 在积累信息 |

### 3. Landauer bound 被"违反"

P1 用 T_stat 重测：32/32 全部"违反"。

**但这不是真的违反** — Landauer bound 要求 T > 0。
当 T < 0 时，bound 的方向反转：系统可以在"低于零度"的状态下做信息擦除。

> [!IMPORTANT]
> 负温度在统计力学中是合法的（如核自旋系统、激光反转布居）。
> 它意味着：高能态的占据数 > 低能态。
> 在本系统中：高能量状态（active neurons）的构型数 < 低能量状态 → 反转。

### 4. `heat_output` 每 tick 被清零

```python
n.heat_output = 0.0  # maintain() L2205
```

这意味着 Q 采集不到正确的累积热——它每 tick 重置。
之前探针测到的 Q > 0 是 activate() 写入后、maintain() 重置前的瞬时值。

## 系统的真实热力学图像

```
                信号输入
                   │
                   ▼
         ┌─────────────────┐
         │   能量 E         │
         │   (neuron energy) │
         │                   │ ← 代谢供能 (feed)
         │   → 驱动学习      │
         │   → 权重稀疏化    │
         │   → S_config ↓   │
         └────────┬──────────┘
                  │
                  ▼
         信息结构 (低熵权重分布)
         
    不是: E → 热 → 耗散 (经典热机)
    而是: E → 信息 → 结构 (Maxwell 妖)
```

**系统不是热机（把热变成功），是 Maxwell 妖（把能量变成信息）。**

## 对 Jaynes-Landauer bridge 的影响

经典 Landauer: "擦除 1 bit ≥ kT ln2 热耗散"
本系统: "不是在擦除信息，是在**积累**信息" (S ↓)

bridge 的正确形式应该是：

```
信息积累 ΔI = S(t) - S(t+1) > 0
所需最小功 W ≥ kT ln2 × ΔI          (Szilard 引擎)
实际做功 W_actual = ΔE_consumed       (代谢消耗)
效率 η = kT ln2 × ΔI / W_actual       (信息热力学效率)
```
