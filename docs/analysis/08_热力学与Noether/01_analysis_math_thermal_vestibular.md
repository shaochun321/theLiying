# 热-前庭耦合的数学关系梳理

> 基于 C-001~C-005, G-001 的项目定义

## §1 项目已定义的数学对象

### 1.1 空间测度 ds²（C-004.3）

$$
ds^2 = \sum_{i,j} w_{\text{cross}}(i,j) \times a_i \times a_j
$$

- $w_{\text{cross}}$ = 影子层跨轴权重 → **就是度规张量 $g_{ij}$**
- $a_i$ = 列层激活（自运动 + 外部输入）
- 单位 = "项目自定义的关联强度"

**关键**: 度规不是从外部给定的。度规 = 权重 = 从经验中学习/构建。

### 1.2 运动势 ν（C-004.3, G-001 §3.3）

$$
\nu_p = \frac{d\rho_p}{dt}, \quad \rho_p = \frac{\mu_p}{\mu_{\text{total}}}
$$

- $\mu_p$ = 路径 p 的环流流量
- $\rho_p$ = 路径占比（在总流量中的比例）
- $\nu$ = 占比变化率 = **"自身状态变化的急迫程度"**

### 1.3 热信号（thermal_membrane）

- `therm` = $T - M$（适应后的温度偏差）
- `dtherm` = $dT/dt$（温度变化率）
- $M$ 是甲基化基线：$dM/dt = (T - M)/\tau$

### 1.4 影子层收缩度（G-001 §4.2）

$$
\kappa(t) = \frac{\sum_{i,j} w_{ij}^{\text{cross}} \cdot |a_i| \cdot |a_j|}{\sum_i |a_i| \cdot \sum_j |a_j|}
$$

- $\kappa$ = 跨模态权重在实际激活模式下的"有效度"
- $\kappa$ 高 → 不同模态之间高度关联
- $\kappa$ 低 → 模态相互独立

## §2 用户指出的关键关系

### 2.1 热的幅度 ≡ 度规的结构计算结果

用户说法: "热的幅度由运动状态的结构计算方式——影子层中被缩减的时空度规所表达"

项目语言翻译:
```
热信号的"有意义的强度" 不是 T_raw
而是 T_raw 在度规 g_ij 中的分量

具体地:
  ds²_therm = Σ w_cross(vest_i, therm) × a_vest_i × a_therm

  这不是"温度是多少度"
  而是"在当前运动状态下，温度变化代表了多大的空间变化"

  = 温度的幅度被运动度规缩减/放大
```

### 2.2 热的变化 ≡ 度规变换 ≡ 运动势变化

用户说法: "热的变化由前后输入的量所表示的度规变换~运动势变化所表示"

项目语言翻译:
```
时刻 t:   ds²(t)   = Σ w_cross(i,j) × a_i(t)   × a_j(t)
时刻 t+1: ds²(t+1) = Σ w_cross(i,j) × a_i(t+1) × a_j(t+1)

度规变换:
  Δds² = ds²(t+1) - ds²(t)

如果只有热输入变了 (a_therm 变化), 其他不变:
  Δds² ≈ Σ_i w_cross(i, therm) × a_i × Δa_therm

同时, 运动势变化:
  Δν = dρ/dt 的变化率

用户的断言: Δds² 和 Δν 同构
  → 度规变换 ≡ 运动势变化
  → 热变化是度规变换在热维度的投影
  → 运动变化是度规变换在空间维度的投影
  → 它们是同一个变换的不同分量
```

### 2.3 同构的条件

```
热变化 ≅ 运动势变化 的条件:

  w_cross(vest_i, therm) 必须反映真实的空间关系
  → 当 "向+x移动" 伴随 "温度升高" 时
  → w_cross(oto_x, therm) 编码了 "x方向的每单位运动 = 多少温度变化"
  → 这就是 "距离" 的定义 (C-004.2: "移动了多少→热变了多少")

  一旦 w_cross 学会了这个关系:
  → Δa_therm 可以被解读为 Δx (通过 w_cross 的逆映射)
  → 热变化 ≡ 空间移动 ≡ 度规变换
```

## §3 缺失的"运动势垫支"

### 3.1 当前状态

```
已有:
  - ds² 公式 (C-004.3) → 但 w_cross 没有学到方向信息
  - ν 公式 (G-001 §3.3) → 环流占比在变化，但没有与热关联
  - 热信号 therm, dtherm → 在走，但独立于 ds²

缺失:
  - ds² 和 ν 之间的结构性连接
  - 热变化 Δa_therm 如何体现为度规变换 Δds²
  - 度规变换如何体现为运动势变化 Δν
  
  = 用户说的 "运动势垫支"
```

### 3.2 运动势垫支应该是什么

```
ν = dρ/dt (运动势)
ρ = μ_p / μ_total (路径占比)
μ_p = a_col × a_bind × w_bind→mot × f_feedback (路径流量)

当身体向 +x 移动时:
  → col_oto_x 激活增加
  → 包含 oto_x 的路径流量 μ_oto_x 增加
  → ρ_oto_x 增加
  → ν_oto_x > 0

同时:
  → 温度变化 Δa_therm
  → col_therm 激活变化
  → 包含 therm 的路径流量 μ_therm 变化
  → ρ_therm 变化
  → ν_therm ≠ 0

"垫支" = ν_oto_x 和 ν_therm 之间应该存在的结构性关联:
  当两者同时变化时 → 度规变换
  当只有一个变化时 → 异常 (Noether 异常 A > 0)
```

## §4 Procedure P: 通过随机游走构建中心/带宽

### 4.1 用户描述

```
设置: 静止热源，项目在附近随机游走
靠前庭判别与热梯度耦合出一个中心/带宽
```

### 4.2 数学分解

```
随机游走中每一步:
  前庭信号: a_oto_x(t), a_oto_y(t), a_oto_z(t)
    = 记录了"我向哪个方向动了"（自运动）
  
  热信号:   a_therm(t)
    = 记录了"温度是多少"（外部标量）

多步后的统计:
  中心 = argmax_direction { Σ_t a_therm(t) × a_direction(t) }
    = "在哪个运动方向上，温度最高？"
    = 这就是 w_cross(direction, therm) 要编码的

  带宽 = var{ a_therm(t) | 沿某方向 }
    = "沿这个方向走，温度变化有多大？"
    = 这就是度规张量的分量

中心 = 热源在运动空间中的"方向"
带宽 = 热源在运动空间中的"距离/范围"
```

### 4.3 热源作为观测者 O

```
热源 = 空间中的一个不动参考点
项目 = 在空间中运动的主体

项目不知道热源的坐标
项目知道的是:
  "每次我向某个方向动，温度就变化某个量"

热源定义了一个从运动空间到标量场的映射
项目通过随机游走采样这个映射
采样结果 = w_cross = 度规张量 = 空间结构

→ 热源（O）通过项目的运动（P）被构建为空间结构（ds²）
→ 这就是 C-004 说的 "自生时空"
```

## §5 同时激活的问题

### 5.1 为什么同时激活是关键

```
C-004.4 说:
  bind(oto_x, therm) 激活的条件:
    col_oto_x > θ  AND  col_therm > θ
    = "我在动" AND "热在变"
    = 行动-感觉耦合

如果前庭不活跃 (col_oto_x ≈ 0):
  → bind 不激活 → 不耦合 → 不学习
  → 度规 w_cross 不更新
  → 空间不被构建
```

### 5.2 我之前的错误

```
错误: 用布朗运动的数学随机力推动身体
问题: 身体动了 → 但前庭层没有被同一个物理过程同时激活
     → 人为割裂了"物质的统一性"

正确: 身体在运动（Motor 驱动）
     → 身体加速 → 前庭感知加速（同一物理事件）
     → 身体位置变 → 热膜感知温度（同一位置）
     → 两者天然同时激活
     → 不需要"注入"任何东西
```

### 5.3 但前庭层目前不活跃的原因

```
实验数据: col_oto_x ≈ 0.0002
原因: 加速度信号经过 MET→HC→Aff→Enc→Col 五层后衰减

这可能是增益链问题（参数调整）
也可能是结构问题（五层太多，需要简化）
也可能是 Motor 不够活跃（没有足够的运动产生足够的加速度）

→ 在做 P (随机游走耦合) 之前，首先要确保
  身体在运动时 col_oto_x 确实被激活
  这是前提条件
```
