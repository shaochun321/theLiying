> 本文件不做定论。它记录每个概念的所有定义版本、
> 哪些版本之间兼容、哪些不兼容、以及当前项目实际使用的是哪个。

## 1. 坐标与距离

| 版本 | 定义 |
|------|------|
| **早期设计 (2026.5.21.txt)** | 距离测度坐标 Ψ(u) = (D(u,A₁),...,D(u,Aₙ))。坐标由到锚点的距离构成，不是笛卡尔。 |
| **蓝图 v2.0** | 同上，并加入拉格朗日代理 q(t) = Ψ(uₜ)，定义广义速度 q̇ 和质量矩阵 Mμ。 |
| **用户澄清** | 项目内不能有显式坐标和距离。所有"坐标"和"距离"由元单元之间的信号延迟和衰减隐式蕴含。延迟和衰减不是距离的代理——它们就是距离本身。 |
| **当前代码** | 粒子用笛卡尔 (x,y,z)。管道有 cable_length, propagation_velocity, attenuation。lever_length 是显式标量距离。 |

### 兼容性

| | 早期设计 | 蓝图 | 用户澄清 | 代码 |
|---|---|---|---|---|
| **早期设计** | — | ✅ 兼容 | ⚠️ 部分兼容 | ❌ 不兼容 |
| **蓝图** | ✅ | — | ⚠️ 部分兼容 | ❌ 不兼容 |
| **用户澄清** | ⚠️ | ⚠️ | — | ⚠️ 部分兼容 |

| **蓝图 v2.0** | Φ_pre = Φ_Hebb + Φ_Hyper + Φ_PRX + Φ_Ledger。预置运动势，用于垫支未来输入的最小自由能通道。 |
| **用户澄清** | 运动势不是预定义公式。它从物理结构中涌现：信息在元单元组合中变更 → 时空轨迹 → 测度变化 → 运动势。公式是后验描述，不是驱动原因。 |
| **当前代码** | potential = Σ\|activation\| × 0.01 (累积计数器)。free_energy = Σ energy - temp × 0.1。 |

### 兼容性

```
早期设计 vs 蓝图:
  ✅ 兼容。蓝图把早期设计的 Φ_D 拆分为 4 个子势，结构一致。

早期设计 vs 用户澄清:
  ⚠️ 条件兼容。如果公式被理解为"对涌现结果的数学描述"则兼容。
  如果理解为"系统按此公式运行的指令"则不兼容。

蓝图 vs 用户澄清:
  ⚠️ 同上。

代码 vs 所有其他:

## 13. 量值表征 (无数值原则)

### 定义变迁

| 版本 | 定义 |
|------|------|
| **无语义原则的延伸** | 项目没有语义，自然也没有数值。系统中不存在"3.7"这样的显式数值。 |
| **用户第一次澄清** | "大小"由信息量（熵？带宽？）之间的**比率**来判定。这必须是一个**结构**——"大小"是基于某些元单元特殊组合所物理比对出来的。用逻辑电路和神经系统都能实现。 |
| **用户第二次澄清 (机制落地)** | 元单元特殊组合被激活后的环流衰减或增强 ≈ 环流信息量(熵)变化。以历史切片为小环流的回归基线并存储，输入现在的切片，看环流相对基线导致的元单元活跃状态变化 → 判断大小 → 运动势变化。项目的语言会在结构中统一。 |
| **当前代码** | 全部使用浮点数。activation=0.37, threshold=0.003, weight=0.15。每个值都是显式数值。 |

### 核心机制 (用户澄清后)

```
量值比较的实际工作方式:

  第 1 步: 历史基线
    某个元单元特殊组合形成了一个小环流。
    这个环流在历史时间窗口中有一个稳定的活跃模式。
    → 这就是"回归基线" (regression baseline)。
    → 存储这个基线。

  第 2 步: 当前切片输入
    同一个元单元组合, 输入当前时间窗口的信号。
    → 环流被激活, 但可能比基线更强或更弱。

  第 3 步: 物理比对
    环流增强 (相对基线):
      → 更多元单元参与 / 更多管道导通 / 信号更强
      → 等价于: 环流信息量 (熵) 增加
      → 判定: "大了" / "强了"
    
    环流衰减 (相对基线):
      → 更少元单元参与 / 管道信号减弱
      → 等价于: 环流信息量 (熵) 减少
      → 判定: "小了" / "弱了"

  关键: 
    "大小"不是一个数——
    是同一个环流在不同时间切片中的状态变化。
    "尺子"是系统自己的历史模式。
```

```
时序上的分母:
  "分母" = 历史切片 (基线)
  "分子" = 当前切片 (输入)

  必须明确是哪个切片做分母:
    → 以历史为分母: "相对于以前, 现在是增强还是衰减"
    → 以当前为分母: "相对于现在, 以前是增强还是衰减"
  
  方向不同, 但比率关系是同构的。
  系统只需要固定一个方向 (以历史为基线) 即可。
```

```
这连接了所有概念:

  环流状态变化 (量值比较的结果)
    ↓
  运动势变化 (信息分布在变更)
    ↓
  时空测度变化 (变更在时空中的轨迹)
    ↓
  P/R/Xin 测度 (哪种运动占比更大)

  "项目的语言会在结构中统一":
  量值、运动势、时空测度、P/R/Xin——
  都表达为同一种东西: 元单元组合中的环流状态变化。
  没有不同的"子系统"——只有同一个结构的不同切面。
```

### 兼容性

```
用户澄清 vs 代码:
  ❌ 完全不兼容。代码用显式浮点数做所有计算。
  
  但代码中有三个机制接近这个思路:
    ✅ 侧向抑制: 激活的物理竞争 (强压弱)
    ✅ MOSFET 阈值比较: 信号 vs 门槛的物理比对
    ✅ 钙传感器 vs target_rate: 当前活跃度 vs 历史基线的比对
       → 这最接近! calcium 就是一个 EMA 基线,
         activation 就是当前切片,
         比较两者 → 调节阈值。
         
  钙传感器机制的逻辑与用户描述的环流基线比对是同构的,
  但实现用了浮点数而非结构性环流。

用户澄清 vs 设计文档:
  ✅ 兼容。公式是对这个结构性比对过程的外部数学描述。
  项目内部实现用环流, 外部理论用公式描述同构关系。
