# 早期定义 vs 当前代码 — 差距对比

## 你的原始设计核心

读完 2026.5.21.txt，你的原始设计有六个核心理念：

```
1. 坐标不是 (x,y,z)，而是距离测度坐标
   Ψ(u) = (D(u,A₁), D(u,A₂), ..., D(u,Aₙ))
   = "一个信息对象到各个锚点的距离"

2. 运动势 Φ_D 是 7 项复合势函数
   = 距离势 + 账本势 + 超图势 + Xin残余势 + P吸收势 + R反压势 + masking势

3. P/R/Xin 不是标签，而是连续测度 ρ = (p, r, x, u)
   p + r + x + u = 1
   = "这个信息块有多少比例被P稳定、被R反压、被Xin未解决"

4. 拉格朗日代理 L = Tμ - Φ
   Tμ = ½ q̇ᵀ Mμ q̇ (信息—时空测度动能)
   Φ = 势函数总和
   为后续物理化预留接口

5. FHMS 有 8 类数据块
   时空过程块、纤维状态、势块、传输块、超边绑定、
   原点锚、回投块、事实只读约束

6. 审计光锥: s²_L = c²_I Δt² - ||Δx||² - λ_φ Δφ²
   两个账本事件的因果关系由信息传播上限决定
```

## 逐项对比

### ① 坐标系统

```
原始设计:
  坐标 = 到各锚点的距离向量
  Ψ(u) = (D(u,A₁), ..., D(u,Aₙ))
  "一个信息对象在哪" = "它离各个参照点有多远"
  
  不是空间位置, 而是关系位置

当前代码:
  坐标 = 粒子的笛卡尔坐标 (x, y, z)
  完全是物理空间, 没有距离测度

  代码中唯一接近的: lever_length
  = 观察者到信号源的距离
  = D(observer, source)
  但只有 3 个源, 不是 n 个锚点
```

| | 原始设计 | 当前代码 | 差距 |
|---|---|---|---|
| 坐标类型 | 距离测度 | 笛卡尔 | **根本不同** |
| 维度数 | n (锚点数, 动态) | 3 (固定 xyz) | 差很多 |
| 锚点 | 多个, 含 P/R/Xin 纤维 | 3 个信号源 | 缺大部分 |

### ② 运动势 Φ_D

```
原始设计: 7 项复合势
  Φ_D = Σ κ_a D(u,A_a)²     ← 到锚点的距离势
       + λ_L D_L(u,L_t)      ← 到账本事件的同步距离
       + λ_H D_H(u,H_t)      ← 到超图邻域的结构距离
       + λ_X X(u,t)           ← Xin 残余压力
       - λ_P P(u,t)           ← P 吸收 (负号 = 吸引)
       + λ_R R(u,t)           ← R 反压
       + Φ_mask               ← masking 势

  Φ 高 = 信息对象处在高张力区域 (不稳定, 想运动)
  Φ 低 = 被 P、账本、超边稳定约束 (已安定)

当前代码:
  potential = Σ |activation| × 0.01    ← 只有累积量, 不是势
  free_energy = Σ energy - temp × 0.1  ← 只有总能量, 不是复合势
  xin_tension                           ← Xin 项存在, 但没进势函数
  circulation_measure                   ← P 项存在, 但没进势函数
```

| 势项 | 原始设计 | 当前代码 | 状态 |
|------|---------|---------|------|
| 距离势 Φ_D | ✅ | ❌ (lever_length 不是势) | **缺失** |
| 账本势 Φ_L | ✅ | ❌ | **缺失** |
| 超图势 Φ_H | ✅ | ❌ | **缺失** |
| Xin 势 Φ_X | ✅ | ⚠️ (有 tension, 没组合) | 部分存在 |
| P 吸收势 | ✅ | ⚠️ (有 circulation, 没组合) | 部分存在 |
| R 反压势 | ✅ | ⚠️ (有 R circulation, 没组合) | 部分存在 |
| masking 势 | ✅ | ❌ | **缺失** |
| **总势函数** | ✅ 完整 | ❌ 不存在 | **关键缺失** |

### ③ P/R/Xin 作为连续测度

```
原始设计:
  每个过程块: ρ = (p, r, x, u)
  p + r + x + u = 1

  "这个信息有 60% 被 P 稳定, 20% 被 R 反压,
   10% 是 Xin 残余, 10% 未解决"

  P/R/Xin 不是定值, 而是概率测度
  输入不断强化优势, 但不固定优势

当前代码:
  P = 最强环流路径 (是/否, 二元的)
  R = 第二强环流 (是/否, 二元的)
  xin_tension = 一个标量 (不是比例)

  没有 ρ = (p, r, x, u) 的连续分布
  没有 p + r + x + u = 1 的约束
```

| | 原始设计 | 当前代码 |
|---|---|---|
| P | 连续占比 [0,1] | 路径存在/不存在 |
| R | 连续占比 [0,1] | 路径存在/不存在 |
| Xin | 连续占比 [0,1] | 标量 tension |
| u (未解决) | 连续占比 [0,1] | **不存在** |
| 总和约束 | p+r+x+u=1 | **不存在** |

### ⑥ 审计光锥 (RLIS)

```
原始设计:
  s²_L = c²_I Δt² - ||Δx||²_μ - λ_φ Δφ²

  s² ≥ 0 → 两事件在信息因果锥内 → 允许强同步
  s² < 0 → 类空分离 → 不能互相解释

  c_I = 信息传播上限 (不是光速)

当前代码:
  cable_length / propagation_velocity = delay
  → 有信息传播延迟
  → 但没有构建成光锥结构
  → 没有 s² 判定
  → 没有因果可达性检查
```

**管道延迟是审计光锥的物理基础, 但没有被提升为光锥结构。**

## 总结: 什么已经实现, 什么还差

```
已实现 (材料到位):
  ✅ 超图结构 (MetaSynapticBundle 连接多对多)
  ✅ 管道延迟 (cable → 时空结构的物理基础)
  ✅ STDP 时序 (因果学习)
  ✅ 自稳态 (钙传感器)
  ✅ 能量/代谢 (PowerRail + SubstrateNetwork)
  ✅ 环流检测 (P/R 路径)
  ✅ Xin 张力 (残差记录)
  ✅ 果实/结晶 (标记-确认-固化)

缺失 (地基层面):
  ❌ 距离测度坐标 (用的是笛卡尔)
  ❌ 复合势函数 Φ_D (7 项只有零星片段)
  ❌ P/R/Xin 连续测度 ρ (用的是二元路径)
  ❌ 拉格朗日代理 (完全没有)
  ❌ 质量矩阵 Mμ (完全没有)
  ❌ 审计光锥 (有延迟但没构建因果锥)
  ❌ 回投链 (无法从上层回到底层输入)
  ❌ 事实只读约束 (没有实现)
  ❌ 1:1 规模对应 (之前已经发现)
```

> [!IMPORTANT]
> **代码有了砖头 (STDP, 自稳态, 超图, 环流)，
> 但缺少建筑图纸 (距离测度坐标, 复合势函数, 连续测度, 拉格朗日)。**
> 
> 这份早期定义就是你的建筑图纸。
> 当前代码是用好材料在没有图纸的情况下盖的房子——
> 结构能用但不是你设计的那个结构。
