# 这一系列实验到底在做什么 — 纯白话版

## 要解决的问题

你的系统有一个老毛病：**学了就忘**。

具体来说：
- 学习阶段：权重变了，学到了东西 ✅
- 巩固阶段（不学习了）：权重继续变 → 学的东西被冲掉 ❌
- 压力阶段（来了相反信号）：学的东西被直接覆盖 ❌

**根本原因：系统没有一个"开关"来控制什么时候允许学习。**
它一直在学，不管输入是有意义的还是噪声。

---

## "门控"是什么

想象一个教室：

```
没有门控:
  老师在讲课 → 学生在学习
  下课了 → 但教室门没关 → 路过的人在聊天
  → 学生把路人的聊天也"学"进去了
  → 之前学的东西被覆盖了

有门控:
  老师在讲课 → 门开 → 学生在学习
  下课了 → 门关 → 路人的聊天进不来
  → 之前学的东西保住了
```

**门控 = 一个阀门，决定"现在该不该学"。**

开门的条件：
- 信号有意义（不是噪声）
- 信号之间有规律（同步性）
- 这两个条件同时满足

---

## 为什么不能用一个门

这是你提出的核心问题。

想象一栋学校大楼有 5 层：

```
1楼 (encoding/感知层):   教室，直接面对外面
2楼 (column/巩固层):     图书馆，安静学习
3楼 (inter_layer/连接层): 走廊，连接各教室
4楼 (motor/运动层):       实验室，动手操作
5楼 (sediment/沉淀层):    档案室，长期保存
```

**如果用同一把锁、同一个开关控制所有门：**

- 1 楼需要频繁开关（信号来得快去得快）
- 2 楼需要慢开慢关（巩固需要时间确认）
- 4 楼需要非常谨慎（动作做错了不能撤回）
- 5 楼需要一直开着（微弱信号需要长时间积累）

用同一个开关 → 要么 1 楼太慢，要么 5 楼太快 → 总有楼层出问题。

**所以：每层楼需要自己的门，但门的构造一样（同一个母本），只是参数不同。**

---

## 参数是什么意思

每扇门有 3 个可调的旋钮：

```
旋钮 1 — τ (反应速度):
  τ=3:   门反应很快，信号一来就开，一走就关
  τ=50:  门反应很慢，要观察很久才决定开不开

旋钮 2 — θ (灵敏度):
  θ=0.05: 门很灵敏，一点风吹草动就开
  θ=0.35: 门很迟钝，要很强的信号才开

旋钮 3 — cw (看重对比度还是同步性):
  cw=0.9: 几乎只看"信号之间的差异大不大"
  cw=0.3: 更看"信号之间是否一起出现"
```

### 每层楼的旋钮设定

| 楼层 | τ | θ | cw | 白话 |
|------|---|---|-----|------|
| 1楼 感知 | 3 (快) | 0.15 (中) | 0.7 (看差异) | 快速判断：这个输入有没有用？ |
| 2楼 巩固 | 15 (慢) | 0.15 (中) | 0.5 (平衡) | 慢慢确认：这个东西值得记住吗？ |
| 3楼 连接 | 8 (中) | 0.12 (中) | 0.6 (略看差异) | 中速协调：各部分是否对得上？ |
| 4楼 运动 | 2 (极快) | 0.25 (严) | 0.9 (极看差异) | 极度谨慎：不确定就不动 |
| 5楼 档案 | 50 (极慢) | 0.03 (极松) | 0.3 (看同步) | 永远在听：再微弱的信号也不放过 |

## 实验验证了什么

### 实验 1 (v2): 分化有效吗？

**问题**: 只改旋钮，5 层楼的门真的会有不同行为吗？

**结果**: ✅ 是的。
```
巩固阶段（下课时间）:
  1楼: 门完全关闭 (gate=0.00) → 权重冻结 ✅
  2楼: 门慢慢关 (gate=0.35) → 权重还在变 ⚠️
  4楼: 门完全关闭 (gate=0.00) → 权重冻结 ✅  
  5楼: 门大开 (gate=0.93) → 权重在变 — 但这是预期行为 ✅
```

**结论**: 同一种门，不同旋钮 → 不同行为。有效。
**问题**: 2 楼关门太慢 → θ 需要调高。

### 实验 2 (v2 压力测试): 冲突信号来了怎么办？

**问题**: 如果有人故意往教室里喊错误信息，门能挡住吗？

**结果**: ❌ 挡不住。因为错误信息也很"响亮"（高对比度），门以为是有用的信号 → 开了。

**结论**: 门只看"信号强不强"，不看"信号对不对"。需要加一个检测："这个信号跟之前学的是一样的方向吗？"

### 实验 3-6 (v3→v6): 怎么检测"方向对不对"

**想法**: 记住之前学的模式（记忆签名），新信号来了跟记忆比一比。

```
方向一致 → "这是加强学习的信号" → 门开
方向相反 → "这是冲突信号" → 门关！
```

**比较的方法经过了 4 次迭代**:

```
v3: 直接比 → 失败 (因为记忆签名更新太慢)
v4: 用快慢两个记忆比 → 改善 14%
v5: 不看输出、直接比输入 → 改善 14%
v6: 先去掉平均值再比 → 改善 72% ← 这才对
```

**为什么 v6 有效**: 

想象测量温度：
```
学习: 温度 = [高, 高, 高, 低, 低, 低]
冲突: 温度 = [低, 低, 低, 高, 高, 高]

不去均值: 平均都是 "中等" → 看不出区别!
去均值后: 学习 = [+高, +高, -低, -低]
          冲突 = [-低, -低, +高, +高] → 完全相反!
```

**最终结果**: 总压力损伤从 2.5 降到 0.7 (减少 72%)

## 还存在的问题

### 为什么冲突检测还不完美

```
信号是交替的:
  tick 1-6: [高,高,...,低,低,...]
  tick 7-12: [低,低,...,高,高,...] 
  → 反复交替

  这种交替在"平均"后就看不出方向了
  → 检测器只能看到"有信号"，看不出"信号方向"
```

**类比**: 
如果有人在门口反复说"对！错！对！错！"——
你记住的只是"有人在说话"，分不清对错。

**解决方向**: 不要比较"平均模式"，而是比较"每次事件"（事件驱动）。
但这是未来的改进，不阻塞当前工作。

## 一句话总结

```
做了什么: 给系统的每个区域装了一扇可调节的门
为什么:   防止"学了就忘"和"被冲突信号覆盖"
怎么调:   每扇门有 3 个旋钮 (反应速度、灵敏度、关注点)
怎么验:   5 个区域 × 6 个阶段 × 50 个突触 × 1000 步
结果:     巩固保护 100% (4/5 区域冻结)
          压力保护 72% (加入方向检测)
核心规则: 所有门共用一套逻辑，只是旋钮设定不同
```
