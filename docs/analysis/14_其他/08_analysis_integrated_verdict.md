# 串联实验裁决 — 电路元件 × Xin 生命周期 × 同步门控

## 实验设计

用项目的电路元件语言构建 60 个突触节点:

```
每个节点:
  Capacitor (C=2.0)  → Xin 累积器 (电荷 = 残差)
  MOSFET (Vth=0.5)   → 相变阈值 (亚阈值=波动, 超阈值=标记)
  Memristor (w=0.3)  → 突触权重 (可塑电阻)
```

4 个阶段: 学习(150t) → 巩固(200t) → 再学习(150t) → 休息(100t)

3 种条件:
- **A**: 无第三因子 (当前系统)
- **B**: 同步门控 (Park et al.)
- **C**: 同步门控 + 4层递归 Xin 传播

---

## 核心结果

### Fruit 生命周期

| 指标 | A (无第三因子) | B (同步门控) | C (同步+递归) |
|------|-------------|------------|-------------|
| Fruit 创建 | 60 | 60 | 60 |
| **Fruit 捕获** | **0** | **60** | **60** |
| Fruit 消失 | 0 | 0 | 0 |
| **结晶率** | **0%** | **100%** | **100%** |

> [!IMPORTANT]
> **A 模式: 所有 60 个 fruit 都在等待第三因子, 但第三因子永远不来。**
> 它们没有消失只是因为 trace 衰减还没到 0.01 (需要 920 ticks)。
> 如果运行更久, 它们全部会消失。
> 
> **B/C 模式: 学习阶段的同步性 (corr=0.7 > 0.5) 触发了 capture。**
> 所有 60 个 fruit 在创建的同一 tick 被立即捕获 → 100% 结晶。

### 巩固保持

| 指标 | A | B | C |
|------|---|---|---|
| 巩固 ΔS | **-0.025** | **+0.000** | **+0.000** |
| 巩固扰动 | 持续 | **完全冻结** | **完全冻结** |

门控在巩固阶段的效果: **100% 减少扰动** (因为 corr=0.1 < 0.5 → gate ≈ 0.19)。

### 再学习效率

| 指标 | A | B | C |
|------|---|---|---|
| 再学习 ΔS | -0.146 | **-0.218** | **-0.218** |
| 相对效率 | 100% | **149%** | **149%** |

> 结晶后的节点权重更稀疏 → 再学习时有更多信息空间 → 效率提升 49%。

### 递归 Xin (模式 C 独有)

```
Circuit Xin:   0 → 168,000 → 开始衰减
Circuit P:     0 → 0.00376 (持续增长)
结构节点:      0 → 5,337 (跨层累积)
```

递归 Xin 不影响突触级行为 (与 B 相同), 但**在更高层产生了结构**:
- 5,337 个结构节点 (bundle/layer/circuit 级)
- Circuit P 从 0 增长到 0.00376 (学会了消解 Xin)
- 这些高层结构为未来的行为提供了"框架"

## 电路语言的物理解释

```
               Vdd (PowerRail, 能量供给)
                │
              R_supply
                │
  signal ──[Memristor]──┬──[Capacitor_Xin]──┤
  (输入)    (突触权重)  │   (Xin 累积器)    │
                       │                   │
                  [R_leak=5Ω]          [MOSFET]
                       │             (相变阈值 Vth=0.5)
                      GND                 │
                                    ┌─────┴─────┐
                                    │           │
                               超阈值        亚阈值
                               (标记!)       (波动)
                                    │
                              [等待同步]
                              corr > 0.5?
                                    │
                              ┌─────┴─────┐
                              │           │
                            yes          no
                         (capture)    (trace衰减)
                              │           │
                          结晶!         消失
                       (Memristor.w    (Xin_cap
                        永久改变)      charge→0)
```

### MOSFET 作为相变阈值的物理意义

MOSFET 的超阈值/亚阈值特性**自然地**实现了相变:

```
V_gate < Vth:  亚阈值电流 ∝ exp((V-Vth)/(nVT))
               → 指数衰减, 几乎为零
               → Xin 是"波动", 不产生结构

V_gate > Vth:  线性电流 I = gm × (V-Vth)
               → 明确的结构形成
               → Xin "成核" 为 fruit
```

这不是人为设定的阈值——这是 MOSFET 物理自带的!

## 结论

> [!IMPORTANT]
> **串联实验确认:**
> 
> 1. **第三因子 = 同步性** 是 Xin 从"波动"到"结构"的必要条件
> 2. **MOSFET 阈值** 自然实现了相变 (不需要 if/else)
> 3. **同步门控同时解决两个问题**: 防止巩固遗忘 + 触发结构捕获
> 4. **递归 Xin** 在高层产生结构节点 + 训练 P 的消解能力
> 5. 所有机制都可以用项目已有的电路元件表达
> 
> **系统从"只会学不会记"到"学而且记住"，需要的是:**
> **把同步性检测 (已有的 convergence_node) 连接为 fruit capture 的第三因子。**
