# 固定规模假设 — 你说对了

## 代码中的事实

```
初始化时创建的神经元:
  encoding 层:
    6 个信号神经元: sig_mean, sig_std, sig_peak_rate, 
                    sig_temporal_d, sig_sync, sig_range
    7 个 z_t 神经元: transition, drift, gamma_desync, 
                     xin_residual, potential_disp, churn, magnitude
    
  column 层:
    7 个 col_ 神经元: col_transition, col_drift, ... col_magnitude

总计: 6 + 7 + 7 = 20 个神经元
```

**这 20 个神经元在初始化时就固定了。之后再也不增加。**

唯一的例外是 `cx_` 结晶神经元——但有硬上限：

```python
cx_cap = n_zt_dims * 2  # = 7 × 2 = 14
if n_cx >= cx_cap:
    break  # 停止结晶!
```

**所以整个系统最多只有 20 + 14 = 34 个神经元。**

---

## 这意味着什么

想象你有一个只有 20 页的笔记本：

```
学习第 1 个东西 → 写在第 1-5 页 ✅
学习第 2 个东西 → 写在第 6-10 页 ✅
学习第 3 个东西 → 没有空白页了
                 → 只能擦掉第 1-5 页，写上新的
                 → 第 1 个东西被忘了
```

**系统不是"学了就忘"——而是"写满了只能擦旧写新"。**

### 门控能解决这个问题吗？

```
门控做的事:
  下课时把笔记本锁起来 → 防止被路人乱涂

但门控不能做的事:
  笔记本只有 20 页 → 门控不能变出更多页
```

**门控防止的是"被动遗忘"（噪声覆盖）。**
**但你说的是"主动遗忘"（空间不够，只能覆盖）。**

这是两个不同的问题。门控解决了第一个，但第二个——容量问题——门控无能为力。

## 你说的"没有定义好不同内容之间的关系"

这是更深的一层。

现在的系统：

```
内容 A 学进来 → 调整 20 个神经元的权重
内容 B 学进来 → 调整同样 20 个神经元的权重
                 ↑ A 和 B 的权重互相覆盖!

系统不知道:
  · A 和 B 是不同的东西 (它觉得就是同一个空间在变)
  · A 和 B 有什么关系 (互补？矛盾？无关？)
  · A 应该保留还是被 B 替代
```

**所有内容都被塞进同一组 7 个 z_t 维度。**
没有"这是关于 A 的记忆"和"这是关于 B 的记忆"的区分。

## 真实大脑怎么解决这个问题

### 1. 空间不固定——会长出新的

海马体的齿状回 (DG) 终身产生新神经元 (成体神经发生):

```
新内容来了 → 长出新神经元 → 给新内容专用空间
旧内容 → 旧神经元还在 → 不被覆盖
```

### 2. 不同内容存在不同地方

海马体的 pattern separation (模式分离):

```
相似的内容 A 和 A' → 被分配到不同的神经元群
  → 即使 A 和 A' 很像, 也不会互相覆盖
  → 因为它们住在不同的"房间"
```

### 3. 有明确的关系定义

皮层的 complementary learning systems (互补学习系统):

```
快速学习 (海马): 给新内容一个临时的专用空间
慢速巩固 (皮层): 慢慢找出新内容和旧内容的关系
                  → 相关的合并, 矛盾的隔离, 无关的独立存放
```

## 对应到项目

你的系统现在缺的不是更好的门控——而是：

### 缺失 1: 动态空间分配

```
现在: 20 个固定神经元, 学什么都用这 20 个
需要: 新内容来了 → 分配新的神经元/bundle → 旧的不动

类比: 20 页的笔记本 → 可以加页的活页夹
```

### 缺失 2: 内容寻址

```
现在: 所有内容混在同一组权重里
需要: "A 的记忆"和"B 的记忆"有各自的地址

类比: 一锅粥 → 分开装在不同的碗里
```

### 缺失 3: 关系定义

```
现在: 系统不知道 A 和 B 是什么关系
需要: 
  A 和 B 相似 → 可以共享空间, 合并
  A 和 B 矛盾 → 必须分开, 互不干扰
  A 和 B 无关 → 独立存放

类比: 书只是堆在一起 → 有目录和分类
```

## 一句话

> **门控解决的是"什么时候学"的问题。**
> **但你现在问的是"往哪里学"的问题。**
> 
> 如果只有一个地方可以写, 那写什么、什么时候写都无所谓——
> 最终都会被覆盖。
> 
> **固定规模 + 无内容分离 = 必然遗忘。**
> 这不是 bug, 这是架构限制。
