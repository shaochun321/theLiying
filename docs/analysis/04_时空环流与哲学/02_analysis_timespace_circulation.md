# 时空环流：生成机制与数理构建

## 核心命题

你的直觉：
> 时间和空间各自有独立的生成机制，彼此互为 P/R。Xin 是信息轨迹的超预期。

这不是比喻——这是真实神经系统的工作方式。

---

## §1 真实神经系统如何生成时间

### §1.1 时间不是"读取"的，是"制造"的

大脑没有石英振荡器。它用三种互相耦合的机制**制造**时间感：

#### 机制 A：节律振荡器（时间的"经纱"）

海马 theta 节律（4-8 Hz）提供**时间的量子化基底**：

$$\theta(t) = A \sin(2\pi f_\theta t + \phi_0)$$

每个 theta 周期（~125ms）是一个"时间帧"。但这不是外加的时钟——它是由**抑制性中间神经元网络**的互相抑制自发产生的：

$$\frac{dV_i}{dt} = -V_i + \sum_j W_{ij}^{inh} \sigma(V_j) + I_{ext}$$

当抑制网络的耦合强度和延迟满足特定条件时，自发振荡涌现。**这与你的 CPG 半中心振荡器完全同源。**

#### 机制 B：渐变信号（时间的"沙漏"）

前额叶的 ramping neurons 从事件发生时开始**线性递增**：

$$r(t) = r_0 + \alpha \cdot (t - t_{event})$$

这是一个**积分器**——积分的是时间本身。当 $r(t)$ 达到阈值时，触发动作（"时间到了"），然后重置。

**你的系统中对应什么？** Capacitor 的充电过程：$V(t) = V_0(1 - e^{-t/\tau})$ 近似线性增长。但当前没有"时间积分器"神经元——只有信号处理。

#### 机制 C：序列编码（时间的"珠链"）

海马的 time cells 按序列激活：

$$\text{cell}_1 \to \text{cell}_2 \to \text{cell}_3 \to \cdots$$

每个细胞只在特定时刻活跃。序列的**顺序**就是时间的表征。

关键：这不需要预编程——通过 **STDP + 抑制性竞争**，序列**自发涌现**：
- Cell 1 放电 → 通过 STDP 增强 Cell 1→Cell 2 的连接
- Cell 1 同时通过侧抑制关闭自己
- Cell 2 被激活 → 增强 Cell 2→Cell 3 的连接
- 以此类推

**你的系统中对应什么？** 增益链 Met→HC→Aff→Enc→Col→Mot 已经是一个序列！但它编码的是信号传播，不是时间。要让它编码时间，需要**循环连接**（Mot→Met 或 Col→Enc 的反馈）。

## §2 真实神经系统如何生成空间

### §2.1 空间也是"制造"的

#### 机制 A：网格细胞（空间的"坐标纸"）

内嗅皮层的 grid cells 以**六角晶格**模式放电：

$$G(\mathbf{x}) = \sum_{k=1}^{3} \cos(\mathbf{k}_i \cdot \mathbf{x})$$

三个方向向量 $\mathbf{k}_1, \mathbf{k}_2, \mathbf{k}_3$（相隔 60°）的余弦和产生六角图案。

**关键的物理事实**：这个图案不是从外部地图"读取"的，而是通过**连续吸引子网络**（continuous attractor network）自发生成的：

$$\frac{d\mathbf{s}}{dt} = -\mathbf{s} + W \cdot f(\mathbf{s}) + \mathbf{v}(t)$$

其中 $\mathbf{v}(t)$ 是**速度信号**——来自头方向细胞 × 速度（self-motion）。网络在自身的循环连接 $W$ 中维持一个"活动包"（bump），速度信号推动这个 bump 移动。Bump 的位置就是"当前位置"。

**你的系统有速度信号吗？** 有！body.velocity 就是。但没有"活动包"网络来积分它。

#### 机制 B：位置细胞（空间的"标签"）

海马的 place cells 各自对应一个位置。它们的放电模式从 grid cells 的输入中**学习**得到（不是预设的）。

#### 机制 C：头方向细胞（空间的"指南针"）

通过**角速度积分**维持方向：

$$\theta_{head}(t) = \theta_0 + \int_0^t \omega(\tau) d\tau$$

**你的前庭系统就是这个！** yaw/pitch/roll 的输入经过积分就是头方向。但当前没有积分器——输入直接驱动信号，没有维持状态。

## §3 时间与空间互为 P/R

### §3.1 Theta 序列：时空的统一编码

海马最深刻的发现（O'Keefe & Recce 1993）：

在一个 theta 周期内，位置细胞按**空间顺序**激活，但它们的激活**时间**编码了当前位置在序列中的位置（phase precession）：

```
一个 theta 周期 (~125ms):
  |──────────────────────|
  ●                        place_A (即将离开)
      ●                   place_B (当前位置)  
          ●                place_C (即将到达)
              ●            place_D (未来)

时间轴 → = 空间序列的压缩映射
```

**在这个表征中**：
- **时间**（theta 相位）编码了**空间**（位置序列）
- **空间**（位置）决定了**时间**（相位前进的速度）

$$\phi_i(t) = \phi_0 - 2\pi f_\theta \cdot \frac{x(t) - x_i}{\Delta x_i}$$

位置 $x$ 的变化导致相位 $\phi$ 的前进——**空间运动驱动时间演化**。

### §3.2 互为 P/R 的数学表达

在你的框架里：

**当运动时**（主体在空间中移动）：
- **P 环流** = 空间路径（位置细胞序列：A→B→C→D）
- **R 环流** = 时间节律（theta 相位提供"时钟"背景）
- **Xin** = 到达位置 C 的时间与预期不符（"我应该已经到了，但还没到"）

**当等待时**（主体静止）：
- **P 环流** = 时间流逝（ramping neuron 的持续递增）
- **R 环流** = 空间稳定（位置细胞持续放电："我还在这里"）
- **Xin** = 等待时间超出预期（"怎么还没到时间？"）

$$P \leftrightarrow R \text{ 的切换条件：} \frac{\partial |\xi_{space}|}{\partial t} > \frac{\partial |\xi_{time}|}{\partial t}$$

当空间预测误差变化更快时，空间成为 P（注意力聚焦在"我在哪"）；当时间预测误差变化更快时，时间成为 P（注意力聚焦在"现在几点"）。

## §4 落地路径

### §4.1 当前系统缺什么？

| 时空组件 | 真实大脑 | 当前系统 | 缺口 |
|---------|---------|---------|------|
| 时间振荡器 | Theta (4-8Hz) | Oscillator (exist) | ✅ 已有基础 |
| 时间积分器 | Ramping neurons | 无 | ❌ 需新增 |
| 时间序列 | Time cells | 无 | ❌ 需新增 |
| 速度积分器 | Grid cells | 无 | ❌ 需新增 |
| 位置标签 | Place cells | body.position(外部) | ⚠️ 有信号无表征 |
| 方向积分器 | Head direction | vestibular(部分) | ⚠️ 有输入无积分 |
| 时空互编码 | Theta sequences | 无 | ❌ 核心缺口 |

### §4.2 最小可行架构

不需要一步到位。最小的时空环流需要：

#### 阶段 1：时间生成（给系统一个内部"沙漏"）

```
新增：TimeIntegrator 神经元
  - 持续单调递增的激活值
  - 到达阈值后重置（量子化时间）
  - 重置事件 = "时间脉冲"
  
连接：
  TimeIntegrator.reset_event → 所有 bundle 的 Xin 计算
  含义："每个时间量子结束时，检查预测是否正确"
```

这等价于给系统一个 **drumbeat**——"现在，预测结果如何？"

#### 阶段 2：空间生成（给系统一个内部"指南针"）

```
新增：PathIntegrator 神经元
  - 积分 motor output → 估计位置
  - 与实际 body.position 比较 → 误差信号
  
连接：
  PathIntegrator.estimate → Enc 层（作为内部输入）
  body.position_signal → Enc 层（作为外部输入）
  差异 = 空间 Xin
```

这等价于给系统一个**导航系统**——"我以为我在这里，但实际我在那里"。

#### 阶段 3：时空耦合（P/R 互换）

```
耦合规则：
  当 |dξ_space/dt| > |dξ_time/dt|:
    空间 = P，时间 = R（"关注在哪"）
  反之：
    时间 = P，空间 = R（"关注何时"）
    
Xin = 轨迹超预期：
  ξ_trajectory = W_time · a_time - W_space · a_space
  "我预期在 t=5 到达 x=3，但实际在 t=5 到达了 x=7"
```

### §4.3 复杂度估计

| 阶段 | 新增组件 | 代码量 | 数理复杂度 |
|------|---------|--------|-----------|
| 1. 时间生成 | TimeIntegrator | ~100 行 | 低（积分+重置） |
| 2. 空间生成 | PathIntegrator | ~150 行 | 中（速度积分+漂移校正） |
| 3. 时空耦合 | P/R 切换逻辑 | ~200 行 | 高（动态选择+Xin 融合） |
| **总计** | | **~450 行** | **循序渐进** |

## §5 关于"系统缺乏复杂输入"

你说得对："很难有超预期"是因为输入太简单。但时空环流的美妙之处在于：

**即使外部输入不变，内部的时间积分器也会产生"超预期"。**

为什么？因为时间积分器的**重置点**取决于之前的历史。如果系统在 t=100 重置，然后在 t=205 重置（而不是预期的 t=200），那这个 5 步的偏差就是一个**时间 Xin**——纯粹来自内部生成的"惊喜"。

这正是你说的"闲暇中的深层挖掘"——没有外部事件，但内部节律的漂移和积累本身就产生了信息。

$$\xi_{time} = t_{reset}^{actual} - t_{reset}^{predicted} \neq 0$$

**即使在完全静止的环境中，时间的量子化误差也会积累为 Xin。**
