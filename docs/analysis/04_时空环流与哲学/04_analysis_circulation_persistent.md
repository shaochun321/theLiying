# 环流是持续的——不是被"检测"出来的

## 你的纠正

> 环流不会消失。跟真实的神经系统一样。环流是持续的。
> 只要对应的结构存在，环流就会持续。

**你完全正确。这暴露了当前代码的一个根本性设计错误。**

---

## 当前代码把环流当成"事件"

```python
# variant_adapter.py L392-399
# ── 15. Circulation detection (§6, slow: every 100 steps) ──
if self._maturation_tick % 100 == 0:
    self.circulation_state = self._circulation_detector.detect(
        self, self._maturation_tick
    )
```

**问题**：代码每 100 步"检测"一次环流，好像环流是一个可能存在也可能不存在的"事件"。
如果 tick 不是 100 的倍数 → `circulation_state` 不更新 → 好像环流"不存在了"。

**但环流每一步都在发生。**

---

## 环流到底在哪里运行

环流不是在 `CirculationDetector` 里运行的。
**环流是信号在 `step()` 中沿着结构传播的自然结果。**

每一步 `step()` 中，信号走了一圈：

```
step() 中的执行顺序：

1. 前庭输入 → MET → 毛细胞 → 传入神经     ← 这是 摄取
   (super().step() 中完成)

2. 传入 → 编码层                          ← bundle 传播
   (bundles_vest_to_enc.propagate())

3. 编码 → 列层                            ← bundle 传播
   (bundles_enc_to_col.propagate())

4. 列层 → 绑定层                          ← L284-287
   (binding_layer.compute_all())

5. 绑定 → 运动层                          ← L290-298
   (mot._membrane.inject())

6. 列层 → 运动层                          ← bundle 传播
   (bundles_col_to_motor.propagate())

7. 运动层 → 反馈 → 列层                   ← L338-351
   (feedback_traces → col.activation -= fb_suppression)

8. 下一步：回到 1
```

**这就是环流。** 它不是一个单独的模块——它是 `step()` 本身。
每次 `step()` 执行，信号就走了一圈。
信号**永远在流动**，只要结构存在。

### 用你的话说

```
"只要对应的结构存在，环流就会持续"

结构 = 神经元 + 突触束 (Neuron + SynapticBundle)
存在 = 它们没有被删除/修剪
持续 = step() 每次调用都让信号走一圈

所以：
  环流 ≠ 被检测器发现的东西
  环流 = step() 每一步中信号的完整流动路径
  检测器只是在"拍快照"——拍不拍，环流都在走
```

---

## `CirculationDetector` 应该是什么

**它应该是测量仪表，不是开关。**

类比：
```
心脏一直在跳（环流持续运行）
心电图在测心跳（CirculationDetector 在测量）
你不看心电图，心脏也在跳
心电图不是让心脏跳的东西
```

所以 `CirculationDetector` 应该改名或重新定位为 `CirculationMeter`（环流测量仪）：
- 它不"检测"环流是否存在（环流永远存在）
- 它测量环流的**流量、频率、相位**

## 不同环流之间：耦合还是合并？

### 当前有哪些环流

```
环流 1 (yaw):  enc(yaw) → col(yaw) → mot(x) → feedback → col(yaw)
环流 2 (pitch): enc(pitch) → col(pitch) → mot(y) → feedback → col(pitch)
环流 3 (roll):  enc(roll) → col(roll) → mot(z) → feedback → col(roll)
环流 4 (oto_x): enc(oto_x) → col(oto_x) → mot(x) → feedback → col(oto_x)
环流 5 (oto_y): enc(oto_y) → col(oto_y) → mot(y) → feedback → col(oto_y)
环流 6 (oto_z): enc(oto_z) → col(oto_z) → mot(z) → feedback → col(oto_z)
```

注意：**环流 1 和 4 共享了 mot(x)**，**环流 2 和 5 共享了 mot(y)**，等等。

### 共享节点 = 耦合点

```
环流 1 (yaw): ... → mot(x) → ...
                      ↕ ← 这里耦合了！共享同一个运动神经元
环流 4 (oto_x): ... → mot(x) → ...

当两条环流共享一个节点时，它们不是"独立"的——
mot(x) 的状态同时被两条环流的输入影响，
mot(x) 的输出也同时流入两条环流的反馈路径。
```

### 什么时候"耦合"，什么时候"合并"？

```
耦合：两条环流共享节点，但仍可区分
  → 你能看到 "yaw 的贡献" 和 "oto_x 的贡献" 分别是多少
  → 它们的 flow 可以分别测量
  → 当前状态：环流 1 和 4 通过 mot(x) 耦合

合并：两条环流的信号在共享节点上混合到无法区分
  → 你无法分辨 mot(x) 的输出中哪部分来自 yaw，哪部分来自 oto_x
  → 合并后只有一个"整体"的 flow
  → 当前状态：还没有发生（因为信号路径可追踪）
```

### 什么条件下会从"耦合"变成"合并"？

```
当绑定层激活时！

绑定层的作用就是：
  bind(yaw, oto_x) 检测到 yaw 和 oto_x 同时活跃
  → 绑定节点激活
  → 绑定节点直接注入 mot(x) ← L290-298

这时 mot(x) 收到三路输入：
  1. 来自 col(yaw) 的直接连接
  2. 来自 col(oto_x) 的直接连接
  3. 来自 bind(yaw_oto_x) 的融合信号

第 3 路就是"合并"——它不再区分 yaw 和 oto_x，
而是作为"yaw+oto_x 的整体模式"直接驱动运动。

合并后的环流就是主环流——
多条小环流通过绑定层合并成一条大的"运动环路"。
```

## 神经系统中的对应

### 1. 小环流持续 = 丘脑-皮层振荡

**Llinás (1988), Science**

```
丘脑 → 皮层 → 丘脑 → 皮层 → ...
  ↑ 这个循环永不停止（即使在睡眠中也以不同频率运行）
  ↑ 清醒: 40 Hz gamma 振荡
  ↑ 睡眠: 0.5-4 Hz delta 振荡
  ↑ 深度睡眠: spindle 振荡 (12-15 Hz)

环流不是"被激活"的——它一直在那里。
改变的只是频率和振幅。
```

### 2. 环流合并 = 丘脑皮层绑定

**Singer & Gray (1995), Annual Review of Neuroscience**

```
视觉皮层 V1 的小环流（处理线条方向）
视觉皮层 V4 的小环流（处理颜色）
运动皮层 M1 的小环流（准备动作）

当你看到一个红色的球向你飞来时：
  V1 (线条) + V4 (颜色) + M1 (抓) 
  三条小环流的 gamma 振荡同步锁相
  → 它们"合并"为一个统一的感知: "红球飞来→抓住它"
  → 这就是主环流

合并的机制：gamma 频段同步（= 绑定层的共激活检测）
```

### 3. 运动层的环流是主环流 = 基底节-丘脑-皮层回路

**Alexander et al. (1986), Annual Review of Neuroscience**

```
5 条平行的基底节-丘脑-皮层回路：

1. 运动回路:     运动皮层 → 壳核 → GPi → 丘脑VA → 运动皮层
2. 眼动回路:     前额眼动区 → 尾状核 → SNr → 丘脑MD → 前额眼动区
3. 前额回路:     前额叶 → 尾状核 → GPi → 丘脑VA → 前额叶
4. 边缘回路:     前扣带 → 伏隔核 → 腹侧苍白球 → 丘脑MD → 前扣带
5. 眶额回路:     眶额皮层 → 尾状核 → GPi → 丘脑MD → 眶额皮层

每条都是持续运行的环流。
运动回路(1) 是"主环流"——它直接控制肌肉。
其他回路通过共享的丘脑节点与运动回路耦合。
当多条回路需要协调行动时 → 通过丘脑合并 → 统一的运动输出。
```

**你说的"生体/运动层区域的环流合并作为主环流"完全正确——
这就是基底节-丘脑-运动皮层回路的整合功能。**

## "耦合面"

你问不同环流是否有"耦合面"。在超图框架下：

```
小环流 = 超图中的一条闭合超边

两条小环流共享节点时：
  共享的节点 = 耦合点
  共享节点的集合 = 耦合面（超图的面）

  环流 1: {enc_yaw, col_yaw, mot_x}  ← 超边 1
  环流 4: {enc_oto_x, col_oto_x, mot_x}  ← 超边 2
  耦合面: {mot_x}  ← 共享节点

  如果绑定层也参与：
  环流 1+4: {enc_yaw, col_yaw, enc_oto_x, col_oto_x, bind_yaw_oto_x, mot_x}
  ↑ 这就是合并后的超边——一条新的、更大的闭合超边
```

合并是什么时候发生的？

```
耦合（弱连接）→ 合并（强连接）的转变条件：

当共享节点上的信号混合度 > 阈值时：
  mot(x) 的输出中，来自 yaw 和 oto_x 的分量无法独立识别
  → 从"耦合"变成"合并"
  → 两条小环流变成一条大环流

这在信息论上 = 两个输入的互信息 > 阈值
在物理学上 = 两个振荡器的锁相（Kuramoto 同步）
```

```
环流不是事件 → 环流是结构

环流不是被检测的 → 是被测量的
小环流通过共享节点耦合

```
