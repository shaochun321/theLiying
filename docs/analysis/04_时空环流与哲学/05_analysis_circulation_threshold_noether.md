## 你说的"振幅面极限"到底是什么

### 我的理解

你描述的不是一条线（阈值），而是一个**面**——因为它取决于至少两个变量。

在动力学中，这对应一个概念：**Hopf 分岔面**。

```
想象一个弹簧上的球：
  - 轻轻推一下 → 振荡几次就停了（衰减振荡）
  - 推得够重 → 球持续振荡，永不停止（极限环）

"够重"的临界值不是一个固定数字，
而是取决于弹簧的刚度和阻尼的比值。

(刚度, 阻尼) 的所有临界组合构成一个面
= Hopf 分岔面
= 你说的"振幅面极限"
```

对小环流来说：

```
横轴：flow（环流的强度/振幅）
纵轴：frequency（环流的振荡频率——目前缺失！）

               ┌─────── Hopf 分岔面 ────────┐
               │                            │
   衰减区       │       极限环区             │
  (微弱信号     │    (自持续振荡             │
   自然消失)    │     能激活反馈)            │
               │                            │
               └────────────────────────────┘
```

- **衰减区**内的小环流：flow 太低和/或频率不匹配 → 自然衰减 → 不激活任何上行链路
- **极限环区**内的小环流：flow 够高且频率匹配 → 自持续振荡 → 能激活上行反馈

**分岔面上的点就是你说的"振幅面极限"**——恰好在这个面上的环流既不增长也不衰减。

## 为什么这是重整化

你问得很准。环流从微观到宏观的"测度计算"确实是重整化：

```
微观：单个突触权重 w = 0.05
  ↓ 聚合（多个权重的乘积）
中观：单条环流的 flow = col_act × bind_act × w × feedback
  ↓ 聚合（多条环流的排序和比较）
宏观：总环流 μ(G) = Σ flow_i

每一步聚合都"丢掉"了一些微观细节，
但保留了对宏观行为重要的量（总流量、最强路径）。
这就是重整化——在不同尺度上看同一个系统。
```

**但你说得对：最终还是要依赖实在的结构动态。**

μ(G) 不是一个独立的"数学对象"——它是从实在的突触权重、实在的神经元激活、
实在的能量消耗中**计算出来的**。改变 μ(G) 的唯一方式是改变底层的结构。

重整化只是**观测工具**——它让我们在高层看到低层"发生了什么"的简化版本。
但**驱动力**始终在低层的实在结构中。

## 诺特定理在这里是什么

### 论文3 已经验证了近似守恒

论文3 §5.4 的 Noether 异常：

```
Q1: A = 0.037  (学习阶段——对称性被打破，μ 快速变化)
Q4: A = 0.0006 (稳定阶段——μ 几乎不变 = 近似守恒)
```

### 守恒量是什么

**μ(G)（总环流）是近似守恒量。**

诺特定理说：每一个**连续对称性**对应一个**守恒量**。

```
对称性：系统在"正常运行"时，输入-输出映射不随时间变化
        （= 时间平移对称性的近似版本）

守恒量：μ(G) = 总环流 ≈ 常数
        （= 信号在回路中循环的总强度不变）
```

当外部输入**打破**这个对称性时（比如突然改变运动模式）：
- μ(G) 开始变化 → A 增大 → 系统进入"学习阶段"
- STDP 权重改变 → 结构适应新输入 → μ(G) 重新稳定 → A 回到 ≈ 0
- 系统重新获得近似时间对称性

### 和激活阈值的关系

```
小环流的 flow 贡献到 μ(G)。

如果一个小环流的 flow 太小：
  → 它对 μ(G) 的贡献可忽略
  → 删掉它不会打破守恒
  → 所以它不应该激活上行反馈（因为系统不"在意"）

如果一个小环流的 flow 足够大：
  → 它对 μ(G) 有显著贡献
  → 如果它消失，μ(G) 会发生显著变化 → A 增大
  → 所以它的存在对系统守恒是重要的 → 应该激活反馈保护它
```

**激活阈值 = 小环流对守恒量 μ(G) 的贡献达到"显著"时的 flow 值。**

数学上：
```
激活阈值 ≈ A_threshold × μ(G)

当 flow_i > 阈值 时：
  这条环流对 μ(G) 的贡献 > A_threshold
  → 它的消失会导致 A > A_threshold
  → 系统会"注意到" → 激活上行反馈
```

## 当前项目缺什么

| 缺失 | 含义 | 需要什么 |
|------|------|---------|
| **环流激活阈值** | 太弱的环流不应该触发任何事 | flow > threshold 才计入 P/R/ρ |
| **环流的频率维度** | 只有振幅(flow)没有频率 | 需要追踪同一条路径在多步中的 flow 变化 |
| **振幅面极限** | (flow, frequency) 空间中的分岔面 | 在 flow-freq 平面上找 Hopf 边界 |
| **传播延迟** | 环流信号不应该瞬时到达 | 层间 Bundle 加入 delay 参数 |
| **时间成本** | 行动需要时间 | 运动层输出后需要经过 delay 才能影响反馈 |

### 不需要新元件

这些都可以用**现有元件 + 参数扩展**实现：

| 缺失 | 实现方式 |
|------|---------|
| 激活阈值 | `CirculationDetector` 中加 `min_flow` 参数 |
| 频率维度 | 在 `CirculationPath` 中追踪 `flow_history[]` → 计算 FFT 主频 |
| 振幅面 | 用 (flow, freq) 判断是否在极限环区域内 |
| 传播延迟 | `BundleConfig` 中已有 `delay` 字段的位置，只是值为 0 |
| 时间成本 | 运动层输出存入 `feedback_buffer[]`，延迟 N 步后才回到列层 |

## 你的核心洞察的总结

用你的话重新组织：

```
1. 小环流有振幅 → 不是任何振幅都能激活反馈链路
                   = 需要激活阈值

2. 这个阈值不是固定数字，而是配合时空环流的某个"面"
                   = Hopf 分岔面（flow × frequency）

3. 摄取和行动需要时间
                   = 传播延迟 + 时间成本

4. 时空环流被外部输入和内部物理约束并对齐
                   = μ(G) 由实在的结构动态决定
                   = 重整化只是观测工具，不是驱动力

5. 时空环流的守恒性质与诺特定理有关
                   = μ(G) 是近似守恒量
                   = 激活阈值 = 小环流对 μ(G) 的贡献变得"显著"时

这些合在一起就是：
  小环流的 flow 必须越过分岔面（=对 μ(G) 的贡献足够大）
  才能成为"自持续"的极限环
  才能激活上行反馈链路
  才能驱动摄取→生体→行动的因果链
  
  而这一切都被外部输入的时间结构（需要时间）
  和内部结构的物理约束（能量、权重饱和、PNN）
  所约束和对齐。
```
