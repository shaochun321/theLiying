# 去抑制门控 — 参数敏感性裁决

## 核心发现：门控机制极其鲁棒

### 1. I₀ (抑制基线) — 宽容区间

```
I₀ = 0.10 ~ 0.99 全部有效 (保护率 > 65%, 学习保留 > 94%)
```

| I₀ 范围 | 保护率 | 学习保留 | 评价 |
|--------|--------|---------|------|
| 0.10-0.30 | 65-95% | 100-109% | ✅ 最佳区 |
| 0.30-0.60 | 83-95% | 101-104% | ✅ 最佳区 |
| 0.60-0.99 | 67-101% | 94-96% | ✅ 仍有效 |

> **I₀ 不需要精确调参。** 从 0.1 到 0.99 都能工作。
> 这是一个**结构性优势**——由网络拓扑保证，不依赖参数微调。

### 2. 同步性对 (g_learn, g_consol) — 关键约束

```
g_consol < 0.15 是保护的前提
g_learn > 0.6 是学习的前提
```

同步性矩阵中的最佳区域：

```
           g_learn
         0.8    0.9    1.0
g_c=0.0  91%   108%   113%    ← 最强保护
g_c=0.05 78%   110%   122%    ← 仍然很好
g_c=0.10 87%    88%   108%    ← 开始衰减
g_c=0.15 49%    76%   106%    ← 临界线
g_c=0.20 48%    84%   106%    ← 保护不足
```

> **临界边界: g_consol = 0.15**
> 超过这个值，巩固阶段的"背景同步"足以穿透门控，导致保护失效。

### 3. θ_M 时间常数 τ — 几乎无影响

```
τ = 3 到 200 全部有效 (保护率 70-135%)
```

这意味着：**门控的效果不依赖于 BCM 的内部参数。**
这是好消息——门控是一个**外部约束**，它不需要与 BCM 的 θ 动力学协调。

### 4. 生物学参数下的表现

Park et al. 实测值 → 我们的模型参数：

```
I₀ = 0.70  (MLI1 基线抑制 ~70%)
g_learn  = 0.70  (error trial CF 同步)
g_consol = 0.15  (correct trial CF 同步)
```

结果：**保护率 69.6%, 学习保留 96.0% ✅**

> [!IMPORTANT]
> 生物学参数不是最优的——数学最优是 I₀=0.45, g_L=0.9, g_C=0.0。
> 但生物参数**足够有效**，这符合进化的逻辑：
> 自然不需要最优，只需要足够好且鲁棒。

## 对系统架构的含义

### 需要什么来实现同步性检测 g(t)？

在当前系统中，g(t) 可以从以下信号中计算：

```python
# 方案 A: 激活相关性
g = mean_pairwise_correlation(encoding_activations)

# 方案 B: 信号熵
g = 1.0 - S_activations / S_max  # 低熵 = 高同步

# 方案 C: 信号对比度
g = std(signal_values) / mean(signal_values)  # CV
```

**方案 B 最自然** — 我们已经在测 S_activations。
S_activations 低 = 信号集中 = 高同步 → g 高。

### convergence_node 作为 g(t) 检测器

当前的 convergence_node 检测共激活 — 这**就是**同步性检测：

```
convergence_matrix[i][j] += activation[i] × activation[j]  # 共激活
g = max(convergence_matrix.values()) / threshold            # 归一化
```

**不需要新的结构** — convergence_node 已经是 g(t) 的雏形，
只是当前强度太弱 (0.00003)，需要调整增益。
