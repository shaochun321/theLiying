# 运动势与时空测度 — 在项目中如何对应

## 先理清两个概念

```
运动势 (motion potential):
  "一个元单元积累了多少运动能量"
  → 只增不减的累积计数器

时空测度 (spacetime measure):
  "信号在网络中走过了什么样的路径, 以什么样的时间顺序"
  → 空间路径 × 时间顺序 = 时空结构
```

---

## 运动势 (potential)

### 在代码中是什么

每个元单元有一个字段叫 `potential`:

```
每个 tick:
  potential += |activation| × 0.01

  注意: potential 只增不减!
  它记录的是 "这个元单元从出生到现在, 一共处理了多少信号"
```

### 类比

```
想象你在爬山。

activation = 你这一步走了多远 (可正可负, 每步不同)
potential  = 你的里程表 (总共走了多远, 只增不减)

你可能原地转圈 (activation 在正负之间波动)
但里程表一直在走 (potential 一直在增)

走得越多 → potential 越高 → 你越"资深"
```

### 运动势决定了什么

```
potential > 50  → 从 spine 晋升为 column (足够资深)
potential > 500 → 从 column 晋升为 area (非常资深)

这就是为什么叫"运动势":
  它不是"当前的活跃程度" (那是 activation)
  它是"累积的运动历史" → 一种势能
  势能够了, 就能触发结构性的变化 (晋升)
```

### 为什么是"势"而不是"能"

```
势 (potential): 储存的, 等待被使用的
能 (energy):    正在被消耗的

元单元的 potential 不会被消耗掉。
它只是一个记录 → "我有多少资格"
当资格够了, 触发晋升 → 但 potential 不减少

就像你的工作经验:
  你工作了 10 年, 不会因为升职而变成 0 年经验
  经验只增不减, 是一种"势"
```

## 时空测度

这个比运动势复杂得多。它有两个维度:

### 空间维度: 环流测度 μ(G)

```
什么是环流:
  信号从 A 出发, 经过 B, C, D, 最终回到 A
  = 一条闭合路径

  A → B → C → D → A  ← 这是一条环流

系统中可能有很多条环流。
环流测度 μ(G) = 所有环流的总强度。
```

```
怎么计算:
  找到所有闭合路径 (用深度优先搜索)
  每条路径的强度 = 路径上所有管道权重的乘积 × 激活值
  
  μ(G) = 所有路径强度的总和

  P = 最强的那条环流 (主环流)
  R = 第二强的那条环流 (竞争环流)
```

```
类比:
  一座城市的交通。
  
  有很多条路可以从 A 走回 A (环路)。
  每条路有不同的"车流量" (权重 × 激活)。
  
  μ(G) = 全城的总车流量
  P = 最繁忙的那条环路 (主干道)
  R = 第二繁忙的环路 (备选路线)
  
  μ(G) 越大 → 城市越"活跃"
  P 越强 → 系统有一个主导的处理循环
  R 存在 → 有竞争/替代方案
```

### 时间维度: 相位排序

```
找到 P 环流后, 问:
  P 路径上的各个元单元, 它们激活的时间顺序是什么?

例子:
  P 路径 = A → B → C → A

  tick 10: A 达到峰值
  tick 13: B 达到峰值  
  tick 17: C 达到峰值
  tick 21: A 再次达到峰值 → 一个完整周期

  相位排序 = [A, B, C]
  周期 = 约 11 tick
```

```
类比:
  交响乐队。

  空间 = 哪些乐器在演奏 (小提琴, 大提琴, 长笛)
  时间 = 它们依次进入的顺序 (先小提琴, 再大提琴, 最后长笛)
  
  空间 × 时间 = 乐谱
  
  如果每次演奏, 乐器进入的顺序都一样 → 高相干性 (coherence)
  如果每次都不一样 → 低相干性 → 混乱
```

### 时空测度 = 空间路径 × 时间顺序

```
时空环流 = (谁连着谁) × (谁先谁后)

        空间                    时间
  A → B → C → A    ×    A在第0相, B在第3相, C在第7相

  = 完整描述了 "信号以什么路径, 以什么时间节奏, 在网络中循环"
```

### 相干性 (coherence) = 时空测度的质量

```
系统检查: 连续两个时间窗内, 相位排序一样吗?

  窗口 1: 排序 = [A, B, C]
  窗口 2: 排序 = [A, B, C]  → 一致! → 相干性 = 1.0

  窗口 1: 排序 = [A, B, C]
  窗口 2: 排序 = [A, C, B]  → 不一致! → 相干性 < 1.0

高相干性: 系统形成了稳定的时空模式 → "我知道在做什么"
低相干性: 模式不稳定 → "还在摸索"
```

## 运动势 和 时空测度 的关系

```
运动势 → 个体层面的积累
  "这一个元单元有多少经验"

时空测度 → 群体层面的组织
  "一群元单元形成了什么样的协作模式"

两者的连接:
  运动势高的元单元 → 更可能成为 P 环流的节点
  (因为它们更稳定, 更不容易被推翻)
  
  P 环流的形成 → 进一步提高参与者的运动势
  (因为它们更频繁地被激活)
  
  = 正反馈循环: 势 → 环流 → 更多势 → 更强环流
```

## 还有一个: 自由能

```
自由能 F = 总能量 - 温度 × 熵

  总能量 = 所有元单元的 energy 之和
  温度   = 系统的"热度" (活动产生的热)
  熵     = 系统的"混乱度" (简化为常数 0.1)

  F 高 → 系统有余力做结构性工作 (学习, 晋升)
  F 低 → 系统处于代谢紧张状态 → 进入节能模式
```

```
类比:
  你的银行余额。
  
  总能量 = 工资收入
  温度×熵 = 必要开支 (房租、吃饭)
  自由能 = 可支配收入
  
  可支配收入高 → 可以投资学习新技能
  可支配收入低 → 只能维持生存
```

## 管道延迟如何创造时空结构

```
管道不是瞬间传输的。信号要"走"过去:

  delay = cable_length / propagation_velocity

  短管道 (length=1, velocity=1): delay = 1 tick
  长管道 (length=3, velocity=0.5): delay = 6 tick

这意味着:
  A 发出信号, B 在 1 tick 后收到
  A 发出信号, C 在 6 tick 后收到
  
  → B 和 C 的 STDP 时序痕迹不同!
  → B 的因果关系强 (间隔短)
  → C 的因果关系弱 (间隔长)
  
  → 管道长度 自然创造了 时间结构
  → 距离 → 延迟 → 时序 → 因果强度 → 学习差异
  → 这就是 "时空" — 空间距离决定了时间关系
```
