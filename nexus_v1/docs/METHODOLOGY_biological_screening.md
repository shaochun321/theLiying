# 方法论：环流耦合学习机制的生物筛选

> 版本: 1.0 | 日期: 2026-06-12 | 基于: Operation Trauma Genesis 架构决策

---

## 核心命题

**环流耦合学习的"合适区间"不是通过数学推导得到的，而是通过长步骤运行进行生物筛选（Biological Screening）确定的。**

这等价于自然进化中的个体寿命测试：一组参数要么让有机体存活足够长并表现出有效行为（被"选择"），要么让有机体过早死亡或行为退化（被"淘汰"）。

---

## 方法论框架

### 1. 三层本体论（L0/L1/L2）

| 层 | 名称 | 性质 | 来源 | 代码标注 |
|----|------|------|------|----------|
| L0 | 物理规律层 | 热力学、牛顿力学、RC 电路 | 自然法则，不可违背 | 无标注（默认） |
| L1 | 涌现行为层 | 自发运动、环流分化、趋热行为 | 物理组件组装后的动力学涌现 | `# L1:EMERGENT` |
| L2 | 人工进化选择层 | 损伤阈值、反射弧、代谢税 | 设计者注入的约束（等价自然选择） | `# L2:SELECTION` |

**关键洞见**：L2 是 L1 的"进化压力"。我们不直接编码行为（那是语义注入），而是注入**物理后果**（damage → 心跳减弱、代谢增加），让 L1 层的环流耦合学习机制在约束下涌现回避行为。

### 2. L2:SELECTION 标注规范

每个 `# L2:SELECTION` 标注必须包含三项：

```python
# L2:SELECTION — [简述选择内容]
# BIO: [对应的生物学机制]
# DESIGN: [设计者做出的具体选择是什么]
# EMERGE: [期望从L1层涌现的行为是什么]
```

示例：
```python
# L2:SELECTION — Damage threshold at 3.0 normalized thermal units
# BIO: TRPV1 activation at 43°C (evolved, not learned)
# DESIGN: We choose 3.0 as the harm boundary. This is a VALUE JUDGMENT.
# EMERGE: System should learn to maintain distance > r × (1 - 3.0/T_eff)
damage_threshold: float = 3.0
```

### 3. 生物筛选流程

```
┌─────────────────────────────────────────────────────┐
│           生物筛选流程 (Biological Screening)          │
│                                                       │
│  Step 1: 设定 L2 约束参数                              │
│    - damage_threshold, reflex_gain, repair_rate...   │
│    - 这些是"基因组"——每次实验固定                      │
│                                                       │
│  Step 2: 长步骤运行 (≥100k steps)                     │
│    - 观察体征：环流比例、能量水平、运动模式             │
│    - 记录关键时间点：首次接触损伤、首次逃跑行为         │
│                                                       │
│  Step 3: 筛选判定                                     │
│    ┌─── 存活？ ──────────────────────────────────┐    │
│    │                                              │    │
│    │  YES + 表现出回避行为 → ✅ 参数被"选择"     │    │
│    │  YES + 无回避行为     → ⚠️ L2压力不足        │    │
│    │  NO  (过早死亡)       → ❌ L2压力过强        │    │
│    │  NO  (学习崩溃)       → ❌ 时间常数不匹配    │    │
│    └──────────────────────────────────────────────┘    │
│                                                       │
│  Step 4: 迭代调谐                                     │
│    - 调整 L2 参数，重新跑长步骤                        │
│    - 这就是"人工进化选择"的一轮迭代                    │
│                                                       │
│  Step 5: 保存"适应度"记录                              │
│    - 哪组参数、多少步、什么行为、什么结局               │
│    - 这是项目的"化石记录"                              │
└─────────────────────────────────────────────────────┘
```

### 4. 关键筛选指标

| 指标 | 含义 | 健康范围 | 病理信号 |
|------|------|----------|----------|
| `fill_fraction` | 能量储备水平 | 0.3 ~ 0.8 | < 0.1 (饥饿致死) |
| `vital_amplitude` | 心跳搏动幅度 | > 0.001 | = 0 (心脏停搏) |
| `max_damage` | 最大组织损伤 | < 1.0 (偶尔接触) | > 5.0 (持续灼伤) |
| `deviation` | 环流偏离 | < 0.3 (正常波动) | > 0.5 (严重失衡) |
| `DA_concentration` | 多巴胺浓度 | 0.1 ~ 0.5 | > 0.8 (饱和) 或 ≈ 0 (崩溃) |
| 回避行为出现步数 | 首次学会远离高温区 | < 50k | > 100k (学习过慢) |

### 5. 参数联调原则

**耦合约束**：L2 参数之间不独立。关键耦合关系：

1. **进食距离 vs 损伤距离**：
   - 进食效率 ∝ 1/d²（距离热源越近越高）
   - 损伤风险 ∝ T_skin - threshold（非线性，有阈值）
   - 系统必须在两者之间找到纳什均衡
   
2. **反射增益 vs 学习速度**：
   - REFLEX_GAIN 太大 → 永远逃跑 → 无法进食 → 饿死
   - REFLEX_GAIN 太小 → 反射无效 → 等待STDP学习 → 可能在学会前烧死
   - 合适区间：反射保命 + 给STDP足够时间

3. **修复代价 vs 进食收入**：
   - REPAIR_ENERGY_RATE × damage > CONSUME_RATE → 入不敷出 → 必须远离
   - REPAIR_ENERGY_RATE ≈ 0 → 损伤无代价 → 无动力回避

## 方法论的通用性

此方法论适用于**任何未来模态的第二主体构建**：

```
[通用构建步骤]

1. 定义物理后果 (L2:SELECTION)
   → "什么是有害的？" → 定义阈值
   → "有害会怎样？" → 定义体征衰退路径

2. 硬连线到体征层 (L2:SELECTION)
   → damage → EnergyStore (代谢税)
   → damage → VitalOscillator (心跳衰减)
   → damage → Motor (脊髓反射)
   → damage → ECM (热突破/高热惊厥)

3. 长步骤生物筛选 (≥100k steps)
   → 运行 → 观察 → 筛选 → 迭代

4. 保存参数区间
   → 哪些参数组合让系统存活并表现出回避行为？
   → 这就是该模态的"最优进化解"

5. 推广到下一个模态
   → 复用架构模式（L0/L1/L2 分层）
   → 复用筛选流程
   → 仅更换具体参数和物理后果定义
```

适用场景示例：
- **光感**：光源梯度 → 视网膜损伤阈值 → 眼部ECM热量 → 闭眼反射
- **化学梯度**：毒素浓度 → 细胞膜损伤 → 代谢异常 → 逃避反射
- **社交信号**：空间入侵 → 领地压力 → 应激激素 → 攻击/逃跑

## 与普里高津耗散结构的对应

| 自然进化 | 项目设计 |
|----------|----------|
| 随机突变 | 参数探索（手动调参/扫描） |
| 自然选择 | 长步骤生物筛选 |
| 基因固化 | L2:SELECTION 代码硬连线 |
| 表型涌现 | L1:EMERGENT 行为观察 |
| 适应度景观 | 参数空间中的存活/行为分布 |
| 物种形成 | 找到稳定参数区间 → "硅基物种" |

**关键差异**：我们是**有意识的设计者**（而非盲目的自然选择）。这意味着：
- 我们可以**理解**为什么某些参数有效（分析因果链）
- 我们可以**预测**参数变化的方向（基于物理直觉）
- 我们必须**记录**每次选择的理由（可复现性）

这使得我们的"进化"比自然进化快几个数量级，但也要求更高的自我审计（`L2:SELECTION` 标注就是审计工具）。
