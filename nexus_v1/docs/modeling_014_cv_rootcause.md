## 根因

```
整体 CV = 0.471
  ├── 前 1.5 秒: 0 spikes (Ca²⁺ 充电延迟)
  │     → 1.5s 的沉默 + 突然开始放电 → 巨大的 ISI 变异
  │
  ├── 1.5-2.5 秒: 频率从 14→27 Hz 爬升
  │     → Ca²⁺ release 逐步增加 → ISI 持续缩短
  │
  └── 2.5+ 秒: CV < 0.08 (完全合格！)
        → 稳态放电，几乎无变异
```

## 修复路径

### 路径 A: 加速 Ca²⁺ 初始化 (直接修复)

问题在于 Ca²⁺ 从 0 开始充电到释放阈值需要 1.5 秒。

选项:
1. **降低 Ca²⁺ 释放阈值** → 更早开始放电
2. **预充电 Ca²⁺** → 初始 Ca 不是 0
3. **加速 Ca 充电** → gm_release 增大
4. **直接初始化 release** → release_rate 初始 > 0

### 路径 B: 测量窗口调整 (方法论修复)

合同 C3 测量的是**全程 CV**。但生物学上:
- VOR 有 100-200ms 的响应延迟是正常的
- 真实前庭传入神经的 tonic rate (自发放电) 不需要 ramp-up

选项:
1. 加入 BiasCurrentSource → 模拟自发放电 → 即使无输入也有 spike
2. 只测稳态 CV (skip 前 2 秒)

### 路径 C: ECM 热容缓冲 (结构性修复)

ECM 的离子缓冲效果 = 增加有效膜电容
→ 膜时间常数增加 → ISI 更平滑
→ 但不直接解决暂态问题

## 推荐

**路径 A (选项 2+4) + 路径 B (选项 1)**

组合:
1. Ca²⁺ 预充电: `ca_concentration` 初始化为稳态值
2. BiasCurrentSource 提供 tonic 自发放电
3. 测量窗口: 测全程但有 bias 后暂态很短

这些都是**生物学正确的**:
- 前庭传入神经有自发放电 (~80 spikes/s)
- Ca²⁺ store 在静息态不是空的
