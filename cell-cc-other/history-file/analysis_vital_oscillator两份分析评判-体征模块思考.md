## 二、体征模块的思考

你提出的核心概念:

> 环路耦合的最少参与者: **运动—体征—行为(进食)**
> 体征模块 = 来自系统内部的搏动，遵循结构计算与物理结算
> 搏动信息可能来自系统时钟
> 底层构建的必要性 = 统一内外于物理内

### 这个思想的深度

两份分析文件都提出了"给 Motor 注入高斯白噪声"作为基底运动的解法。但这是一个**语义补丁** — 从外部注入人工噪声，违反了"所有信号必须有物理来源"的铁律。

你的体征模块提供了一个**根本性更好的答案**:

```
文件 5 方案:  motor += gaussian_noise()     ← 人工注入，无物理来源
文件 6 方案:  vestibular += noise()          ← 同上

你的方案:    体征搏动 → motor 基底驱动     ← 有物理来源（内部时钟）
```

### 体征搏动 = 系统时钟的物理表征

在真实生物体中:

| 搏动源 | 频率 | 功能 |
|--------|------|------|
| 心跳 | 1-2 Hz | 血液循环 → 组织灌注 → 代谢供能 |
| 呼吸 | 0.2-0.3 Hz | 气体交换 → 线粒体 ATP |
| 肠蠕动 | 0.05 Hz | 消化吸收 → 能量摄入 |
| 自发放电 | 宽频 | 神经网络自维持 → 可塑性基底 |

**所有这些搏动的共性**: 它们是能量耗散结构的自持振荡。不是"噪声"，是**有物理来源的周期性信号**，由系统的能量代谢本身驱动。

### 在我们的架构中如何实现

体征模块应该是一个**自振荡器**，由 EnergyStore 驱动:

```
EnergyStore (E_fill)
     ↓ 能量供给
VitalOscillator (自持振荡)
     ↓                    ↓                  ↓
  Motor 基底驱动        体征→Enc→Col       进食激活信号
  (代替人工噪声)        (内感受)           (能量低→进食)
```

关键设计约束:

1. **能量驱动**: 振荡幅度 ∝ E_fill。能量充足→搏动强→运动活跃。能量枯竭→搏动弱→运动减弱→死亡。
2. **结构计算**: 振荡频率由 RC 时间常数决定（电容+电阻），不是 `sin(t)` 参数。
3. **物理结算**: 每次搏动消耗能量（通过热耗散），进入 Noether 会计。
4. **三路耦合**: 
   - 搏动→Motor = **基底随机游走**（解决瓶颈②，替代高斯噪声）
   - 搏动→Enc = **内感受信号**（系统知道自己"活着"）
   - E_fill 低→搏动频率变化→进食行为激活

### 统一内外于物理内

这正是你说的"底层构建的必要性"的精髓:

```
外部世界:  热源 → SkinPatch → Fourier 积分 → 物理信号
                    ↑ 外感受

体征模块:  EnergyStore → VitalOscillator → RC 振荡 → 物理信号
                    ↑ 内感受

两者统一:  都是能量耗散 → 物理元件 → Noether 会计
           没有"外部 vs 内部"的分界，只有物理
```

当前系统的 `dt` (步长 0.001) 确实可以视为"系统时钟"。但体征搏动不应该是 `dt` 本身，而是由 dt 驱动的 **RC 振荡** — 就像心脏的窦房结节律细胞：

```python
# 概念: VitalOscillator（体征振荡器）
class VitalOscillator:
    """Self-sustaining oscillation driven by EnergyStore.
    
    BIO: sinoatrial node pacemaker.
    Physics: RC relaxation oscillator.
    """
    def __init__(self):
        self.voltage = 0.0
        self.threshold = 0.8    # 阈值
        self.tau_charge = 500   # 充电时间常数 (steps)
        self.tau_reset = 10     # 复位时间常数
        self.energy_coupling = 0.01  # 每次搏动的能量消耗
    
    def step(self, energy_level: float, dt: float):
        # 充电速率 ∝ 能量水平
        charge_rate = energy_level * dt / self.tau_charge
        self.voltage += charge_rate
        
        # 达到阈值 → 搏动 + 复位
        if self.voltage >= self.threshold:
            self.voltage = 0.0
            return self.energy_coupling  # 返回搏动信号 + 消耗能量
        return 0.0
```

> [!IMPORTANT]
> **核心洞察**: 体征模块不是一个"功能模块"——它是**系统存在的物理证明**。没有搏动 = 没有能量耗散 = 热力学死亡。它同时解决了：
> 1. 基底运动的物理来源（不再需要人工噪声）
> 2. 环流耦合的第三路参与者（运动—体征—进食）
> 3. 内外统一于同一套物理元件（RC + Noether）
