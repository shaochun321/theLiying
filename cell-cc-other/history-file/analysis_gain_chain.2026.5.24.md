## §2 根因分析: PowerRail 崩溃

Motor 神经元的供电公式:

$$V_{supply} = \frac{V_{DD} \times E}{1 + I_{total} \times R_{supply}}$$

代入实测值:

$$V_{supply} = \frac{1.0 \times 0.508}{1 + 1437.56 \times 0.1} = \frac{0.508}{144.76} = 0.00351$$

| 量 | 值 | 说明 |
|----|-----|------|
| V_supply | **0.0035** | 电源被拉垮到 3.5 mV |
| V_threshold | 0.300 | Motor 放电阈值 |
| 差距 | **85×** | 差 85 倍! |

> [!CAUTION]
> **Motor 不是"信号太弱"——是信号太强导致电源崩溃。**
> 类比：100W 负载接在 USB 2.0 口上 → 电压跌落到 0。

## §3 电流过载计算

Col→Motor 束的电流:

$$I_{motor} = \sum_{i=1}^{7} |a_{col,i}| \times G_i \times g_{syn}$$

代入:

| 参数 | 值 | 来源 |
|------|-----|------|
| col 源数 | 7 | 6 前庭 + 1 热 |
| avg \|a_col\| | 6.85 | yaw=7.5, ..., therm=1.47 |
| Σ\|a_col\| | 47.9 | 7 个列层总和 |
| G (w=1.0) | ≈0.99 | w²/(w²+0.01) |
| g_syn | 3.0 | col_to_motor 增益 |
| **I_motor** | **1437** | 实测 (propagate) |

> [!WARNING]
> 电流 1437 vs 允许最大值 23.3 → **过载 61.7 倍**

## §4 约束方程: I_max 推导

要求 Motor 能放电: $V_{supply} \geq V_{threshold} = 0.3$

$$\frac{V_{DD} \times E}{1 + I \times R_s} \geq 0.3$$

$$I \leq \frac{V_{DD} \times E / 0.3 - 1}{R_s}$$

取 E=0.5 (典型值):

$$I_{max} = \frac{1.0 \times 0.5 / 0.3 - 1}{0.1} = \frac{0.667}{0.1} = 6.67$$

**Motor 总输入电流必须 ≤ 6.67**

## §5 正确参数计算

### 方案: 降低 col→motor 增益 + 限制权重上限

需要:
$$\sum_{i=1}^{7} |a_{col,i}| \times G(w_{max}) \times g_{syn} \leq I_{max} = 6.67$$

取 Σ|a_col| = 48 (典型值):

$$48 \times G(w_{max}) \times g_{syn} \leq 6.67$$
$$G(w_{max}) \times g_{syn} \leq 0.139$$

## §6 Afferent 能量耗尽

```
Aff_yaw:   E = 0.015  (从 1.0 耗尽到 1.5%)
Aff_oto_x: E = 0.003  (几乎为零!)
```

原因: Afferent 是 spiking 神经元，每次放电消耗能量。
在 20k 步中大量放电 → 能量耗尽 → PowerRail 限流 → 无法放电。

但 Encoding 仍有 act=9.89! 说明 Afferent 在早期放电过、
留下了高权重，现在 Enc 靠自身 RC 维持高电压。

**修复**: 需要两个机制:
1. 自发放电基线 (bc_current) — 让 Afferent 有持续低频活动
2. 能量恢复率提高 — VR (Voltage Regulator) 补充速度需要匹配消耗

## §7 完整修复方案

| 组件 | 参数 | 当前值 | 建议值 | 理由 |
|------|------|--------|--------|------|
| Col→Motor 束 | g_syn | 3.0 | **0.1** | 防止 PowerRail 崩溃 |
| Col→Motor 束 | w_max | 1.0 | **0.5** | 限制最大电流 |
| Motor 神经元 | bc_current | 0 | **0.02** | 自发放电基线 |
| Motor 神经元 | v_threshold | 0.3 | **0.2** | 降低阈值(配合弱电流) |
| Aff 神经元 | bc_current | 0 | **0.05** | 自发放电 ~50Hz 基线 |
| Aff 神经元 | vr_base_rate | 0.02 | **0.05** | 加快能量恢复 |
| Enc 神经元 | — | — | 不变 | 当前工作正常 |
| Col 神经元 | — | — | 不变 | 当前工作正常 |

### 预期效果

```
修复后 Motor 电流:
  I = 48 × 0.962 × 0.1 = 4.6
  V_supply = 0.508 / (1 + 4.6 × 0.1) = 0.508 / 1.46 = 0.348
  V_threshold = 0.2
  V_supply (0.348) > threshold (0.2) ✅ → Motor 放电

修复后 Afferent:
  bc_current = 0.05 → 持续有基线电流
  vr_rate = 0.05 → 能量恢复更快
  → 不会耗尽到 E=0.003
```
