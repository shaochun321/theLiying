# 所有结构概念 — 完整词典

## 首先: 三组容易混淆的概念

```
容易混淆 1:  spine / column / area  vs  crystal
  → spine/column/area 是同一个元单元的"成长阶段"
  → crystal 是"新生的元单元" (出生方式不同)

容易混淆 2:  column (成熟阶段)  vs  column layer (层)
  → column (成熟阶段) = 一个元单元升级到了"班长"
  → column layer = 一个专门做巩固的区域 (里面有很多元单元)
  → 名字一样, 概念完全不同!

容易混淆 3:  crystal (结晶)  vs  frozen (冻结)
  → crystal = 一个新元单元被创建出来
  → frozen = 一条管道的学习被锁定
  → 一个是"出生", 一个是"停止改变"
```

---

## 第一组: 成熟阶段 (一个元单元的成长)

这三个不是三种不同的结构。它们是**同一个元单元的三个人生阶段**。

### spine (树突棘) — "学徒"

```
什么样:
  - 刚出生的默认状态
  - 可塑性最高 (plasticity=0.18) → 学东西最快
  - 但也最不稳定 (inertia=1.0) → 学了容易忘
  - 衰减快 (decay_rate=0.025) → 不用就忘
  - 没有特权: 不能影响周围, 不能发射蛋白

类比:
  新来的学徒——什么都在学, 什么都没定型。
  师傅说什么就信什么, 容易被带偏。
```

### column (柱) — "熟练工"

```
什么样:
  - 从 spine 升级而来 (需要: 电位积累>50 + 激活>20次)
  - 可塑性降低 (plasticity=0.01) → 学东西慢了, 但更谨慎
  - 更稳定 (inertia≥2.0) → 不容易被推翻
  - 衰减慢 (decay_rate=0.005) → 记得更久

  获得的特权:
  ┌─────────────────────────────────────────────┐
  │ 1. 侧向抑制 (半径=3)                         │
  │    "我已经代表这个模式了,                      │
  │     周围 3 个 spine 不需要重复学同样的东西"    │
  │                                             │
  │ 2. STDP 加速 (2×)                           │
  │    "我学的方向是对的, 加速巩固"               │
  │                                             │
  │ 3. PRP 发射                                  │
  │    "我可以远程帮助其他 spine 固化学习成果"     │
  └─────────────────────────────────────────────┘

类比:
  熟练工——已经有了自己的判断力。
  不再什么都信, 而是有选择地加深理解。
  还能指导新学徒。
```

### area (区域) — "老师傅"

```
什么样:
  - 从 column 升级 (需要: 电位积累>500 + 激活>200次)
  - 可塑性极低 (plasticity=0.001) → 几乎不再变化
  - 极度稳定 (inertia≥5.0) → 非常难被推翻
  - 衰减极慢 (decay_rate=0.001) → 长期保持

  获得的特权:
  ┌─────────────────────────────────────────────┐
  │ 1. 侧向抑制 (半径=5) → 影响范围更大          │
  │ 2. STDP 加速 (3×) → 学得更快更准             │
  │ 3. PRP 发射 → 帮助范围更广                   │
  └─────────────────────────────────────────────┘

类比:
  老师傅——代表了这个区域的"定论"。
  除非有压倒性的新证据, 否则不会改变。
  影响力覆盖整个工作区。
```

### 成长路径

```
出生 → spine → [积累足够] → column → [积累更多] → area
                    │                      │
                    │ 大部分元单元          │ 少数元单元
                    │ 停在这里              │ 能到达这里
                    ▼                      ▼
              "组成网络的主体"         "网络的骨架节点"
```

## 第二组: 结晶 (crystal) — "新元单元的诞生"

这是容易跟 column 搞混的概念。**完全不同的东西。**

### 什么是收敛检测 (convergence)

```
系统发现: 维度 A 和 维度 B 经常同时被激活
  → 创建一个"收敛节点" (ephemeral, 暂时的)
  → 记录: "A+B 一起出现的强度是多少"
  → 每 tick 衰减 (×0.99) → 如果 A+B 不再一起出现, 慢慢消失

类比:
  你注意到"每次下雨, 地上都湿"——但你还不确定这是规律。
  你先在脑子里记一笔。
  如果反复验证, 这笔记就加强。
  如果只是巧合, 这笔记就淡忘。
```

### 什么是结晶 (crystallization)

```
收敛节点存活了 >50 tick 且 强度 >0.01:
  → 它从"暂时笔记"变成"永久知识"
  → 系统创建一个全新的 MetaNeuron (编号 cx_xxx)
  → 这个新神经元接收 A 和 B 的输入
  → 它代表了 "A+B 一起出现" 这个知识

结晶 = 一个新的元单元出生了!
  - 它出生时是 spine (最初阶段)
  - 它可以自己再慢慢成熟为 column, 甚至 area
  - 它有自己的电容器, 场效应管, 电源轨
  - 它参与所有正常的电路活动
```

### crystal vs column 的区别

```
crystal (结晶):
  = 一个新元单元如何诞生
  = "我是从收敛检测中被创造出来的"
  = 出生方式

column (柱):
  = 一个元单元处于什么成长阶段
  = "我已经积累了足够的经验"
  = 成长阶段

一个 crystal 神经元出生时是 spine,
然后可以成长为 column,
最终可以成长为 area。

crystal 和 column 不在同一个维度上。
一个说的是"从哪来", 一个说的是"长到哪了"。
```

## 第三组: 果实 (fruit) — "错误标记"

### 什么是休眠果实 (dormant fruit)

```
一条管道 (MetaPipe) 发现:
  预测残差 Xin > 0.5 (预测和实际差太多了!)
  → 在这条管道上贴一张"便利贴" (dormant fruit)
  → 便利贴上写着:
      "什么时候发现的" (created_tick)
      "当时的 Xin 张力是多少" (tension)
      "当时各维度的状态快照" (z_t snapshot)
      "化学痕迹" (trace = 1.0, 慢慢衰减)
```

### 果实的三种命运

```
命运 1: 被激活 (capture)
  如果有"第三因子信号"到来, 且方向一致:
    → 果实被捕获 → 管道权重被修正 → 学习成功!
    → 相当于: 有人确认了你的笔记是对的, 写入正式文件

命运 2: 自然消亡 (dissolution)
  化学痕迹衰减到 <0.01:
    → 果实消失 → 这次错误被遗忘
    → 但! 消亡时的 z_t 快照被送往沉淀层 (sediment)
    → 相当于: 笔记作废了, 但扔进了档案箱

命运 3: 一直挂着
  如果信号不够强也不衰减:
    → 果实继续休眠 → 等待
    → 可能最终被捕获, 也可能最终消亡
```

## 第四组: 沉淀 (sediment) — "地质层"

```
什么是沉淀层:
  一个特殊的区域, 运行在"慢动作"模式下
  (每 20 个 tick 才更新一次)
  
  它的输入: 那些消亡的果实的 z_t 快照
  它做的事: 把这些快照慢慢融入"地形"
  
  不改变"波浪" (activation) → 改变"海底地形" (resting_potential)
  
类比:
  你每天产生很多想法, 大部分都忘了。
  但忘掉的想法不是完全消失——
  它们沉淀在你的直觉里, 改变了你看待事物的"默认角度"。
  
  这就是沉淀: 不是记住具体的事, 而是改变默认状态。
```

## 第五组: 冻结 (frozen) 和 代理 (proxy)

### frozen (冻结)

```
管道的学习规则可以是:
  - "oja"   → 正常学习
  - "bcm"   → 竞争学习
  - "frozen" → 完全不学

frozen 的管道保持权重不变 → 保护已学的知识。
相当于: 把这条路铺好水泥, 不再允许修改。
```

### proxy (代理)

```
当一个信号特征始终为零 (比如 sig_sync 一直是 0):
  → 该神经元实际上没有贡献
  → 但它的结构位置很重要 (其他东西连着它)
  → 所以让另一个神经元"代理"它的功能

类比:
  部门经理去度假了, 副经理代行其职。
  位置还在, 职能由别人顶替。
```

## 全结构总览

```
元单元的状态:
  spine ──→ column ──→ area          (成长阶段, 同一个元单元)

元单元的诞生方式:
  初始化创建 (build_circuit 时直接加的)
  结晶创建 (convergence → crystal, 运行时动态产生的)

管道上的附着物:
  dormant fruit (果实 — 错误标记, 等待确认)
  frozen (冻结 — 学习锁定)

特殊区域:
  sediment layer (沉淀层 — 吸收消亡的果实, 改变默认状态)

特殊角色:
  proxy (代理 — 替代失效的神经元)
  gate (门控 — 特化的元单元, 控制管道是否允许学习)
```

> [!IMPORTANT]
> **核心区分:**
> 
> | 概念 | 是什么 | 改变了什么 |
> |------|--------|----------|
> | **spine→column→area** | 一个元单元的成长 | 可塑性、稳定性、影响范围 |
> | **crystal** | 一个新元单元的诞生 | 网络结构 (多了一个节点) |
> | **fruit** | 管道上的临时标记 | 等待确认 (不直接改变权重) |
> | **frozen** | 管道的学习状态 | 权重是否还能变 |
> | **sediment** | 一个慢速区域 | 默认状态 (resting_potential) |
