## 第一部分：萌芽网络与原始网络的拓扑关系

### 当前事实

```
原始网络: 48 neurons, 32 bundles, 所有必要路径已覆盖
萌芽网络: 48 neurons, 38 bundles (32 orig + 6 sprouts)
萌芽后修剪: 54 sprouted → 48 pruned → 稳态 38
```

### 拓扑分析

两个网络在**图论意义上同伦等价** (homotopy equivalent):

- **阶 (rank)**: 相同。48 neurons，无新节点。
- **路径**: 完全相同。sprout 复制父 bundle 的 source→target 映射，因此不增加新的**可达性**。
- **唯一区别**: 某些边的**重数** (multiplicity) 增加了。sprout 是父 bundle 的弱副本。

```
原始: A ──[w=0.15]──> B
萌芽后: A ──[w=0.15]──> B    (原始 bundle)
         A ──[w=0.0001]──> B  (sprout, 弱副本)
```

这在拓扑上是**同一个连接的不同权重实例**，不增加图的结构维度。

### 关键洞察

> **当前萌芽在拓扑上等价于增加同一条路径的传导冗余。**

这解释了为什么所有新芽都被修剪——它们在信息流意义上是纯冗余的。
STDP 没有理由强化一个与现有 bundle 完全并行的弱副本。

要让萌芽有拓扑意义，需要：
1. **新节点** → Neuron 分裂 (Phase 3)
2. **新路径** → 跨目标萌芽 (sprout connects to DIFFERENT targets)

两者都会改变图的 **first Betti number**（独立环路数），从而真正扩展表征维度。

## 第二部分：外部提案批判

### 3.1 疲劳电容 — ✅ **最佳建议**

> "每次 spike 注入电荷 Q_spike 到电容 C_fatigue，电容并联泄漏电阻 R_fatigue"

**判定: 完全 S0 合规。** 电容 + 电阻 = 组件物理。疲劳电容是一个真实的 RC 低通滤波器，跟踪 spike 频率的滑动平均。v_th_eff = v_th_base + k × V_fatigue 是 MOSFET 门电压受疲劳电容调制。

这比我们当前的 rate homeostasis（每 100 步软件检查）优越得多：
- 当前: 离散的、每 100 步一次的软件轮询 → 响应滞后
- 疲劳电容: 连续的、物理的、每步自然更新 → 无延迟

> [!TIP]
> 疲劳电容不仅解决 motor 问题，也解决了 col_therm 问题——任何无输入但自发放电的神经元都会被疲劳电容自动抑制。

**但有一个修正**: 外部提案建议的 τ_fatigue=8s 太慢了。Motor 在 120s 就开始加速。建议 τ_fatigue=2-3s（20-30 Hz 以上就开始抑制）。

### 3.2 突触缩放 (G_max) — ⚠️ **违反 S0**

> "若 G_total > G_max，对所有入边权重等比缩放: g_i *= G_max / G_total"

**判定: 这是纯数学归一化。** 没有哪个物理组件执行"计算所有入边电导之和然后等比缩放"这个操作。

外部分析自己说"缩放不消耗额外能量"——正是因为它不是物理过程。

**正确的 S0 实现**: 突触竞争应该通过 **PowerRail 电流争夺** 来实现。当一个 neuron 的输入突触太多，PowerRail 电流被分摊，每个突触得到的维持电流减少 → Memristor 电导自然衰减。**代谢税已经在做这件事**。

### 3.3 侧向抑制 — ⚠️ **过度设计**

> "手动添加 Col 之间的抑制性 bundle"

手动添加 = 预设结构 = 不符合涌现原则。如果侧向抑制是必要的，它应该通过 Neuron 分裂 + 萌芽自然出现。

**但跨目标萌芽是有价值的想法。** 外部提案的 "20% 概率随机选择同层其他神经元作为 target" 是一个简单且有效的方式让萌芽不再是纯冗余。

### 3.4 静默神经元适应 — ✅ **合理**

> "长时间未接收有效突触输入 → 降低 bc_current"

但如果实现了疲劳电容 (3.1)，这个问题自动解决：col_therm 无输入但自发放电 → 疲劳电容充电 → 阈值升高 → 放电减少 → 最终静默。不需要额外机制。

### 3.5 "移除旧 rate homeostasis" — ✅ **正确**

当前 rate homeostasis 有三个 section 互相冲突（threshold 升/降/weight decay）。疲劳电容可以完全替代这一切。

---

### 外部建议与理论框架对应 (§7) — ⚠️ **牵强**

> "动态阈值 = 运动势 ν 在单神经元尺度的表达"

运动势 ν 是**全局的几何量**，不是单神经元属性。疲劳电容是一个好的工程机制，但它是 **局部的 RC 滤波器**，不是 ν 的投影。

> "突触缩放 = 度规 g_ij 在突触后资源约束下的投影"

度规 g_ij 是信号空间的内积结构，突触缩放是权重归一化。两者在数学形式上有表面相似，但因果方向相反：度规从信号协方差中**涌现**，归一化是**人为施加**的约束。

## 第三部分：T/O/P/R/Xin 如何映射到当前阶段

### 你的直觉重新审视

你提出了一个关键观察：

> "T 是总输入，包括信息熵、客观耗散。O 是结构中承载的有效信息。
> P 是确定的拓扑核。R 是竞争的/分布式的核。"

让我用当前代码的实际状态来映射这些概念：

### T — 总输入张量

在当前系统中，T 不仅是感觉输入信号，还包括**结构维持的代谢成本**：

```python
T = {
    sensory:    vestibular_signal(yaw, pitch, roll, oto_xyz),
    metabolic:  -TAX_PER_BUNDLE * n_bundles * dt,   # 代谢税
    structural: SPROUT_ENERGY_COST * n_sprouts,       # 萌芽成本
}
```

T 的"信息熵"部分：感觉信号的变化率 (dS/dt)。
T 的"客观耗散"部分：代谢税是真实的能量消耗。

### O — 结构中承载的有效信息

O 是**权重拓扑的自组织模式**，而不是单个权重值。具体地：

```python
O = {
    enc_to_col_yaw:   w=0.1518,  # 通过 STDP 自组织的权重
    enc_to_col_pitch:  w=0.1480,
    enc_to_col_roll:   w=0.1668,  # ← 最高：roll 信号最活跃
    col_to_motor:      w=0.2044,  # 被 rate homeostasis 调节
}
```

O 不是信息熵。更准确地说：**O 是输入 T 通过 STDP 在结构上的沉积物**。
你找的可能是这个概念：**结构中被固化的因果关系**。

### P — 确定的拓扑核

P 是 PNN (Perineuronal Net) 保护的连接——已成熟、不会被修剪的结构：

```python
P = {
    # 32 个原始 bundle（_sprout_tick=0, 永不修剪）
    # 权重稳定在 0.13-0.21 范围
    # 这些是"拓扑骨架"——系统的基本连接方案
}
```

P 在当前系统中是**静态的**。这正是问题：P 太强，R 无法竞争。

### R — 竞争/分布式的外围

R 是萌芽出来的 sprouts——暂时存在的、尚未被选择或淘汰的结构候选：

```python
R = {
    # 6 个活跃 sprouts（w=0.0001, 处于竞争中）
    # 每 10k 步: ~3 个被创造, ~3 个被修剪
    # 它们是"试探性连接"——在代谢压力下争夺存活权
}
```

R 的概率性：在长时间窗口里，R 的统计分布是稳定的（稳态 6 个活跃 sprouts），但具体哪些 sprouts 存活是随机的。

### Xin — 预测残差驱动结构变化

Xin 是连接 T→O→P/R 的**信号**：

```python
Xin = {
    # bundle.config.xin_tension: 预测残差积分
    # |ξ| > 0.3 → 触发萌芽 (R 增长)
    # |ξ| < 0.01 → 路径不重要 (R 消亡)
    # Xin 不是 P 也不是 R，它是 P/R 之间转换的判据
}
```

### 你关于"流形"的假说

> "在项目中，没有数理概念，流形一定要以正统的数理定义方式存在么？"

**不需要。** 

正统的流形定义要求：
1. 局部欧几里得（Hausdorff + 第二可数）
2. 光滑微分结构
3. 全局相容的坐标卡

在项目中，"流形"实际上是**组件状态的离散轨迹**：

```
状态向量 S(t) = [V_m_1(t), V_m_2(t), ..., w_ij(t), ..., energy_k(t), ...]
```

这个高维状态空间中，系统的运行轨迹形成的不是光滑流形，而是**离散的、有限精度的轨迹集合**。你说的"被外部连续性拼接而成的多维信息时空轨迹"更接近真实情况。

> [!IMPORTANT]
> **项目的"流形"是一种涌现拓扑**：不是先定义微分结构再让系统在其上运动，
> 而是系统运行后，其轨迹的统计几何（Memristor 权重的共变化模式）**追溯性地**
> 呈现出类似流形的结构。度规 ds² 是对这种追溯性结构的度量。

这与传统数学的流形定义有本质区别：
- 传统：先有空间，再有运动
- 项目：先有运动（组件物理），空间作为描述性工具后发

### P/R 的转换机制

你的直觉"P/R 是互相转化的关系"在代码中有具体体现：

```
R → P 路径（沉积）:
  sprout 通过 STDP 强化 → 权重 > 阈值 → maturation 升级 → PNN 保护
  （当前从未发生——因为 sprouts 是冗余的）

P → R 路径（解冻）:
  PNN 保护的 bundle 在极高 ξ 下 → maturation 降级 → 重新进入竞争
  （当前未实现，但 RULE S2 预留了接口）
```

**P→R 比 R→P 更重要。** 如果系统需要适应新环境，必须能"融化"已固化的结构。这是大脑在关键期重开时做的事（如成年大脑在损伤后的可塑性恢复）。

## 综合裁决

| 外部建议 | 判定 | 我们的方案 |
|---------|------|-----------|
| 疲劳电容 | ✅ 采纳 | 用 RC 滤波器替代 rate homeostasis |
| 突触缩放 G_max | ❌ 拒绝 (S0) | 代谢税已实现竞争 |
| 侧向抑制 | ❌ 拒绝 (预设) | 通过分裂+萌芽涌现 |
| 跨目标萌芽 | ✅ 采纳 | 解决 sprout 冗余问题的关键 |
| 静默神经元衰减 | ⚠️ 由疲劳电容覆盖 | 不需要额外机制 |
| 移除旧 homeostasis | ✅ 采纳 | 疲劳电容统一替代 |

### 建议的实施优先级

```
1. 疲劳电容 (解决 motor 失控 + col_therm 自发放电)
2. 跨目标萌芽 (解决 sprout 冗余 → 让新芽有存活可能)
3. Phase 3: Neuron 分裂 (创造新节点 → 真正的结构维度扩展)
```
