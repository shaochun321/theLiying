# 实践层：五个问题的回答

## Q1：时间分辨率 — 大规模与小规模粒子趋同吗？

### 统计力学的回答：是的，趋同。

根据 **热力学极限（thermodynamic limit）** 理论：

当粒子数 N → ∞，体积 V → ∞，且密度 N/V 保持恒定时，
系统的**宏观性质收敛到确定值**，微观涨落消失。

关键的定量关系：

```
涨落幅度 ∝ 1/√N

N = 20:   涨落 ≈ 22% — 已经接近热力学极限
N = 50:   涨落 ≈ 14%
N = 214:  涨落 ≈ 6.8%
N = 1000: 涨落 ≈ 3.2%
N = 10⁶:  涨落 ≈ 0.1% — 几乎确定性
```

> [!IMPORTANT]
> **你的直觉是正确的**：大规模与小规模的宏观行为（平均应力、平均V_m、
> entropy 分布形状）趋同。**变化的确实只是信息带宽** —— 更多粒子意味着
> 更精细的统计采样，但统计量本身（均值、方差、分布形状）收敛。

### 实际含义

| N | 宏观行为 | 细节分辨率 | 计算时间 |
|:-:|---|---|:-:|
| 20 | ≈ 正确 | 粗糙 | ~0.5s/窗口 |
| 50 | ✓ 正确 | 中等 | ~1s/窗口 |
| 214 | ✓✓ 精确 | 精细 | ~3s/窗口 |
| 1000 | ✓✓✓ 确定性 | 极精细 | ~70s/窗口 |

**结论**：实践层可以用 **N=20~50 做低分辨率快速仿真**（用于闭环学习），
必要时用 N=214 做高分辨率验证。这不是近似 —— 是统计力学保证的收敛。

### 时间压缩方案

```
实践层时间分辨率：
  每个 circuit tick → 运行 1 个物理窗口（100步 × N=30）
  每步 ≈ 0.1ms → 每窗口 = 10ms 物理时间
  300 ticks × ~50ms/窗口 = ~15 秒总计算时间

  对比当前（开环）：
  100 窗口 × 214 粒子 = ~3 分钟（但这是预计算所有窗口）
  闭环只需逐窗口运行，而且用小 N
```

## Q2：运动自由度 — 为什么是 7 个 z_t？应该有多少运动 DOF？

### 当前 7 个 z_t 的来源

7 个 z_t 维度不是随意选择的，它们**对应 signal entropy 的 7 个通道**：

```
transition      ← 相邻窗口的 V_mean 变化幅度
drift           ← V_mean 的长期漂移趋势
gamma_desync    ← 高频（γ波段）去同步化程度
xin_residual    ← Xin 残差张力
potential_disp  ← 电位离散度
churn           ← 细胞身份更替率
magnitude       ← 整体信号幅度
```

这 7 个维度描述的是"正在发生什么" — 信号的统计特征。

### 真实运动系统的 DOF

| 系统 | 总 DOF | 有效 DOF |
|---|:-:|:-:|
| 人体骨骼 | ~244 关节自由度 | — |
| 人手 | 27 DOF | — |
| Atlas 机器人 | 56 DOF | — |
| Unitree G1 | 23~43 DOF | — |
| **Bernstein 运动协同** | — | **~6-8 个协同模式** |

### Bernstein 的关键发现

Nikolai Bernstein 发现了**运动冗余问题**：人体有 244 个关节自由度，
但完成一个动作（如拿杯子）只需要 ~6 个有效维度。

**解决方案：运动协同（Synergies）**

神经系统不独立控制每个关节，而是将多个关节绑定为**协同单元**。
一个"伸手"指令同时协调肩、肘、腕、指的运动。

> [!TIP]
> **对项目的启示**：实践层不需要 56 个运动 DOF。
> 借鉴 Bernstein，应该有 **~6 个运动协同维度**（motor synergies），
> 每个协同维度控制一组粒子的力场参数。

### 提议的 6 个运动协同维度

| 运动协同 | 物理载体 | 生物类比 |
|---|---|---|
| `motor_push` | 施加定向推力 | 伸展/推 |
| `motor_pull` | 施加定向拉力 | 收缩/拉 |
| `motor_rotate` | 施加扭矩 | 旋转 |
| `motor_squeeze` | 修改局部弹簧刚度 ↑ | 抓握/收紧 |
| `motor_release` | 修改局部弹簧刚度 ↓ | 松开/释放 |
| `motor_fixate` | 选择观察区域 | 注视/注意力 |

这 6 个维度通过 circuit 的输出层激活，作为 perturbation force 施加到物理引擎。

## Q3：双驱动对齐

你说"双驱动对齐"，这与发展神经科学的发现完全一致：

### 真实婴儿的运动发展

```
阶段 1：原始反射（preset reflexes）
  ├── 抓握反射、吮吸反射、步行反射
  ├── 由脑干控制，不经过大脑皮层
  └── 提供初始的运动-感知对（硬编码的 P）

阶段 2：运动噪声（motor babbling）
  ├── 随机运动 → 观察感觉后果 → 建立因果模型
  ├── 通过 Hebbian 学习巩固有效的运动-感知对
  └── 逐步覆盖反射未涵盖的运动空间

阶段 3：反射整合（reflex integration）
  ├── 反射逐渐被自主控制替代
  ├── 但反射模式被保留为"紧急回路"
  └── 最终形成分层控制：皮层 > 反射
```

### 在项目中的实现

**Bernstein 的冻结-释放策略**（freeze-then-release）：

```
初始化阶段：
  Phase A（冻结 — 反射驱动）：
    6 个运动协同中只激活 2 个（motor_push, motor_fixate）
    预设简单的触发规则：
      if energy_H > threshold → motor_push(away)     # 回避反射
      if spectral_H 变化大 → motor_fixate(toward)   # 朝向反射
    
  Phase B（释放 — babbling 驱动）：
    逐步解冻其余 4 个运动协同
    每个 tick 以 ε 概率发出随机运动指令
    记录 motor_output → Δ(signal_entropy) 的因果对
    通过 STDP 巩固有效的 motor-sensory 映射

  Phase C（整合）：
    反射回路仍存在但阈值提高
    大部分运动由学习到的 P 固化模式驱动
    新的 convergence nodes 连接 motor 和 sensory 维度
```

## Q4：为什么 CPG 要生成节律？心跳眼动是否属于运动层？

### 你的直觉是对的：CPG 就是运动层的一部分

```
运动系统的分层：

    运动皮层（目标选择）
        │
        ↓ "走到那里"
    脑干 / 基底节（运动规划）
        │
        ↓ "左右交替迈步"
    脊髓 CPG（节律生成）
        │
        ↓ "屈肌-伸肌交替 @ 1.5Hz"
    运动神经元 → 肌肉
```

### 为什么要节律？

**节律是运动的默认模式（default mode）**：

| 运动类型 | 节律性 | 控制层 |
|---|---|---|
| 心跳 | ✅ ~1Hz | 心脏自律 + 脑干调节 |
| 呼吸 | ✅ ~0.2Hz | 脑干 CPG |
| 步行 | ✅ ~1.5Hz | 脊髓 CPG |
| 咀嚼 | ✅ ~1Hz | 脑干 CPG |
| 眼球扫视 | ✅ ~3Hz | 上丘 + 脑干 |
| 伸手抓取 | ❌ 非节律 | 运动皮层 |

绝大多数运动是节律性的！非节律运动（有目标的伸手、精细操作）反而是少数。

> [!IMPORTANT]
> **CPG 生成节律的原因**：节律性是能量最优的运动方式。
> 
> 步行时，腿像钟摆一样摆动 — 利用重力势能和弹性势能的交换，
> 几乎不需要额外能量。CPG 提供的节律频率恰好匹配身体的固有频率。
> 
> 心跳和呼吸同理 — 它们是维持生存的最低能耗持续运动。

### 对项目的含义

当前的 CPG（`cpg_fast_a/b`, `cpg_slow_a/b`）**已经是运动层的一部分**：

```
当前架构：
  CPG（类生体层）= 自主运动（心跳/呼吸 类比）
  └── 提供基础节律 → 防止 circuit 热死

实践层扩展后：
  CPG = 自主运动基底
    ├── visc_rhythm → 持续的基础力场振荡（类似呼吸）
    └── visc_baseline → 基础张力维持（类似姿势维持）
  
  Motor Cortex = 目标导向运动
    ├── motor_push/pull/rotate → 离散的力场指令
    └── motor_fixate → 注意力窗口选择

  两者的关系：
    motor_cortex 输出 叠加在 CPG 节律之上
    就像你走路时（CPG 驱动）可以同时伸手拿东西（皮层驱动）
```

### 心跳和眼动属于运动层吗？

**是的**。在神经科学中：

- **心跳**：心脏有自己的 CPG（窦房结），脑干通过迷走神经调节
- **呼吸**：脑干的 Pre-Bötzinger Complex 是呼吸 CPG
- **眼动**：上丘 + 脑干眼动核 生成扫视节律

它们都是**自主运动** — 不需要意识参与但可以被意识调节。

在项目中：
- `visc_rhythm` 对应心跳/呼吸的持续搏动
- `visc_baseline` 对应姿势/张力的基础维持
- 实践层的 `motor_fixate` 对应眼动的主动扫描

## Q5：外部熵账本

已确认。每个运动指令需要在外部熵账本中记录：

```
运动做功 = F · d = perturbation_force · particle_displacement
能量变化 = ΔKE + ΔPE + 阻尼耗散
熵变化   = Δ(signal_entropy) 各通道的变化

运动熵效率 = |Δ(signal_entropy)| / 运动做功
  → 高效率 = 少量力产生大量信息变化（精巧的探测动作）
  → 低效率 = 大量力产生小量信息变化（笨拙的蛮力）
```

这个效率指标可以驱动运动学习：
系统应该逐渐学会用最少的力获取最多的信息。
