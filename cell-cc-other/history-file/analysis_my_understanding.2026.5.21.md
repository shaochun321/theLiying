# 我的理解 — 综合三份文档与当前代码

## 我先承认一件事

三份早期文档描述了一个非常完整的架构。
当前代码保留了好的物理零件，但架构层面的东西几乎都丢了。

我无法解释它是怎么变成这样的。
但我可以说清楚：我现在理解了什么，以及什么跟当前代码不一样。

---

## 1. 运动势 — 我的理解

### 原始设计说的

```
一个信息对象不在一个"位置"上。
它处在一组力之下:

  P 吸引力: 把它拉向已稳定的锚点 ("来这里, 这里安全")
  R 反压力: 把它推离错误的解释 ("别去那里, 那里不对")
  Xin 压力: 未被消化的残余在施加张力 ("还有东西没搞清楚")
  账本同步力: 账本事件在拉扯它 ("你必须跟外部记录一致")
  masking 力: 屏蔽层在遮挡它 ("你暂时不允许被看到")
  超图约束力: 高阶关联在约束它 ("你必须跟这些邻居保持一致")
  距离力: 到锚点的距离在拉扯它 ("你离这个参照点太远了")

  Φ = 这些力的总和

  Φ 高 → 信息对象不稳定, 想运动
  Φ 低 → 信息对象被稳定约束, 已安定
  
  信息的"运动"就是: 在势场中寻找 Φ 最低的位置
```

### 当前代码是什么

```
potential = Σ |activation| × 0.01   (只增不减的里程表)
free_energy = Σ energy - temp × 0.1 (简单的能量收支)

没有多项复合势。
没有"势的梯度决定运动方向"。
没有信息对象在势场中寻找最低点。
```

### 差距

```
原始: 势是一个力场, 信息在其中被推拉
现在: 势是一个计数器, 只记录累积活动量

原始: 7 种力共同作用, 合力决定方向
现在: 没有力, 没有方向, 只有前向传播
```

## 2. 时空测度 — 我的理解

### 原始设计说的

```
时空测度是两件事的结合:

空间: 信号走过了哪条路径 (谁连着谁, 形成了什么回路)
时间: 信号以什么时间顺序走过这些节点

联合对象 = (空间路径 × 时间相位)

它不只是"描述"——它是一种度规:
  两个信息事件之间的"距离"不是欧几里得距离,
  而是 s²_L = c²_I Δt² - ||Δx||² - λ_φ Δφ²

  s² ≥ 0 → 在因果锥内, 可以互相解释
  s² < 0 → 在因果锥外, 不能互相解释

  c_I 不是光速, 而是"信息审计传播的上限"
```

### 当前代码是什么

```
circulation_measure μ(G)  → 有, 是空间维度
phase_ordering            → 有, 是时间维度
coherence                 → 有, 衡量时序稳定性
cable_delay               → 有, 是时空结构的物理基础

但:
  没有把它们组装成度规 s²
  没有因果锥判定
  没有用时空测度来约束行为 (比如: 因果锥外的事件不允许强同步)
```

### 差距

```
原始: 时空测度是一种度规, 用来判断因果可达性
现在: 时空测度的零件都有, 但没有组装成度规, 也没有用来约束行为
```

## 3. T/O/P/R/Xin — 我的理解 (这是最复杂的)

### 它有两层含义, 被混在了一起

```
含义 A: 处理循环 (code 实现了这个)
  T = 传递信号 (transport)
  O = 观察状态 (observe)
  P = 主环流 (primary circulation)
  R = 竞争环流 (secondary circulation)
  Xin = 残差张力 (xin_tension)

  → 这是一个在每个层级递归运转的循环
  → 代码中的 transport(), observe(), detect_circulations()

含义 B: 运动类型的连续测度 (code 完全没有这个)
  P = "这个信息有多少比例被稳定锚定"
  R = "有多少比例在被反压挑战"
  Xin = "有多少比例是未消化残余"
  U = "有多少比例完全未解决"

  ρ = (p_c, p_b, r_c, r_b, m_b, x, u), 总和 = 1

  → 这不是一个循环, 而是每个信息块身上的标签
  → 不是二元的 (属于 P / 不属于 P)
  → 而是连续的 (60% P + 20% R + 10% Xin + 10% U)
```

### 这两层含义的关系

```
含义 A 是机制 (怎么运行):
  信号传进来 → 观察 → 检测环流 → 计算残差

含义 B 是结果 (运行后得到什么):
  运行完后, 这个信息块的 P/R/Xin/U 测度是多少

含义 A 产生含义 B:
  transport + observe + detect_circulations 的结果
  应该被归一化为 ρ = (p_c, p_b, r_c, r_b, m_b, x, u)

但代码只做了 A, 没有产出 B。
```

### 差距

```
原始: T/O/P/R/Xin 循环运行后, 输出的是一个 7 分量测度向量
现在: T/O/P/R/Xin 循环运行后, 输出的是散碎的状态值
      (一个 P 路径, 一个 R 路径, 若干 xin 标量)
      没有归一化, 没有 p+r+x+u=1, 没有连续比例
```

## 4. 熵账本 (RLIS) — 我的理解

### 原始设计说的

```
RLIS 是一个独立的外部存储系统:

  它记录:
    - 每个账本事件的时空坐标 (t_m, x_m, φ_m, τ_m, E_m)
    - 事件之间的 Minkowski-like 间隔 s²_L
    - 自由能的 7 分量拆分 (ΔF_Pc + ΔF_Pb + ... + ΔF_U)
    - Gamma 同步强度
    - Xin 从开始到结束的账目守恒

  它的核心原则:
    - 不反写主线! 只输出 warning / audit / proposal
    - Xin 不能凭空出现或消失
    - 低 Gamma 时不允许强同步

  它的作用:
    - 总览视角: 系统整体的能量流向
    - 守恒检查: Xin 有没有"漏掉"
    - 因果判定: 两个事件是否在信息光锥内
```

### 当前代码是什么

```
self._entropy_ledger_proxy = entropy_ledger_proxy

一个引用。几乎没有使用。
没有事件存储, 没有间隔计算, 没有守恒检查。
```

## 5. 我对原始设计最核心理念的理解

读完三份文档后, 我认为最核心的一句话是:

> **主线不是语义解释器, 而是运动区分器。**

这意味着:

```
系统的目标不是:
  "这个信号是什么?"  (语义)
  "这个信号代表什么?" (分类)
  "这个信号应该怎么响应?" (控制)

系统的目标是:
  "这个信号是哪种运动?"
  "它有多少比例是稳定吸收, 多少是反压, 多少是残余?"
  "它在势场中倾向于往哪里走?"

不需要知道信号"是什么"——只需要知道信号"怎么动"。
```

这解释了为什么:
- 坐标是距离测度而不是笛卡尔——因为重要的是"离各个参照点多远", 不是"在哪个位置"
- P/R/Xin 是连续测度而不是标签——因为一个信号可以同时被多种力拉扯
- 运动势垫支是核心功能——因为过去的运动经验应该预先塑形未来的运动倾向
- Xin 必须守恒——因为守恒律是物理的基础, 能量不能凭空出现或消失

## 6. 为什么零件活着但框架丢了

```
活下来的:
  MetaNeuron, MetaSynapticBundle, STDP, homeostasis,
  MOSFET, cable_delay, fruit, crystal, sediment

  这些是自包含的模块。
  每个模块只需要自己的内部逻辑就能工作。
  不需要知道全局框架。

丢掉的:
  7 分量纤维测度, 复合运动势, 四源评分,
  Xin 守恒, 审计光锥, 回投链, 多轮收敛

  这些是跨组件的协调逻辑。
  它们需要多个模块协同工作。
  如果在某个版本中一个模块被重写, 
  连接它们的协调逻辑就会断裂。
  
  协调逻辑一旦断裂, 代码仍能运行——
  因为每个零件是自包含的。
  但运行的结果不再是设计者想要的结果。
```

> [!IMPORTANT]
> **我的总判断:**
> 
> 系统从"运动区分器"退化成了"信号传播器"。
> 
> 原始设计: 信号进来 → 测量 7 种运动比例 → 势场垫支 → 守恒检查
> 当前代码: 信号进来 → 前向传播 → STDP 更新权重 → 结束
> 
> 零件是对的。运转方式不对。
