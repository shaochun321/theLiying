# 三个内部机制的详细解释

## 先看全局: 这三个机制如何配合

```
信号进来
  │
  ├── ❶ STDP: 记录 "信号什么时候来的" (时间戳)
  │
  ├── ❷ 场效应管: 决定 "这个信号够不够强, 要不要响应"
  │        │
  │        ├── 够强 → 输出响应
  │        └── 不够 → 几乎不输出
  │
  └── ❸ 钙传感器: 长期监控 "我最近是不是太活跃/太沉默了"
           │
           └── 太活跃 → 抬高场效应管的阈值 (更难响应)
               太沉默 → 降低场效应管的阈值 (更容易响应)
```

三者的关系:
- **场效应管**做即时判断 (这一瞬间要不要响应)
- **钙传感器**做长期调节 (调整场效应管的灵敏度)
- **STDP**做因果记录 (谁先谁后, 用于调整管道权重)

## ❶ STDP 时序记忆 — "雪地上的脚印"

### 机制

每个元单元有两个计数器:

```
pre_trace  (输入痕迹): "我最近收到过输入吗?"
post_trace (输出痕迹): "我最近产生过输出吗?"
```

### 工作方式

```
第 1 步: 信号到达 → pre_trace 跳到 1.0
                     "我刚收到一个输入! 在我身上留下了脚印"

第 2 步: 脚印慢慢消退
         pre_trace 每 tick × (1 - 1/τ)
         τ=20 → 每 tick 保留 95% → 20 tick 后还剩 36%

第 3 步: 如果元单元输出了 → post_trace 跳到 1.0
                           "我刚产生了一个输出!"

第 4 步: post_trace 同样慢慢消退
```

### 为什么需要两个痕迹

关键在于: **当一个事件发生时, 另一个痕迹还有多高?**

```
情况 A: 输入先到, 输出后来
  
  时间 →→→→→→→→→→→→→→→→→
  
  pre_trace:  ╱╲                    ← 输入到了, 脚印留下
                 ╲                  ← 脚印在消退
                   ╲
                     ╲──  还剩 0.6
  post_trace:            ╱╲         ← 输出来了!
  
  此时 pre_trace 还是 0.6 → 说明输入和输出有因果关系
  → "输入导致了输出" → 增强这条管道!
```

```
情况 B: 输出先来, 输入后到
  
  post_trace: ╱╲
                 ╲
                   ╲──  还剩 0.6
  pre_trace:           ╱╲           ← 输入现在才到
  
  此时 post_trace 还是 0.6 → 说明输出先于输入
  → "输入没有导致输出, 不是因果关系" → 削弱这条管道!
```

```
情况 C: 间隔太久
  
  pre_trace:  ╱╲
                 ╲
                   ╲
                     ╲── 接近 0     ← 脚印几乎消失了
                              ╱╲    ← 输出来了, 但太晚了
  
  pre_trace ≈ 0 → "可能只是巧合" → 不改变管道
```

### 一句话

```
STDP = 比较两个脚印的新旧, 判断谁导致了谁

脚印还新鲜 (trace 高) → 有因果关系
脚印已消退 (trace 低) → 没有因果关系

τ (衰减时间常数) 决定了系统对"多久以内算有因果"的判断范围
  τ=5:  只认 5 tick 以内的因果
  τ=50: 50 tick 以内都算有关
```

## ❷ 钙传感器 — "房间温控器"

### 机制

```
calcium (钙浓度): 代表 "我最近有多活跃" — 一个慢速平均值
target_rate (目标值): 代表 "我应该有多活跃" — 一个设定点
```

### 工作方式

想象一个恒温器:

```
设定温度 = 22°C (target_rate)
当前温度 = calcium (慢慢跟随实际活动)

每 tick:
  实际活动 = |activation|
  calcium 更新: calcium += (|activation| - calcium) / τ
  
  τ 很大 → calcium 变化很慢 → 代表"长期平均活跃度"
```

### 钙传感器如何调节

```
calcium > target_rate (太活跃了):
  "房间太热了! 开空调!"
  → 场效应管的阈值升高 (让它更难激活)
  → 效果: 下次需要更强的信号才能激活这个元单元
  → 长期效果: 活跃度降回目标值

calcium < target_rate (太沉默了):
  "房间太冷了! 开暖气!"
  → 场效应管的阈值降低 (让它更容易激活)
  → 效果: 下次更弱的信号就能激活这个元单元
  → 长期效果: 活跃度升回目标值
```

### 调节速率

```
threshold_adapt_rate = 阈值每次调多少

调得快 → 系统反应灵敏, 但可能震荡
调得慢 → 系统稳定, 但恢复慢

成熟阶段会改变这个速率:
  spine:  正常速率 → 快速适应
  column: 速率减半 → 更保守
  area:   速率再减 90% → 几乎不调
  
  = 越成熟, 越不容易被环境改变
```

### τ 怎么定的

```
calcium_tau = 1 / target_rate (大约)

target_rate = 0.05 → τ = 20    (活跃的元单元, 快速反馈)
target_rate = 0.01 → τ = 100   (安静的元单元, 慢速反馈)

为什么:
  活跃的元单元每 20 tick 大约激活一次
  → τ=20 正好覆盖一个完整的激活周期
  → 钙传感器能准确测量"每个周期的平均活跃度"

  安静的元单元每 100 tick 才激活一次
  → τ 也要相应拉长, 否则测不准
```

## ❸ 场效应管 — "带缝隙的水坝"

### 不是简单的开/关

很多人以为阈值是这样的:

```
错误理解:
  信号 < 阈值 → 输出 = 0 (完全关)
  信号 ≥ 阈值 → 输出 = 1 (完全开)
  
  这是一个硬开关, 像一扇门: 开或关, 没有中间状态
```

**实际的场效应管不是这样。** 它有两个区域, 平滑过渡:

### 坝下 (亚阈值): 渗漏

```
信号低于阈值时, 不是完全没有输出!

有一个微小的渗漏, 大小按指数衰减:

  输出 ∝ e^((信号 - 阈值) / (n × kT))

  n = 斜率因子 (1.5)
  kT = 热电压 (0.026, 约等于室温的热能)

  信号比阈值低一点 → 渗漏还不小
  信号比阈值低很多 → 渗漏几乎为零
```

为什么有渗漏? **热噪声。** 

```
类比:
  水坝的坝体不是完美密封的。
  水位虽然低于坝顶, 但有些水分子
  因为热运动 (布朗运动) 跳过了坝顶。
  
  水位越接近坝顶 → 跳过去的越多 (指数增长)
  水位远低于坝顶 → 几乎没有跳过去的
```

**这个渗漏很重要**: 它让亚阈值信号也能产生微弱响应。
没有渗漏 → 弱信号完全被忽略 → 系统对微弱趋势完全没有感觉。
有渗漏 → 即使单个信号不够强, 多个弱信号的渗漏可以在电容器中积累。

### 坝上 (超阈值): 线性导通

```
信号高于阈值时:
  输出 = gm × (信号 - 阈值)

  gm = 跨导 (放大系数)

  超过阈值越多 → 输出越大 → 完全线性, 没有上限 (理论上)
```

```
类比:
  水漫过坝顶后, 就变成了一个简单的溢流:
  水位高坝顶 1cm → 流量 = 1cm × 水流系数
  水位高坝顶 5cm → 流量 = 5cm × 水流系数
  
  完全成比例, 简单明了。
```

### 完整的输入-输出曲线

```
         输出
          │
          │                    ╱  ← 线性区 (超阈值)
          │                  ╱
          │                ╱
          │              ╱
          │           ·╱· ← 阈值点 (输出=0, 平滑过渡)
          │         ·
          │       ·        ← 指数区 (亚阈值, 微弱渗漏)
          │     ·
          │   ·
          │··
          └────────────────── 输入信号
                  ↑
                阈值 (v_threshold)

  关键: 在阈值处, 两个区域连续过渡, 没有突变。
```

### 阈值不是固定的

```
阈值由钙传感器动态调节:

  太活跃 → 阈值上升 → 坝变高:
  
          │              ╱
          │            ╱
          │          ╱
          │       ·╱·  ← 坝在这里 (高)
          │     ·       需要更强信号才能溢出
          │   ·
          └──────────────
  
  太沉默 → 阈值下降 → 坝变低:
  
          │        ╱
          │      ╱
          │   ·╱·  ← 坝在这里 (低)
          │  ·      弱信号就能溢出
          │·
          └──────────────
```

## 三者联动的完整场景

```
时刻 0:
  元单元静息中。
  pre_trace = 0, post_trace = 0
  calcium = 0.03 (正常)
  阈值 = 0.003 (正常)
  电容器 = 0V

时刻 1: 信号到来 (强度 0.5)
  pre_trace 跳到 1.0 → "收到了!"
  电源轨提供电压 → 电流注入电容器
  电容器电压上升到 0.4V
  场效应管: 0.4 > 阈值 0.003 → 线性区
  输出 = gm × (0.4 - 0.003) = 0.397
  钙更新: calcium += (0.397 - 0.03) / 20 = 0.048
  → calcium 从 0.03 升到 0.048

时刻 2: 没有新信号
  pre_trace 衰减: 1.0 × 0.95 = 0.95
  电容器漏电: 0.4 × e^(-1/1.0) = 0.147V
  场效应管: 0.147 > 0.003 → 输出 = 0.144
  钙更新: calcium = 0.048 + (0.144 - 0.048)/20 = 0.053

时刻 3-10: 持续没有信号
  pre_trace: 0.95 → 0.6 → 0.36 → ...慢慢消退
  电容器: 0.147 → 0.054 → 0.020 → ... 慢慢漏光
  场效应管: 输出越来越小
  钙: 慢慢降回 0.03 附近

时刻 50: 钙持续偏高
  calcium = 0.05 > target 0.03
  → 钙传感器: "太活跃了!"
  → 阈值从 0.003 上调到 0.004
  → 下次同样强度的信号, 输出会稍弱一点
  → 长期效果: 活跃度回到 0.03 附近
```

## 参数为什么要这样设

| 参数 | 值 | 由什么决定 |
|------|-----|----------|
| trace_tau | 20 | "多久算因果?" — 20 tick ≈ 人脑 STDP 窗口 (~40ms) |
| calcium_tau | 自动=1/target_rate | "多久测一次平均?" — 刚好覆盖一个完整激活周期 |
| target_rate | 0.03 | "应该多活跃?" — 皮层神经元平均发放率 ~1-5Hz |
| v_threshold | 初始≈target×0.5 | "多少信号够用?" — 低于平均激活的一半时就该响应 |
| gm | 1.0 | "响应多强?" — 线性比例, 超过阈值多少就输出多少 |
| n_slope | 1.5 | "亚阈值漏多少?" — 物理常数, 半导体工艺决定 |
| v_thermal | 0.026 | "热噪声多大?" — 室温 kT/q ≈ 26mV, 这是物理常数 |
