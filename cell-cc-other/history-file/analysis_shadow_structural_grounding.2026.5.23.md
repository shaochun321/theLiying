# 影子层的结构化落地 — 对纯数学抽象的修正

## 你指出的问题

当前 `shadow_sandbox.py` 的实现是一个**浮点距离矩阵**——纯数字。这违反了项目的基本原则：

> **结构计算 + 物理计算**

影子层应该和主系统一样，由**元件**构成，有**能量**、有**耗散**、有**拓扑**。数字收缩不是收缩——只有当真实的链路被改变、活跃度被迁移、能量被消耗时，才是收缩。

---

## 1. 影子层由什么构成？

### 1.1 不是抽象度量空间，是真实的神经元+束

影子层应该是一组**真实的 Neuron 和 SynapticBundle 实例**——和 encoding/column/motor 层完全相同的元件类型。

```
主系统:                          影子层:
  enc_reg_yaw  ─bundle─→ col_yaw     shadow_0 ─bundle─→ shadow_6
  enc_irr_yaw  ─bundle─→ col_yaw     shadow_1 ─bundle─→ shadow_6
  enc_reg_pitch ─bundle─→ col_pitch   shadow_2 ─bundle─→ shadow_7
  ...                                  ...
```

影子层是主系统的**稀疏投影副本**：
- 每个主系统的 encoding/column/motor 神经元 → 影子层中有一个对应的 shadow neuron
- 影子神经元之间通过 shadow bundles 连接
- 初始拓扑 ≈ 主系统拓扑的副本

### 1.2 影子神经元的特殊配置

影子神经元不是主系统的精确复制——它们有不同的时间常数：

| 参数 | 主系统 | 影子层 | 理由 |
|------|--------|--------|------|
| τ_RC | 5-10 ms | 50-100 ms | 慢 10× — 影子在慢时间尺度上运作 |
| 能量初始 | 1.0 | 0.5 | 影子层能量预算更低 |
| spiking | 可选 | 否 | 影子不需要离散脉冲，只需要连续激活 |
| VR base_rate | 0.001 | 0.0005 | 更低的基础代谢 |

### 1.3 影子层的输入

影子层的输入不是外部世界的感觉信号，而是**主系统的 Xin 张力**。

$$\text{shadow\_input}_i = \xi_i^{\text{main}} \quad \text{(主系统 bundle } i \text{ 的预测残差)}$$

这就是"被主神经系统驱动"的具体含义：影子层**不直接看到外部世界**，只看到主系统的**内部预测误差**——它是主系统的"内省镜像"。

## 2. "收缩"到底是什么？

### 2.1 不是距离收缩，是活跃度迁移 + 链路重塑

当两个影子神经元的 Xin 模式相关时，发生的不是"它们之间的浮点距离变小"，而是：

**活跃度迁移**：
```
t=0:   shadow_yaw_enc: activation = 0.3
       shadow_pitch_enc: activation = 0.3
       它们各自独立地处理各自的 Xin 输入

t=N:   shadow_yaw_enc: activation = 0.5  ← 接管了部分 pitch 的表征
       shadow_pitch_enc: activation = 0.1  ← 活跃度"迁移"走了
       shadow_yaw→pitch bundle: weight = 0.4 ← 新长出的跨轴链接
```

这就是你说的"活跃程度被迁移到不同的神经元"。

**链路重塑**：
```
t=0:   shadow_yaw_enc → shadow_yaw_col  (w=0.2, 轴内链路)
       shadow_pitch_enc → shadow_pitch_col (w=0.2)
       shadow_yaw_enc → shadow_pitch_col (w=0.001, 几乎不存在)

t=N:   shadow_yaw_enc → shadow_yaw_col  (w=0.15, 弱化)
       shadow_pitch_enc → shadow_pitch_col (w=0.05, 大幅弱化)
       shadow_yaw_enc → shadow_pitch_col (w=0.18, 跨轴链路增强!)
```

跨轴链路增强 = "两个节点在影子中靠近"的结构化表达。

### 2.2 收缩的生物学对应

| 你的描述 | 生物学对应 | 机制 |
|---------|-----------|------|
| "活跃程度被迁移" | **海马重放**（hippocampal replay） | 睡眠/静息时，海马将经验模式"重放"到皮层，皮层神经元接管表征 |
| "改变神经链路" | **突触重塑**（synaptic remodeling） | 长时程增强(LTP)/抑制(LTD)改变连接强度 |
| "耗费很多能量" | **蛋白质合成依赖的可塑性** | 持久的突触变化需要新蛋白质合成 → ATP 消耗 |
| "不可能无限收缩" | **突触饱和** + **能量限制** | 突触权重有上限；每次重塑消耗 E_remodel |

### 2.3 收缩的能量成本

每次链路重塑（bundle 权重变化 $|\Delta w| > \Delta w_{\min}$）消耗能量：

$$E_{\text{remodel}} = \kappa \cdot |\Delta w| \cdot |\text{pre}| \cdot |\text{post}|$$

这从影子神经元的 `energy` 中扣除。当能量耗尽 → 无法继续重塑 → **收缩自然停止**。

这就是为什么收缩不能无限——不是因为数学上设了 $d_{\min}$ 下界，而是因为**能量用完了**。

## 3. "主系统"这个名字对吗？

### 3.1 你的直觉

> "不是主，而是交界层？处于外部内部信息的汇聚区域？"

这个直觉非常准确。在真实神经系统中：

| 区域 | 功能 | 对应 |
|------|------|------|
| **初级感觉皮层** | 直接接收感觉输入 | 前庭链（MET→HC→Aff） |
| **联合皮层** | 多感觉汇聚 | Encoding + Column |
| **前额叶/默认网络** | 内部模型、自我参照 | 影子层 |
| **丘脑** | 中继 + 门控 | bundle routing |

你说的"交界层"可能对应的是**联合皮层**（association cortex）——它既接收外部感觉信号，又接收内部预测信号，是两者的**汇聚区**。

### 3.2 建议的命名

| 当前 | 修正 | 理由 |
|------|------|------|
| "主系统" | **感觉-运动层**（Sensorimotor Layer）或**交界层** | 它处理外部输入并产生运动输出 |
| "影子层" | **内省层**（Introspective Layer）或保持"影子层" | 它只看到内部预测误差 |

不过在代码中，"主系统"这个名字已经深入使用。建议在文档中注释，但代码中不改名（避免大规模重构）。

## 4. 收缩路径记录

### 4.1 你的问题

> "还可以有一个机制来记录收缩路径？一些被修改的、弃置的路径？"

这在生物学中确实存在：

**静默突触**（Silent Synapses）：
- 突触在结构上存在（树突棘仍然在），但功能上失活（AMPA 受体被内吞）
- 它们是"弃置的路径"的物理残留
- 在适当条件下可以被**重新激活**（AMPA 受体重新插入）

**突触标记**（Synaptic Tagging, Frey & Morris 1997）：
- 弱刺激在突触上留下一个"标签"（tag），持续 ~2 小时
- 如果在此期间有强刺激触发蛋白质合成 → 标签被"捕获" → 弱突触也被巩固
- 标签 = 路径记录的分子实现

### 4.2 在项目中的实现

收缩路径记录可以用**影子层自己的下葬系统**（和主系统 §9 完全相同的机制）：

```
影子层链路重塑:
  shadow_yaw→pitch 增强 (w: 0.001 → 0.18)
  shadow_pitch→pitch 弱化 (w: 0.2 → 0.05)
  
  当 w < w_prune 时:
    下葬到影子层的 shadow_store:
      {
        bundle_id: "shadow_pitch_to_pitch",
        weights_at_death: [0.05],
        xin_at_death: -0.12,
        energy_at_death: 0.3,
        tag: "remodel_tick_5000",  ← 重塑时刻标记
      }
    
    这就是"弃置路径的记录"
    
  衰减:
    shadow_store 中的记录 energy *= 0.999 每步
    
  重新激活:
    如果新的 Xin 模式与 shadow_store 中某个记录的相关性 > 0.9:
      → 旧路径被"唤醒"（重新创建 bundle，初始权重 = 记录值）
      → 这就是"静默突触被重新激活"
```

**关键**：影子层的 shadow_store 和主系统的 shadow_store 是**两套独立的下葬系统**。影子层自己也有记忆——"关于我是如何变化的记忆"。

这形成了一个递归结构：
```
主系统 → 影子层记录主系统的弃置路径
影子层 → 影子层的 shadow_store 记录影子层自己的弃置路径
```

这可能就是你一直在想的"收缩路径记录"——不是一个新机制，而是**同一套下葬系统的递归应用**。

## 5. 对当前实现的修正方向

当前 `shadow_sandbox.py` 需要从**浮点距离矩阵**重构为**真实元件网络**：

| 当前 | 修正为 |
|------|--------|
| `ShadowNode`（只存 Xin 历史） | **真实的 `Neuron` 实例**（有膜电位、能量、RC 电路） |
| `d_shadow[i][j]` 浮点矩阵 | **真实的 `SynapticBundle` 实例**（有 STDP、权重、memristor） |
| `eta * corr` 数学公式 | **STDP 学习**（Xin 作为输入驱动影子 bundle 的权重变化） |
| `epsilon_0 / d_s^2` 排斥 | **能量消耗**（每次权重变化扣除 E_remodel） |
| 无 | **影子层的 ECM**（有温度、PNN、能量预算） |
| 无 | **影子层的下葬系统**（记录弃置路径） |

> [!IMPORTANT]
> 这意味着影子层的代码量会显著增加——它不再是一个 350 行的观测器，而是一个完整的**迷你 HebbianCircuit**，有自己的 step()、learn()、energy accounting。
>
> 估计代码量：从 350 行 → ~800-1000 行。
> 但架构上更简洁：因为可以直接继承/复用现有的 Neuron、Bundle、ECM 组件。
