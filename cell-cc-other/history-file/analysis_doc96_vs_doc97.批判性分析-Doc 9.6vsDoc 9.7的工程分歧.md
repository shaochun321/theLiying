## 逐点验证

### 1. Doc 9.7 的 `max(0, da_gain-1) * 20` 方案

**数学推演**（当前 gain_factor 公式）:
```
gain_factor = 1 + α_gain × (conc - baseline)
            = 1 + 0.3 × (conc - 0.1)
```

| DA conc | gain_factor | max(0, gain-1) × 20 | 每步 ΔV (C=0.2, dt=0.001) |
|---|---|---|---|
| 0.00 | 0.970 | 0 × 20 = 0.00 | 0 |
| 0.05 | 0.985 | 0 × 20 = 0.00 | 0 |
| 0.10 | 1.000 | 0 × 20 = 0.00 | 0 |
| 0.20 | 1.030 | 0.03 × 20 = 0.60 | 0.60 × 0.6 × 0.001 / 0.2 = **0.0018** |
| 0.50 | 1.120 | 0.12 × 20 = 2.40 | 2.40 × 0.6 × 0.001 / 0.2 = **0.0072** |
| 0.80 | 1.210 | 0.21 × 20 = 4.20 | 4.20 × 0.6 × 0.001 / 0.2 = **0.0126** |
| 1.00 | 1.270 | 0.27 × 20 = 5.40 | 5.40 × 0.6 × 0.001 / 0.2 = **0.0162** |

**分析**:
- ✅ U 型**确实被消灭**——DA < baseline 时电流 = 0（没有负注入）
- ✅ DA 0.1→1.0: 电流单调递增
- ⚠️ DA 0→0.1: **完全无响应**——一个 10% 宽的死区
- ⚠️ 0.0018V/step 到 0.0162V/step 的范围**可能仍然太窄**
  - 但 V2 实验中 50.0 × 0.12 = 6.0 已经把 Column 推到饱和
  - 20.0 × 0.12 = 2.4 → ΔV ≈ 0.007/step → 累积到 V_ss ≈ 2.4 × 5 = 12V（会饱和但更慢）
  - 配合 v_peak 从 0.15→0.25 可能获得足够的线性区间

**潜在问题**: 加法注入仍然在与突触驱动**竞争**——Column 的工作点由 enc→col 决定。DA 只是在上面叠加偏移。如果 enc→col 驱动把 Column 推到 v_peak 附近，DA 的额外推力无处可去（headroom 用尽）。

### 2. Doc 9.6 的乘法 synapse_gain 方案

**概念验证**:
```python
# 当前 enc→col propagate 流程:
current = memristor.conduct(a_src) * synapse_gain  # synapse_gain = 3.0
# 改为:
current = memristor.conduct(a_src) * synapse_gain * dopamine.gain_factor()
```

| DA conc | gain_factor | effective_gain (base=3.0) | 相对倍数 |
|---|---|---|---|
| 0.00 | 0.970 | 2.91 | 0.97× |
| 0.10 | 1.000 | 3.00 | 1.00× |
| 0.50 | 1.120 | 3.36 | 1.12× |
| 1.00 | 1.270 | 3.81 | 1.27× |

**分析**:
- ✅ **全范围连续调制**——没有死区，没有截断
- ✅ DA 不注入额外电流，而是缩放已有电流 → 不会与突触驱动竞争
- ✅ Column 工作点由 enc→col 信号决定 → DA 只改"音量"
- ✅ 能量路径合法: 放大的电流仍然走 `inject()` 通道，Noether 全程覆盖

**潜在问题**:
- ⚠️ 调制幅度仅 ±30%（alpha_gain=0.3），可能太弱
  - 解决: 提高 alpha_gain 到 1.0-2.0
- ⚠️ Doc 9.3 枪毙的是**乘法修改膜电荷** (`charge = v × gain × C`)
  - Doc 9.6 提出的是**乘法缩放突触增益**（不同操作！）
  - synapse_gain 乘法发生在 `inject()` **之前**，不绕过 Capacitor
  - **不违反 Doc 9.3 的禁令** ← 关键判断点

### 3. 两份文档的共识

两份文档**完全一致**的点:
1. ❌ U 型曲线必须消灭
2. ✅ 解除 xin_to_da 的 weight_max=1.0 钳位
3. ✅ DA 自然浓度 0.027 太低，必须修复信号源

| 指令 | 准确性 | 备注 |
|---|---|---|
| `max(0, gain-1) * 20` 二极管 | **正确但粗暴** | 消除 U 型，但留下 10% 死区 |
| 乘数 50→20 | ✅ **正确** | 50 导致饱和，20 是合理退火 |
| v_peak 0.15→0.25 | ✅ **正确** | Column headroom 确实不够 |

## 关键判断：Doc 9.3 的禁令适用范围

Doc 9.3 枪毙的是：
```python
# 直接修改积分状态变量（电荷）→ 指数爆炸 → Noether 违规
col._membrane.charge = v * da_gain * col._membrane.capacitance
```

Doc 9.6 提出的是：
```python
# 在 propagate() 中缩放突触增益 → 电流通过正常 inject() 入膜
current = memristor.conduct(a_src) * synapse_gain * gain_factor
# 然后 tgt.step(current, dt) → membrane.inject(current, dt) → 正常路径
```

**这两者完全不同**。Doc 9.6 的方案不修改任何积分状态变量，不绕过 `inject()`，不违反 KCL。
Doc 9.3 的禁令**不适用于** synapse_gain 乘法。

## 综合建议

> [!IMPORTANT]
> 两份文档都有准确和不准确的部分。建议**综合执行**:

**采纳 Doc 9.6 的核心方案（乘法 synapse_gain）**:
- 删除 variant_adapter.py 中的 `da_current = ... * 50.0` 加法注入
- 在 enc→col 和 col→motor 的 `propagate()` 中乘上 `gain_factor`
- 提高 `alpha_gain` 到 ~1.0 以获得足够调制范围

**采纳 Doc 9.7 的辅助修复**:
- ✅ 解除 weight_max 钳位 (shadow: 5.0, xin: 10.0)
- ✅ v_peak 提升到 0.25 (拓宽 headroom，即使乘法调制也有好处)
- ✅ D2R 负电压不修 (物理涌现)
- ❌ 不需要 `max(0,…)` 二极管 (乘法模式下不存在负电流)

**不采纳**:
- ❌ Doc 9.7 的 "保留加法注入 + 二极管" — 被 9.6 的更优方案取代
- ❌ Doc 9.6 的 "给 DA 神经元加 bias_current" — D2R 自受体是合法物理涌现
- ⚠️ Doc 9.7 的 "数值气蚀" 解释 — dt/τ 远不到不稳定区

> [!WARNING]
> 最终决策权在您。核心分歧是：**加法注入 + 截断** vs **乘法 synapse_gain 调制**。
> 两者都能消灭 U 型。乘法在物理上更干净，但改动稍大（需在 bundle.propagate 中传入 gain_factor）。
> 加法 + 截断更简单（3 行改动），但保留了"竞争"问题和 10% 死区。
