## 根因分析

### 为什么能量平衡是负的？
1. **P_out 较高**：motor_mean=0.549（远高于 C6 阈值 0.01），高运动量大量消耗能量
2. **P_in 不足**：max_deposit=0.05/步的上限太低，即使在热源旁边也无法快速补充
3. **温觉趋性不稳**：STDP 振荡导致导航效率低下，有效进食时间减少

### 能量平衡不足的数学
```
P_in_needed > P_out
P_in_eff = max_deposit × (时间在热源内)
         = 0.05 × 0.51 = 0.0255/步

P_out_eff ≈ basal_drain × level + muscle_cost
          ≈ 0.0001 × 192 + muscle ≈ 0.019 + 0.0027 ≈ 0.0167/步

赤字 = 0.0255 - 0.0167 = +0.0088/步 (应该是正的?)
```

**重新检查**：理论上 P_in 应该是正的。实测 P_net=-0.00069 说明实际 在热源区时间 < 51.4% 或 max_deposit 未总是饱和。

**修正估算**（基于 fill_mean=0.19 时 basal_drain）：
- 若 basal_drain 是 proportional to fill_fraction：0.0001 × 1000 × 0.19 = 0.019/步
- P_in_eff ≈ 0.016/步（从实测）
- P_net = 0.016 - 0.017 = -0.001/步 ✅ 与实测吻合

## 修复方案

### 方案 A：增加 max_deposit_per_step（推荐）

将 `max_deposit_per_step` 从 0.05 → 0.08：
- 新 P_in_eff = 0.016 × (0.08/0.05) = **0.0256/步**
- P_out_eff ≈ 0.0167/步
- 新 P_net ≈ **+0.009/步** ✅

预测效果：
- fill_mean 从 0.19 上升至约 0.40-0.45（正能量积累）
- fill 不再归零（C1 PASS）
- fill@500k 约 0.40+（C2 PASS）
- P_net > 0（C7 PASS）

### 方案 B：减少 basal_drain（保守）

将 `basal_drain` 从 0.0001 → 0.00007：
- P_out_eff 降低 30%
- 效果较温和，但可能不够

### 推荐：方案 A + 验证
最小化干预：仅修改一个参数 (max_deposit_per_step: 0.05 → 0.08)。
