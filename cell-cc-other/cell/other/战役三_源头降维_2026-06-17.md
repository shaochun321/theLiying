## 一、病理诊断

### 战役三：XIN_GAIN 虚假放大器

**现象**：影子层 Shadow Enc 神经元接收到极高输入电流，导致积分饱和。

**数学诊断（文档 §2.3）**：

前庭链天然物理误差 |Ξ| 已高达 15~30（Langevin 噪声 + 机械信号）。  
原 `XIN_GAIN = 3.0` 将此信号放大三倍再注入影子层：

$$V_{ss}^* \approx \frac{3.0 \times 30}{g_{leak} + 0.01 + \text{弱反馈}} \gg 150 \text{（归一化单位）}$$

影子层钙积分器瞬间撞死在天花板，下游 DA 失去对温度梯度的编码权。

**根本问题**：原始 XIN_GAIN=3.0 的注释说"Without gain, dV too small for STDP"——这是一个错误假设。前庭链 |Ξ| 本身已足够大，人工放大器是多余的。

### 影子归一：vr_base_rate 过小

Shadow 神经元的 VoltageRegulator `vr_base_rate=0.01` 对应除法归一化分母 `r_base` 极弱。  
稳态方程（文档 §2.2）：

$$V_{ss}^* = \frac{\beta_{gain} \sum \Xi_{raw}}{g_{leak} + r_{base} + \alpha V_{ss}^*}$$

`r_base=0.01` 使分母薄弱，无法压制高输入时的积分饱和。

## 二、修复方案

### 代码改动

**文件**：`nexus_v1/components/shadow_sandbox.py`

**变更1：XIN_GAIN 废除人工放大器**
```python
# 修改前
XIN_GAIN = 3.0  # 原注释：Without gain, too small for STDP

# 修改后
XIN_GAIN = 1.0  # 1:1 物理映射，废除人工放大器
                # REF: 大一统方案 §4.4
```

**变更2：vr_base_rate 增强除法归一化分母**
```python
# 修改前（encoding 和 column 层均为）
"vr_base_rate": 0.01,

# 修改后
"vr_base_rate": 0.05,   # 除法归一化分母×5 → 稳态 V_ss 压制饱和
                         # REF: 大一统方案 §2.2 + §5.1 影子归一
```

### 验算（文档 §6.1）

修复后影子层稳态（XIN_GAIN=1.0, r_base=0.05）：

$$V_{ss}^{new} \approx \frac{1.0 \times 30}{g_{leak} + 0.05 + \alpha V_{ss}} \approx \frac{30}{0.15} = 200 \text{（归一化单位）}$$

相比修复前 >150 mV 等效值，新稳态被除法归一化压制在合理区间。
