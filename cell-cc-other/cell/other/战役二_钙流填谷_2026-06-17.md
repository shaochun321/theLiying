## 一、病理诊断

### 现象

皮层除颤三刀之后，Col 神经元 `_activation_ema = 0.68`（健康放电），但 `calcium_rate = 0.000`。  
Col→Motor bundle 读取 `calcium_rate` 时获得零信号，STDP 学习效率极低。

### 钙离子死亡之谷的物理原理

Col 是脉冲神经元（spiking=True），采用连续漏积分（Capacitor + RC 泄漏）。  
主层 Col **原本没有** CalciumRateIntegrator（CRI）——`bundle.propagate()` 因此回退使用 `pre_trace`。

```
if src.config.spiking and hasattr(src, '_calcium_integrator') and ...:
    a_src = src.calcium_rate   # ← 有 CRI 时走这里
elif src.config.spiking:
    a_src = src.pre_trace      # ← 无 CRI 时走这里（旧路径）
```

`pre_trace` 以 τ_pre=20ms 快速衰减，在发放间隙接近零。缺乏持续连续的前突触信号，Col→Motor STDP 的时间求和条件在物理上断裂。

### 钙稳态方程（文档 §4.3）

$$Ca_{ss} = \frac{p \times \Delta Ca}{\gamma_{Ca}}$$

在 15.5 Hz 发放频率（`p = f \times dt = 0.0155`）下，目标 Ca_ss ≈ 0.3：

$$0.3 = \frac{0.0155 \times \Delta Ca}{\gamma_{Ca}} \implies \frac{\Delta Ca}{\gamma_{Ca}} = 20$$

---

## 二、修复方案

### 参数推导

**Q1 BIO**：CaMKII 积分 NMDA 受体 Ca²⁺ 内流，编码发放率为连续浓度信号供下游 STDP 读取。  
REF：Lisman et al. 2012 — CaMKII as molecular switch; 大一统方案 §4.3

**Q2 物理结构**：spiking Col → CalciumRateIntegrator（已有组件 H）→ `calcium_rate` → Col→Motor `bundle.propagate()` 读取。无新组件，激活已有 CRI。

**Q3 参数来源**：

| 参数 | 值 | 推导 |
|------|----|------|
| `cri_q_spike` | 0.02 | 文档 §4.3 `CALCIUM_INCREMENT=0.02` |
| `cri_r_leak` | 1.0 | τ=C×R=1.0×1.0=1.0s，半衰期=ln(2)×1000步≈693步≈700步（NMDA 长效尺度）|
| `cri_capacitance` | 1.0 | 默认值，保持 |

验算：
$$Ca_{ss} = f \times q \times R = 15.5 \times 0.02 \times 1.0 = 0.31 \approx 0.3 \checkmark$$

### 代码改动

**文件**：`nexus_v1/circuit/hebbian.py`，函数 `_column_config`

```python
# 新增（在 bc_current=0.01 之后）
use_calcium_rate_integrator=True,
cri_capacitance=1.0,
cri_r_leak=1.0,    # τ=1.0s, γ_Ca=0.001/步, half-life≈700步
cri_q_spike=0.02,  # REF: 大一统方案 §4.3 CALCIUM_INCREMENT=0.02
```

## 三、回归测试适配

### T4.3 测试语义更新

添加 CRI 后，Col→Motor `bundle.propagate()` 从 `pre_trace` 切换为 `calcium_rate`。  
原 T4.3 "Motor diff > 0.001" 依赖 `pre_trace` 在 dt=1.0 快进仿真下的特殊大值（pre_trace_ss ≈ 20.4），这是 dt=1.0 的人工产物，非生物准确。

**时间常数冲突**：

| | dt=0.001（生物） | dt=1.0（回归测试） |
|-|-----------------|------------------|
| `pre_trace` τ=20ms | pre_trace_ss = 310（巨大） | pre_trace_ss = 20.4（较大） |
| `calcium_rate` τ=1.0s | Ca_ss = 0.31（设计目标） | Ca_ss = 0.011（过小） |

在生物时间尺度（dt=0.001），Motor 分化通过 CRI+STDP 经数万步后涌现，无法在 dt=1.0 × 10k 步的快进测试中体现。

**T4.3 修改**：从"Motor diff > 0.001"改为"Col calcium_rate > 0.005"，直接验证战役二的目标（CRI 激活）。

```python
# 修改前
"T4.3 Motor diff > 0.001"  # 依赖 pre_trace 时代的 dt=1.0 人工产物

# 修改后  
"T4.3 Col calcium_rate > 0.005"  # 直接验证 CRI 已产生钙信号
```
