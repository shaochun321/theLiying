### 二、核心诊断：神经解剖学错位

V8引擎验证报告揭示了结构盲区：

**现有拓扑**：
```
Thermal Col → cross-axis bundle (gain=0.7, w_max=0.15) → Motor
```

**正确拓扑**：
```
Thermal Col → axis-specific bundle (gain=10.0, w_max=0.5) → Motor
```

热源皮层（Thermal Columns）是热梯度信号的**第一站**。在现有拓扑中，它们没有轴特异束连接到Motor，所有热信号被迫通过低增益、低权重的cross-axis束到达运动神经元：

- Col[therm_front]钙信号达到0.38
- 驱动电流 = 0.38 × 0.7 × 0.15 = 0.04（≈零）
- 而V8引擎真正的增益（axis-specific, gain=10.0）从未作用于热信号

这解释了200k步热趋性Δx=0.27无法突破5.0的终极拓扑原因。


### 三、执行内容

#### [Phase 3] Thermal Col → Motor 轴特异束

**目标**：为热梯度Xin开辟高增益结构容置路径，使预测误差能直接驱动物理位移。

**执行位置**：`hebbian.py`，`_build_col_to_motor_bundles`中，在现有axis映射基础上追加thermal映射。

```python
# 新增thermal→motor轴特异束（与半规管通路同构）
thermal_motor_map = [
    (['therm_front', 'therm_back'], 'move_x'),   # 前后温差→纵向运动
    (['therm_left', 'therm_right'], 'move_y'),   # 左右温差→横向运动
]
for therm_axes, mot_name in thermal_motor_map:
    sources = [self.column_neurons[ax] for ax in therm_axes]
    b_therm = SynapticBundle(
        config=BundleConfig(
            bundle_id=f"therm_{'_'.join(therm_axes)}_to_{mot_name}",
            learning_rule="stdp",
            initial_weight=0.1,           # 低初始权重，让STDP自己学
            weight_max=0.5,
            stdp_lr=0.05,                 # 战役四确立的高可塑性
            synapse_gain=10.0,            # V8增益（与半规管通路一致）
            bundle_role="feedforward",
            coupler_capacitance=1.0,
            coupler_r_leak=2.0,
            coupler_adapt_vth=0.2,
            coupler_adapt_gm=2.0,
            coupler_blayer_c_slow=100.0,
            coupler_blayer_r_slow=10.0,
            coupler_blayer_gm=0.01,
            coupler_blayer_k=2.0,
            decay_rate_by_stage=(0.5, 0.1, 0.01),
        ),
        sources=sources,
        targets=[self.motor_neurons[mot_name]],
    )
    self.bundles_col_to_motor.append(b_therm)
```

**同步操作**：确认thermal columns已在`all_axes`中定义（`therm_front/back/left/right`），确保映射有效。


#### [Phase 0] Memristor 物理防呆钳位

**目标**：从半导体底层防止数值溢出导致的负电阻和无限电导。

**执行位置**：`semiconductor.py`，`resistance`和`conductance`属性。

```python
@property
def resistance(self) -> float:
    w_safe = max(0.0, min(1.0, self.w))
    return self.r_min + (self.r_max - self.r_min) * (1.0 - w_safe)

@property
def conductance(self) -> float:
    resistance_val = self.resistance
    g = 1.0 / max(resistance_val, self.r_min)
    return min(g, 1.0 / self.r_min)
```

**两重保护**：
- `w_safe`防止权重溢出导致负电阻
- `conductance`上限防止导纳爆炸

#### 不执行的内容

- **Phase 2 Klinotaxis**：dT>0→加速，dT<0→转向的硬编码逻辑，违反结构构建的物理计算原则
- **放弃现有皮层STDP成果**：Langevin、CRI、DA净空锁、六刀参数，全部保留
- **上调cross-axis增益**：保持gain=0.7，维持自由度隔离


### 四、预期物理结果

| 阶段 | 时间窗口 | 物理机制 | 预期Δx |
|------|----------|----------|--------|
| 反射驱动 | 0-50k步 | Thermal→Motor高增益通路提供直接热梯度驱动 | 快速积累 |

### 六、与V8引擎的关系

| 通路 | 源 | 目标 | 增益 | 产生的Xin类型 |
|------|----|----|------|-------------|
| V8通路 | Col[yaw] | Motor[move_x] | 10.0 | 旋转-平移耦合预测误差 |
| Phase 3通路 | Col[therm_front/back] | Motor[move_x] | 10.0 | 热梯度-运动方向预测误差 |

两种通路是**并行结构，互不冲突**。它们向同一个T（拓扑结构）提供不同类型的Xin，各自驱动结构容置的不同面向。多轴热梯度与半规管噪声共同向Motor提供驱动电流，比单一路径更鲁棒。


### 七、执行的物理依据（与TOPRXin的对应）

这份执行清单并非简单的参数调整，而是在TOPRXin框架下的**结构容置**：

- **Xin来源**：热梯度预测误差（Thermal Col激活）长期存在但无法到达Motor（低增益cross通路）
- **T迭代**：建立Thermal→Motor高增益轴特异束，为这种Xin提供结构容置路径
- **递归循环**：行为产生新的Xin（运动-热预测误差），推动T进一步迭代

**Phase 0是硬件保护，Phase 3是拓扑修补。两者都不违反TOPRXin原则。** 它们是让现有物理底座正常工作的必要结构修正，而非语义层面的行为编程。
