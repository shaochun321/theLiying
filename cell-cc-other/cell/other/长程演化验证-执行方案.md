### 目标
在 **100k-200k 步** 的闭环仿真中，验证四阶段完成后的系统整体行为：
1. Δx 是否突破历史阈值
2. 权重剪刀差（Δw）是否持续为正
3. 代谢约束（fill）是否在有效范围内
4. AGC 是否在能量低迷期介入
5. 方向偏置是否持续正确

### 场景参数
与 EXP-020 保持一致（便于对比）：
- 热源：x=70，半径=20
- 起始：x=50
- 仅 oto_x 机械输入（单轴趋热）
- 步数：100k 步

### 关键指标（每 10k 步记录）
| 指标 | 含义 | 预期 |
|------|------|------|
| x_position | 物理位移 | 向热源移动（Δx>0）|
| fill_fraction | 能量状态 | 不应在 50k 步前归零 |
| Δw = w_front - w_back | 剪刀差 | 保持 >0 |
| AGC_gain | 增益输出 | 能量低时上升 |
| DA_concentration | 价值信号 | 适时出现 |

### 判断标准
| 结果 | 判定 |
|------|------|
| Δx > 0，Δw > 0，fill 在 100k 步内未归零 | ✅ 系统行为稳定 |
| Δx < 0 或 Δw < 0 | ⚠️ 某一机制失效 |
| fill 提前归零 | ⚠️ 代谢约束不足 |

### 预期对比
| 阶段 | Δx (100k) | Δw (100k) | fill (100k) |
|------|-----------|-----------|-------------|
| Phase 3 联合验证（EXP-020）| -41.9 | +0.070 | ~0.121 |
| Phase 4 AGC（预期）| >0 | >0 | >0.1 |

准备运行长程验证脚本。基于现有 `diag_agc_validation.py` 和 `diag_integration_filter_phase3.py`，创建一个包含完整监控的长程演化脚本。

```bash
python -m nexus_v1.tests.diag_long_term_evolution
```

计划运行 100k 步，每 10k 步输出一次关键指标。
