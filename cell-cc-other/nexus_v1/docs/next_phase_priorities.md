## §1 方向学习的诚实限制

### 问题 1.1: 输入不对称 → 结果不对称

```
实验条件: oto_x=5.0, oto_y=1.0 (我们设的)
结果:     w(oto_x)/w(oto_y) = 1.49

如果 oto_x = oto_y = 3.0:
  ratio ≈ 1.0 (几乎必然)
  → "方向学习"消失

→ ratio 1.49 部分是输入不对称的反映，不是纯粹的涌现
```

**要验证**: 在对称输入下（oto_x = oto_y = oto_z = 3.0），
让闭环运动产生的自然不对称（身体向热源移动 → oto_x 方向温变更大）
是否能产生 ratio > 1.0。

**这是涌现的真正测试**: 不对称来自环境，不来自输入设计。

- **优先级**: ★★★★★
- **难度**: 中等（需要闭环运动产生足够的温度变化）
- **预期障碍**: 身体移动太慢 (0.003/step)，温变 dT 太小

---

### 问题 1.2: |绝对值| 阻止正负区分

```
Hebbian: dw = η × |a_src| × |a_tgt|

|a| 丢弃了符号:
  "向 +x 走时变暖" 和 "向 -x 走时变暖"
  在 |a| 下产生相同的权重更新

→ 系统能学到 "x 轴与热有关"
→ 但分不清 "+x 有热" 还是 "-x 有热"
```

**需要**: ON/OFF 双通路（暖觉/冷觉），或带符号的学习规则

- **优先级**: ★★★★
- **难度**: 中等（添加 COLD 编码通道）
- **生物依据**: 皮肤有暖觉(C-fiber)和冷觉(Aδ-fiber)两种受体

## §2 Motor 选择性的缺失

### 问题 2.1: 三个 Motor 完全同步

```
Motor_x : Motor_y : Motor_z = 33.3% : 33.3% : 33.3%

原因: 所有 Column 等权全连接到所有 Motor
     → 相同总电流 → 相同放电时间
     → STDP 无法区分 → 权重永远相等
```

**需要对称性破缺**:

1. **噪声驱动探索** (最小改动):
   给 Motor 加独立随机噪声 → 打破同步 → STDP 可区分

2. **三因子调制** (更正确):
   dT/dt > 0 时 DA 释放 → 选择性增强刚才的运动方向
   当前 DA 系统存在但耦合太弱

3. **拓扑涌现** (终极目标):
   全连接 + 噪声 + 三因子 → 训练后:
   w(col_oto_x → motor_x) >> w(col_oto_x → motor_y)
   = 拓扑从学习中涌现

- **优先级**: ★★★★
- **难度**: 高（需要噪声 + DA 强化学习的协调）

## §3 参数调整的本质

### 问题 3.1: 过多的手动调参

```
FIX-001~018 中:
  大量修复是 "参数 X → Y" 的形式
  gain: 10→3→0.1
  threshold: 0.3→0.2
  tau: 50→1.0
  
每次都是外部工程师（我们）在调
系统自己不会发现"gain=3.0 导致 PowerRail 崩溃"
```

**根本问题**: 系统没有元学习（learning-to-learn）能力。
它不能自动发现并修正参数不匹配。

**可能的方向**:
- R 相强化: 让 Homeostasis 更智能（不只是 threshold/gain）
- Noether 异常: A 递增 → 自动降低增益
- 代谢预算: 限制总能量消耗 → 迫使系统自己找到平衡

- **优先级**: ★★★
- **难度**: 高
- **风险**: 过度自动化可能导致不可调试

## §4 TOPR/Xin 的实效问题

### 问题 4.1: P 相在主电路中无实效

```
compute_xin() 每步运行 → xin_tension 在累积
但主电路的学习(STDP/BCM)完全不看 xin_tension

唯一使用者:
  - fruit lifecycle (boost 0.001 → 几乎无效)
  - shadow input (有效但间接)

→ P 相的直接影响 ≈ 0
```

**可能的方向**:
- 让 xin 调节 STDP 学习率: 高 tension → 高可塑性
- 让 xin 调节 PNN 成熟速度: 高 tension → 减缓冻结

- **优先级**: ★★★
- **难度**: 中等

---

### 问题 4.2: R 相碎片化

```
VR、Homeostasis、PNN、NDR 都是 R 相成员
但它们之间没有协调:
  - VR 不知道 Homeostasis 在做什么
  - PNN 不知道 VR 够不够
  - NDR 不知道 PNN 的成熟度

→ 各自为政，可能互相矛盾
```

**可能的方向**:
- 统一的 R 相调度器: 收集所有 O 数据 → 协调 R 决策
- 但这可能太"中央集权"，违背涌现原则

- **优先级**: ★★
- **难度**: 高

## §5 热趋性闭环的真正测试

### 问题 5.1: 身体移动太慢

```
50k 步: 身体移动 0.20 单位 (从 50.0 → 50.2)
热源距离 15 单位 → 只走了 1.3%
温度变化: 2.10 → 2.14 (ΔT = 0.04, 0.2%)

→ 信号太弱，学习几乎看不到温度变化的方向性
```

**需要**:
- 更强的 Muscle gain
- 或更近的热源
- 或更长的运行时间

- **优先级**: ★★★
- **难度**: 低（调参即可，但又是手动调参...）

---

### 问题 5.2: 热源消耗后的记忆测试

```
当前权重 w(oto_x, therm) = 0.0037 < silence_threshold = 0.005
→ 权重已经在休眠边缘
→ 热源消失后，Hebbian decay 会继续降低 → 可能直接归零

PNN 是否足够成熟来保护这些权重？
影子层的 silent synapse 快照机制能否保存方位记忆？
```

- **优先级**: ★★★★
- **难度**: 低（运行实验即可）
- **理论意义**: 高（测试"记忆"是什么）

## §6 项目不算突破的诚实原因

```
1. 每个单独的行为都能溯源到具体的规则或参数
   → 没有"整体 > 部分之和"的涌现

2. 方向学习 ratio 1.49 部分来自输入不对称
   → 不是纯粹从对称条件涌现的

3. 系统不会自己调参数
   → 需要外部工程师 (FIX-001~018)

4. TOPR/Xin 框架指导了构建但未指导行为
   → 元架构是设计工具，不是行为驱动力
```

**项目什么时候才算突破？**

当系统展示出**无法用各组件单独解释**的行为时：
- 例：在对称输入下，闭环运动产生方向偏好（涌现的方向性）
- 例：热源消失后，系统主动重新寻找热源（目标导向行为）
- 例：系统自己发现"gain 太高"并降低（元学习）

我们还没到那里。但框架在了，问题清晰了。
