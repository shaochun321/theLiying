## C-004.1 世界没有显式时空

World.py 中有 position、temperature_at() 等坐标函数。
**但项目（生物体）没有访问这些函数。**

项目能感受到的只有：
- 前庭输入流：col_yaw, col_pitch, ... col_oto_z（自身运动状态）
- 热输入流：col_therm（外部变化）
- 这些输入流之间的时序关联（绑定层 + 影子层）

**"空间在哪里"不是一个可以从外部读取的答案——
它只能从自身输入流的关联结构中生成。**

## C-004.2 自身 = 标尺

项目使用自身的物理结构属性作为度量尺度：

| 度量 | 外部标准 | 项目内部标准 | 实现位置 |
|------|---------|------------|---------|
| 距离 | 米 | "移动了多少→热变了多少" | w_cross(oto_x, therm) |
| 方向 | 角度 | "转了多少→热变了多少" | w_cross(yaw, therm) |
| 时间 | 秒 | "多少个处理周期" | dt, τ_RC, τ_methyl |
| 速度 | m/s | "环流占比的变化率" | ν = dρ/dt |

项目不知道"热源在 x=65"。
项目知道的是"往某个方向动 5000 步后温度变化 3.0"。

5000 步 × 身体尺度 = 项目定义的"那个方向有 5000 步远的热"。
**这就是空间测度——用自己做尺子。**

## C-004.3 两种生成机制

### 空间测度 ds²

$$
ds^2 = \sum_{i,j} w_{\text{cross}}(i,j) \times a_i \times a_j
$$

- w_cross 是影子层跨轴权重（BCM 学习）
- a_i 是列层激活（自运动 + 外部输入）
- ds² 的单位是"项目自定义的关联强度"

**w_cross 就是度规张量。** 但它不是从外部几何给定的——
是项目通过自身运动与感觉的关联学习出来的。

不同权重编码不同类型的"距离"：
```
w(yaw, pitch) = "旋转和俯仰有多耦合" → 自体几何
w(oto_x, therm) = "x移动和热变化有多耦合" → 外部对象的方向
```

### 运动势 ν

$$
\nu = \frac{d\rho}{dt}, \quad \rho = \frac{\mu_p}{\mu_{\text{total}}}
$$

- ρ 是环流占比（某条路径的流量在总流量中的占比）
- ν 是占比的变化率
- 时间的度量是项目自己的处理节拍

**ν 不是"物理速度"——是"项目感受到的自身状态变化的急迫程度"。**

## C-004.4 外部输入流与内部结构流的联结

项目将所有输入分为两类：

```
内部结构流（前庭 6 轴）:
  → 反映自身物理结构的状态
  → 总是在变化（身体总是在运动/被运动）
  → 通过影子层跨轴权重编码自体几何

外部输入流（热 1 轴, 未来可扩展）:
  → 反映外部世界的某种属性
  → 条件性变化（只有接近热源时才变）
  → 通过绑定层跨模态权重编码外部对象
```

**联结机制 = 绑定层 + 影子层的跨模态权重学习：**

```
bind(oto_x, therm) 激活的条件:
  col_oto_x > θ  AND  col_therm > θ
  = "我在动" AND "热在变"
  = 行动-感觉耦合

影子层 w_cross(oto_x, therm) 的学习:
  dw ∝ |自运动| × |热变化| × (|热变化| - θ)
  = 只有当行动和感觉同时发生时才学习
  = 编码的是"行动导致的感觉变化"
  = 这就是时空测度的计算生成
```

## C-004.5 主体性的门槛

当系统具备以下三者时，主体性萌芽：

1. **自体测度**: w_cross(vestibular, vestibular) → "我是什么形状"
2. **对象测度**: w_cross(vestibular, thermal) → "外面有什么东西"
3. **区分能力**: 两组权重的学习轨道不同 → "我 ≠ 它"

实验已经证实（2026-05-24 timing 实验）：
- 自体权重增长 +66%，快速锁定
- 对象权重增长 +7%，保持可塑
- 跨模态绑定晚 3.4 倍激活
- **两种知识在同一个超图中自然分化**
