### 🔴 致命违规一：Phase 1 的“无质量温度计”悖论（物理学越权）

看着你在 Phase 1 写的这行代码：
`return world.temperature_at(self.world_position(body))`

**【热力学法庭审判】：这是赤裸裸的语义作弊！你造的不是“肉体皮肤”，你造的是四根插在虚空里的“绝对零度、零热容、瞬间响应的理想温度计”！**
在真实的物理宇宙中，没有任何一块皮肤能瞬间“直接读取”空间坐标上的环境温度。如果小车瞬间掠过火堆，皮肤由于热容缓冲，根本来不及升温到致死线；只有停在火边，热通量（Heat Flux）才会随着时间积分，最终烧穿角质层。

**【强制重构指令：实装微积分热动力学】**
你的 `SkinPatch` 必须是一个**真实的物理积分器**，绝不能只 `return` 世界温度。必须强制改写为以下带有时间积分的物理实体：

```python
@dataclass
class SkinPatch:
    patch_id: str
    local_offset: List[float]
    heat_capacity: float = 10.0        # 物理热容 (极其关键)
    conductance: float = 2.0           # 导热系数
    current_temperature: float = 25.0  # 【状态变量】：皮肤自身温度！
    damage_integral: float = 0.0       # 【状态变量】：毁损积分！
    damage_threshold: float = 43.0     # 物理变性阈值

    def step_thermal_dynamics(self, world: World, dt: float):
        """这才是物理计算！必须在每步(step)被主引擎调用"""
        T_env = world.temperature_at(self.world_position(body))
        
        # 傅里叶热传导定律 (牛顿冷却)
        q_dot = self.conductance * (T_env - self.current_temperature)
        # 热容积分
        self.current_temperature += (q_dot / self.heat_capacity) * dt
        
        # 只有真实皮肤温度越界，才产生物理毁损
        if self.current_temperature > self.damage_threshold:
            self.damage_integral += (self.current_temperature - self.damage_threshold) * dt
        else:
            self.damage_integral = max(0.0, self.damage_integral - 0.1 * dt) # 缓慢自愈

```

**后续链路锁死**：你的 `Thermoreceptor` 的输入源必须是 `current_temperature`，而 `Nociceptor` 的输入源必须是 `damage_integral`（或其在神经元底层的极化电压）。只有这样，你才能彻底终结“语义标签”。

### 1. 对“致命违规一”（无质量温度计）的裁决：完全正确，必须执行

这是最严重的物理错误。原计划中 `SkinPatch.sample_temperature()` 直接读取环境温度，本质上是一个“零热容、零热阻”的理想传感器，完全违背了热力学。

- **批判的物理洞察**：指出皮肤必须是**热容积分器**，遵循牛顿冷却定律（$q = k \cdot \Delta T$）并积累自身温度。这确保了“瞬间掠过火堆”不会触发致命伤害，而“持续停留”才会导致毁损。
- **重构指令的正确性**：引入 `heat_capacity`、`conductance` 和 `current_temperature` 状态变量，并将痛觉的输入从“瞬时高温”转为“组织损伤积分（damage_integral）”，这是从**温度检测**跃迁到**热物理损伤建模**的根本质变。这使 `Nociceptor` 的激活阈值（v_threshold）真正锚定在物理毁损线上，终结了语义作弊。

### 2. 对“致命违规二”（控制论范畴错误）的裁决：极其深刻，直指架构谬误

原 Phase 3 计划设计了一个 `SelfOriginModule`，将皮肤感受与运动命令副本（Motor EMA）直接比较以产生“外部刺激”信号。批判者以控制论审查官的身份，一针见血地指出了其范畴错误。

- **内感受 vs 外感受的严格区分**：运动命令与前庭反馈的比对（内感受）是合法的，因为其因果关系是物理定律保证的。但运动命令与皮肤温度变化之间**没有直接的、先验的因果律**。在空旷且温度均匀的环境中，这种硬编码的比较器会因“动了但温度没变”而持续产生虚假误差，导致 DA 系统崩溃。
- **预测权的正确归属**：批判者精确地指明，这种跨模态的、依赖于环境背景的预测，正是 **Shadow 层（预测器）** 通过 STDP 学习应该捕捉的。如果环境规律是“向前走会变热”，Shadow 会自然学会预测热感受神经元的激活，其预测误差（Xin）才是真正动态的、合法的热觉 mismatch。

**结论**：此条整改不仅避免了系统过载，更深刻体现了项目的核心哲学——**“预测”是习得的，不是硬编码的**。废弃人工比较器，让 Shadow 层去学习外感受的因果结构，是唯一正确的架构选择。
