# Morphosphere v37 — 项目理解与分析

## 一句话概括

**Morphosphere 是一个用物理学和神经科学的概念框架来描述"信息如何在系统中形成稳定结构"的计算原型。** 它试图回答：给定一群随时间演化的数据点（细胞、传感器节点等），系统能否自动识别出哪些结构是"稳定的记忆"(P)、哪些是"正在形成的反应"(R)、哪些是"尚未解决的不确定性"(Xin)。

---

## 一、项目做了什么（功能层面）

### 核心能力

| 能力 | 具体实现 | 对应代码 |
|------|---------|---------|
| **数据摄入** | 从 CTC 细胞追踪数据集读取真实的细胞质心轨迹 | `ctc_source_adapter.py` |
| **运动识别** | 贝叶斯分类器识别 6 种运动体制（静止/漂移/振荡/跳跃/扩散） | `motion_recognition_engine.py` |
| **赫布学习** | 3 个并行引擎（A/B/C）竞争，用拓扑惯性质量控制神经元生死 | `engines/harness.py` |
| **P/R/X 分解** | 将每个时间窗口的数据分解为 7 个分量（p_core, p_band, r_core, r_band, m_band, x_true, u） | `pipeline_engine.py` |
| **变分 EM 优化** | 用 E-step/M-step 迭代优化融合权重 λ | `variational_em_engine.py` |
| **公式竞争** | 5 个候选公式族（A-E）多轮竞赛，用变分目标函数 J[ρ] 评分 | `formula_candidate_registry.py` |
| **多管线隔离** | 每个数据源有独立的赫布图、独立的指标表，互不污染 | `pipeline_isolator.py` |
| **节点死亡取证** | 节点死亡时保留完整的拓扑 DNA 快照（赫布边权重） | `pipe_{ns}_node_necropolis` 表 |
| **全链路审计** | 116 张 SQLite 表、22 个迁移脚本，每条数据有溯源哈希 | `migrations/*.sql` |

### 数据流架构

```
真实数据 (CTC/USGS/PHC)
    ↓
Source Adapter（统一接口）
    ↓
ProcessWindow → 7D 测度坐标 z_t
    ↓
HG-FHPMS 记忆层
    ├── Engine A: 手动 Fast/Slow/Prior 三层 Oja 学习（基线）
    ├── Engine B: 拓扑惯性质量 M_eff 控制的赫布学习（候选）
    └── Engine C: A 的结构 + B 的 α 调制（折中）
    ↓
P/R/Xin/U 转换边界判定
    ↓
RLIS 账本 / Xin 守恒检查
    ↓
A/B Stress Benchmark → 晋升决策
```

## 二、核心概念框架（这是项目最有价值的部分）

### P/R/X 三元分类范式

这是 Morphosphere 最原创的思想：

| 分类 | 含义 | 物理类比 |
|------|------|---------|
| **P (Persistence)** | 已经稳定下来的结构/记忆 | 固态 — 冻结的冰 |
| **R (Reactivity)** | 正在形成中的响应/关联 | 液态 — 流动的水 |
| **Xin (不确定性残余)** | 系统尚无法处理的噪声/冲突 | 气态 — 蒸发的水汽 |

系统强制执行一个热力学相变路径：**Xin → R → P**（气→液→固），不允许跳过中间态。这通过代码中的"Markov Blanket Iron Law"实现——P 状态的冻结**必须**有对应的 R 前驱。

### 拓扑惯性质量 M_eff

Engine B 的核心创新：每条赫布连接不只有权重 w，还有一个"惯性质量" M_eff：

- 外部输入（跨域数据）增加 M_eff → 连接变得"更重"→ 更难被全局衰减删除
- 内部输入增加 w 但不增加 M_eff → 连接容易被衰减
- 当 M_eff > 0.9 × M_max → 节点被判定为"死亡"（过度刚性 → 惯性奇点）

这实质上是一个**信息老化模型**：知识既可以因为太少证据而消亡（衰减），也可以因为太多矛盾证据而僵死（奇点）。

### 运动体制识别（MotionRegimeOracle）

从细胞的时空位移中识别 6 种运动体制：

```
stationary  — 均值位移 < 1.0（不动）
slow_drift  — 位移 1.0-5.0（缓慢漂移）
fast_drift  — 位移 5.0-15.0（快速漂移）
oscillation — 负自相关 → 交替方向（振荡/极限环）
jump        — 突然大位移 > 15.0（跳跃）
diffusion   — 高方差 + 非负自相关（随机游走）
```

这个分类器使用 lag-1 自相关来区分振荡和扩散 — 这是物理学上的正确做法。

## 三、项目的工程成熟度

### 已经做到的 ✅

1. **真实数据接入** — CTC (Cell Tracking Challenge, Fluo-N2DH-GOWT1) 的真实细胞追踪数据，有 DOI 和 CC-BY-4.0 许可
2. **变分 EM 优化** — 不再是固定权重的 softmax，而是有 E-step/M-step 的迭代参数估计
3. **闭环赫布学习** — 基于性能反馈（ΔJ + 运动识别准确率）的定向权重更新
4. **数据集分离** — calibration/validation/holdout 三集隔离，holdout 使用不同序列
5. **非平凡收敛** — drift > 0（前几轮有真实漂移）然后收敛
6. **全面的审计治理** — 116 张表、溯源哈希、节点死亡 DNA 快照、Xin 生命周期闭环
7. **48 项自动化检查** — A/B/C 三引擎 40 项 + 集成管线 8 项，全部通过

### 仍然存在的局限 ⚠️

1. **数据规模有限** — 只有 CTC 一个真实数据集（2D 荧光成像），还没有接入 Allen Brain Observatory 的钙成像数据
2. **"变分"仍然简化** — 有 EM 迭代，但不是严格的纤维丛变分法（没有 Euler-Lagrange 方程）
3. **传输不是最优传输** — 用的是欧氏距离 + 自适应阈值，不是 Wasserstein 距离
4. **v37.5 晋升被阻塞** — 需要 class_diversity ≥ 3 + motion_regimes ≥ 5 才能解锁下一阶段

## 五、诚实评估

项目自身已经有一份极其坦诚的自我评估（`REPORT_诚实评估.md`），核心结论是：

> **这是一个有野心的概念原型的脚手架。** 哲学-数学框架有原创想法，工程骨架可以运行，但要声称"做到了尚未有人尝试的东西"，需要用真实数据证明 P/R/X 分解能产生现有方法无法产生的发现。

| 维度 | 状态 |
|------|------|
| 概念框架原创性 | ⭐⭐⭐⭐ — P/R/X 范式 + 信息热力学类比是有价值的 |
| 工程完整度 | ⭐⭐⭐⭐ — 116 张表、48 项检查、全链路溯源 |
| 数学严格度 | ⭐⭐⭐ — 有 EM 迭代，但远未达到变分几何的严格标准 |
| 真实数据验证 | ⭐⭐⭐ — 已接入 CTC 真实数据，但仅一个数据源 |
| 学术发表就绪 | ⭐⭐ — 需要多数据源 + 与基线方法对比 + 统计检验 |

## 六、总结

**Morphosphere 本质上是一个"信息物理学"实验平台**。它试图用物理学的语言（相变、守恒律、惯性质量、极限环）来描述信息系统中的模式形成和消亡。

它能做什么：
- 从真实的细胞追踪数据中识别运动体制
- 用竞争性赫布学习发现稳定的关联结构
- 追踪每个"知识单元"的完整生命周期（诞生→成长→死亡→验尸）
- 通过变分 EM 优化自动调整多源融合权重

它还不能做什么：
- 在真实生物学数据上产生超越现有方法的发现
- 声称数学上的严格性（变分、最优传输等术语的实现低于其数学定义）
- 处理大规模数据（当前仅几千个数据点级别）

**一句话：这是一个将物理学直觉编码为可运行代码的概念验证系统，有原创的哲学框架和扎实的工程骨架，正在从"能跑通的原型"向"能证明其价值的系统"过渡中。**
